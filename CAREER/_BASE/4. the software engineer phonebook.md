# The Software Engineer Phonebook

**A Comprehensive Reference for the .NET Backend Software Engineer**

_This is not a roadmap. It is a phonebook. Every entry contains the actual detail — the definition, the mechanism, the tradeoffs, the .NET implementation, the gotchas, and the real-world context. Use it like a dictionary: look up what you need, read the full entry, understand the thing completely._

_Built from: The Software Engineer, The Action Map, The Engineering Curriculum — all three distilled into one encyclopedic reference._

---

**How to Use This Document**

Each entry follows a consistent structure:
- **What it is** — precise definition
- **How it works** — the mechanism underneath
- **Why it matters** — the engineering consequence
- **In .NET / C#** — the concrete implementation
- **Tradeoffs** — what you gain and what you give up
- **Gotchas** — what trips people up
- **See also** — related entries

The document is organized into categories. Within each category, entries are alphabetical or logical. Use Obsidian search (Ctrl+F) to find any specific term.
