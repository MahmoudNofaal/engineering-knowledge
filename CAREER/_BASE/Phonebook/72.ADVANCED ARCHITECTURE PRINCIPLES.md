# CATEGORY 72 — ADVANCED ARCHITECTURE PRINCIPLES

---

## 72.01 — Conway's Law

**"Organizations which design systems are constrained to produce designs which are copies of the communication structures of these organizations."**

**Implication**: Your software architecture mirrors your team structure. If you have a database team, a UI team, and a backend team — your system will naturally have those three horizontal layers.

**Inverse Conway Maneuver**
Design your organization to match the architecture you want:
- Want microservices by domain? Create teams by domain (Orders team, Products team, Payments team)
- Want service ownership? Each team owns full stack for their domain

**In practice**
```
Team: Orders team (2 engineers)
  Owns: Orders microservice (backend + DB)
  Deploys: Independently
  API: Orders REST API
  DB: Orders database (only this team accesses it directly)
  
Team: Products team (3 engineers)
  Owns: Products microservice
  
Teams communicate via: API contracts, not direct database access
```

## 72.02 — Evolutionary Architecture

**What it is**
Design systems so they can change over time. Architecture is never "done" — it evolves as requirements change.

**Fitness functions (automated architectural checks)**
```csharp
// Architecture tests that run in CI — enforce rules automatically

// "No class in Application layer imports from Infrastructure"
[Fact]
public void Application_MustNotReferenceInfrastructure()
{
    Types.InAssembly(ApplicationAssembly)
        .Should()
        .NotHaveDependencyOn(InfrastructureAssembly.FullName)
        .GetResult()
        .IsSuccessful
        .Should().BeTrue();
}

// "All public API endpoints must be versioned"
[Fact]
public void AllEndpoints_MustBeVersioned()
{
    var endpoints = app.Routes
        .Where(r => r.RoutePattern.RawText?.StartsWith("/api/") == true);
    
    foreach (var endpoint in endpoints)
        endpoint.RoutePattern.RawText
            .Should().MatchRegex(@"^/api/v\d+/");
}

// "Response time for critical endpoints < 200ms (P99)"
// Measured in load tests, not unit tests
```
