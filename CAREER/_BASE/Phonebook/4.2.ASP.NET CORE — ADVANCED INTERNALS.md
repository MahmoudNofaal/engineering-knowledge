## 1. HTTPCONTEXT — THE FULL PICTURE

### What It Actually Is
`HttpContext` is not an object you construct — it is the living record of a single HTTP request/response cycle, created by Kestrel and passed through the entire middleware chain. Everything about a request — the path, headers, body, user identity, response status, and per-request services — lives on `HttpContext`. Understanding every property prevents you from re-implementing things ASP.NET Core already provides.

### How It Works — Mechanically

`HttpContext` is implemented by `DefaultHttpContext`. Most of its properties delegate to the `IFeatureCollection` — a bag of feature interfaces populated by Kestrel during request parsing (see §2). When you access `context.Request.Path`, you're actually calling into `IHttpRequestFeature.Path` from the feature collection.

```
HttpContext
├── Request  (IHttpRequestFeature, IQueryFeature, IFormFeature, IRouteValuesFeature)
├── Response (IHttpResponseFeature, IHttpResponseBodyFeature)
├── User     (IHttpAuthenticationFeature.User)
├── Session  (ISessionFeature.Session)
├── Items    (Dictionary<object,object> — per-request scratch space)
├── Features (IFeatureCollection — the raw feature bag)
├── RequestServices (IServiceProvider — the request's DI scope)
├── Connection (IHttpConnectionFeature)
├── WebSockets (IHttpWebSocketFeature)
└── TraceIdentifier (string — unique ID for this request)
```

### The Real-World Code

```csharp
// ── Everything HttpContext exposes — reference card ───────────────────────────
app.Use(async (HttpContext ctx, RequestDelegate next) =>
{
    // ── Request ───────────────────────────────────────────────────────────────
    string method     = ctx.Request.Method;           // "GET", "POST", ...
    PathString path   = ctx.Request.Path;             // "/api/orders"
    string? query     = ctx.Request.QueryString.Value; // "?page=1"
    IQueryCollection qs = ctx.Request.Query;          // Parsed query params
    IHeaderDictionary headers = ctx.Request.Headers;  // All request headers
    Stream body       = ctx.Request.Body;             // Raw body stream
    long? bodyLen     = ctx.Request.ContentLength;
    string? ct        = ctx.Request.ContentType;
    bool isHttps      = ctx.Request.IsHttps;
    string scheme     = ctx.Request.Scheme;           // "https"
    HostString host   = ctx.Request.Host;

    // Route values (populated after UseRouting)
    object? id = ctx.GetRouteValue("id");
    var routeData = ctx.GetRouteData();

    // Form data (only valid if Content-Type is form)
    if (ctx.Request.HasFormContentType)
    {
        IFormCollection form = await ctx.Request.ReadFormAsync();
        IFormFileCollection files = form.Files;
    }

    // ── Connection ────────────────────────────────────────────────────────────
    IPAddress? remoteIp = ctx.Connection.RemoteIpAddress;
    int remotePort      = ctx.Connection.RemotePort;
    string connectionId = ctx.Connection.Id;

    // ── User identity ─────────────────────────────────────────────────────────
    ClaimsPrincipal user   = ctx.User;
    bool isAuthenticated   = user.Identity?.IsAuthenticated ?? false;
    string? userId         = user.FindFirstValue(ClaimTypes.NameIdentifier);
    bool isAdmin           = user.IsInRole("Admin");

    // ── Per-request state ─────────────────────────────────────────────────────
    // Items: pass data between middleware without polluting method signatures
    ctx.Items["correlationId"] = Guid.NewGuid().ToString("N");
    ctx.Items.TryGetValue("correlationId", out var corrId);

    // ── DI scope ──────────────────────────────────────────────────────────────
    // RequestServices is the scoped container for this request
    var service = ctx.RequestServices.GetRequiredService<IOrderService>();

    // ── Abort token ──────────────────────────────────────────────────────────
    // Fires when the client disconnects — stop processing early
    CancellationToken aborted = ctx.RequestAborted;

    await next(ctx);

    // ── Response ──────────────────────────────────────────────────────────────
    ctx.Response.StatusCode = StatusCodes.Status200OK;
    ctx.Response.ContentType = "application/json";
    ctx.Response.Headers.Append("X-Custom-Header", "value");

    // OnStarting: runs just before first byte is written — last chance to set headers
    ctx.Response.OnStarting(() =>
    {
        ctx.Response.Headers["X-Request-Id"] = ctx.TraceIdentifier;
        return Task.CompletedTask;
    });

    // OnCompleted: runs after response is fully sent — for cleanup
    ctx.Response.OnCompleted(() =>
    {
        // Don't write to response here — it's already sent
        // Good for: releasing resources, logging completion
        return Task.CompletedTask;
    });

    await ctx.Response.WriteAsJsonAsync(new { ok = true });
});
```

## 2. IFEATURECOLLECTION — UNDER THE HOOD

### What It Actually Is
`IFeatureCollection` is the extensibility backbone of the ASP.NET Core request pipeline. It's a type-keyed dictionary: `Get<IHttpRequestFeature>()` returns the object implementing that feature. Kestrel populates it. Middleware can read, replace, or add features. This is how the framework is composed without inheritance hierarchies — any middleware can swap out any feature implementation.

### How It Works — Mechanically

```
Kestrel parses TCP bytes
     │
     ├── Sets IHttpRequestFeature  (method, path, headers, body)
     ├── Sets IHttpResponseFeature (status code, headers)
     ├── Sets IHttpResponseBodyFeature (body pipe)
     ├── Sets IHttpConnectionFeature (remote IP, port)
     └── Sets IHttpWebSocketFeature (upgrade capability)

HttpContext wraps IFeatureCollection:
  ctx.Request.Path == ctx.Features.Get<IHttpRequestFeature>()!.Path
```

### The Real-World Code

```csharp
// ── Reading features directly ─────────────────────────────────────────────────
app.Use(async (ctx, next) =>
{
    var requestFeature = ctx.Features.Get<IHttpRequestFeature>()!;
    var connectionFeature = ctx.Features.Get<IHttpConnectionFeature>();
    var tlsFeature = ctx.Features.Get<ITlsConnectionFeature>();

    // TLS feature: access client certificate
    if (tlsFeature is not null)
    {
        var cert = await tlsFeature.GetClientCertificateAsync(ctx.RequestAborted);
        if (cert is not null)
            ctx.Items["ClientCert"] = cert;
    }

    await next(ctx);
});


// ── Replacing a feature — swap the response body for buffering ────────────────
public class ResponseBufferingMiddleware
{
    private readonly RequestDelegate _next;

    public ResponseBufferingMiddleware(RequestDelegate next) => _next = next;

    public async Task InvokeAsync(HttpContext context)
    {
        var originalBody = context.Response.Body;
        using var buffer = new MemoryStream();

        // Replace the body feature so downstream writes to our buffer
        context.Response.Body = buffer;

        await _next(context);

        buffer.Position = 0;
        var responseBody = await new StreamReader(buffer).ReadToEndAsync();

        // Log or transform the response body
        buffer.Position = 0;

        // Restore and copy to the real response
        context.Response.Body = originalBody;
        await buffer.CopyToAsync(originalBody, context.RequestAborted);
    }
}


// ── Custom feature: add per-request tenant resolution ─────────────────────────
public interface ITenantFeature
{
    TenantInfo Tenant { get; }
}

public class TenantFeature : ITenantFeature
{
    public TenantInfo Tenant { get; }
    public TenantFeature(TenantInfo tenant) => Tenant = tenant;
}

public class TenantResolutionMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ITenantResolver _resolver;

    public TenantResolutionMiddleware(RequestDelegate next, ITenantResolver resolver)
    {
        _next = next;
        _resolver = resolver;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        var tenant = await _resolver.ResolveAsync(context);
        if (tenant is null)
        {
            context.Response.StatusCode = StatusCodes.Status400BadRequest;
            await context.Response.WriteAsJsonAsync(new { error = "Unknown tenant" });
            return;
        }

        // Add to feature collection — accessible anywhere in the pipeline
        context.Features.Set<ITenantFeature>(new TenantFeature(tenant));

        await _next(context);
    }
}

// Consuming the feature downstream
app.Use(async (ctx, next) =>
{
    var tenantFeature = ctx.Features.Get<ITenantFeature>();
    // tenantFeature is null if TenantResolutionMiddleware didn't run first
    await next(ctx);
});
```

## 3. CUSTOM MODEL BINDERS

### What It Actually Is
Model binding is the mechanism that populates action method parameters from the HTTP request. Custom model binders let you intercept this process for specific types — binding from unusual sources (e.g., a JWT claim, a custom header, a comma-separated string) or custom parsing logic. `IModelBinder` does the work; `IModelBinderProvider` tells ASP.NET Core when to use your binder.

### How It Works — Mechanically

1. For each action parameter, ASP.NET Core asks each registered `IModelBinderProvider` (in order) whether it can bind this parameter type.
2. The first provider that returns non-null binder wins.
3. `IModelBinder.BindModelAsync(ModelBindingContext)` runs, reading from the value providers (route, query, form, body) and setting `context.Result`.

### The Real-World Code

```csharp
// ── Custom binder: bind comma-separated query string to a typed list ──────────
// GET /api/orders?productIds=1,2,3,4

public class CommaDelimitedArrayModelBinder : IModelBinder
{
    public Task BindModelAsync(ModelBindingContext bindingContext)
    {
        var modelName = bindingContext.ModelName;
        var valueProviderResult = bindingContext.ValueProvider.GetValue(modelName);

        if (valueProviderResult == ValueProviderResult.None)
        {
            // No value provided — binding succeeds with null (let validation handle required)
            bindingContext.Result = ModelBindingResult.Success(null);
            return Task.CompletedTask;
        }

        bindingContext.ModelState.SetModelValue(modelName, valueProviderResult);

        var value = valueProviderResult.FirstValue;
        if (string.IsNullOrEmpty(value))
        {
            bindingContext.Result = ModelBindingResult.Success(Array.Empty<int>());
            return Task.CompletedTask;
        }

        // Parse "1,2,3" → int[]
        var elementType = bindingContext.ModelType.GetElementType()
            ?? bindingContext.ModelType.GetGenericArguments().FirstOrDefault()
            ?? typeof(string);

        try
        {
            var parts = value.Split(',', StringSplitOptions.RemoveEmptyEntries);
            var result = parts
                .Select(p => Convert.ChangeType(p.Trim(), elementType))
                .ToArray();

            // Create a properly-typed array
            var typedArray = Array.CreateInstance(elementType, result.Length);
            result.CopyTo(typedArray, 0);

            bindingContext.Result = ModelBindingResult.Success(typedArray);
        }
        catch (Exception ex)
        {
            bindingContext.ModelState.TryAddModelError(
                modelName, $"Cannot parse '{value}' as {elementType.Name} array: {ex.Message}");
            bindingContext.Result = ModelBindingResult.Failed();
        }

        return Task.CompletedTask;
    }
}

public class CommaDelimitedArrayModelBinderProvider : IModelBinderProvider
{
    public IModelBinder? GetBinder(ModelBinderProviderContext context)
    {
        if (context.Metadata.ModelType.IsArray ||
            (context.Metadata.ModelType.IsGenericType &&
             context.Metadata.ModelType.GetGenericTypeDefinition() == typeof(List<>)))
        {
            return new CommaDelimitedArrayModelBinder();
        }
        return null;
    }
}

// Register
builder.Services.AddControllers(options =>
    // Insert at 0 to run before built-in providers
    options.ModelBinderProviders.Insert(0, new CommaDelimitedArrayModelBinderProvider()));

// Use via attribute
[HttpGet]
public async Task<IActionResult> GetOrders(
    [ModelBinder(typeof(CommaDelimitedArrayModelBinder))] int[] productIds)
{
    // productIds = [1, 2, 3, 4] from ?productIds=1,2,3,4
}


// ── Binder from JWT claims ────────────────────────────────────────────────────
[AttributeUsage(AttributeTargets.Parameter)]
public class FromClaimAttribute : Attribute, IBindingSourceMetadata
{
    public string ClaimType { get; }
    public BindingSource BindingSource => BindingSource.Special;

    public FromClaimAttribute(string claimType) => ClaimType = claimType;
}

public class ClaimModelBinder : IModelBinder
{
    private readonly string _claimType;
    public ClaimModelBinder(string claimType) => _claimType = claimType;

    public Task BindModelAsync(ModelBindingContext ctx)
    {
        var user = ctx.HttpContext.User;
        var value = user.FindFirstValue(_claimType);

        if (value is null)
        {
            ctx.ModelState.TryAddModelError(
                ctx.ModelName, $"Claim '{_claimType}' not found in token");
            ctx.Result = ModelBindingResult.Failed();
            return Task.CompletedTask;
        }

        // Convert to target type
        try
        {
            var converted = Convert.ChangeType(value, ctx.ModelType);
            ctx.Result = ModelBindingResult.Success(converted);
        }
        catch
        {
            ctx.ModelState.TryAddModelError(
                ctx.ModelName, $"Cannot convert claim value '{value}' to {ctx.ModelType.Name}");
            ctx.Result = ModelBindingResult.Failed();
        }

        return Task.CompletedTask;
    }
}

// Use in action
[HttpGet("my-orders")]
public async Task<IActionResult> GetMyOrders(
    [FromClaim("user_id")] int userId)  // populated from JWT claim
{ /* ... */ }
```

## 4. CUSTOM ACTION RESULTS & IRESULT

### What It Actually Is
`IActionResult` (MVC controllers) and `IResult` (Minimal APIs) are the abstractions for "something that writes an HTTP response." The built-in results cover 95% of cases, but custom results let you encode complex response patterns (multi-part, custom content types, CSP headers, signed responses) without polluting action methods with response-writing code.

### The Real-World Code

```csharp
// ── Custom IActionResult: signed JSON response ────────────────────────────────
public class SignedJsonResult : IActionResult
{
    private readonly object _value;
    private readonly string _signingKey;

    public SignedJsonResult(object value, string signingKey)
    {
        _value = value;
        _signingKey = signingKey;
    }

    public async Task ExecuteResultAsync(ActionContext context)
    {
        var response = context.HttpContext.Response;
        var json = JsonSerializer.Serialize(_value, new JsonSerializerOptions
        {
            PropertyNamingPolicy = JsonNamingPolicy.CamelCase
        });

        // HMAC-sign the body and add signature header
        var signature = ComputeHmac(json, _signingKey);

        response.StatusCode = StatusCodes.Status200OK;
        response.ContentType = "application/json";
        response.Headers.Append("X-Body-Signature", signature);
        response.ContentLength = Encoding.UTF8.GetByteCount(json);

        await response.WriteAsync(json, context.HttpContext.RequestAborted);
    }

    private static string ComputeHmac(string data, string key)
    {
        var keyBytes = Encoding.UTF8.GetBytes(key);
        var dataBytes = Encoding.UTF8.GetBytes(data);
        using var hmac = new HMACSHA256(keyBytes);
        return Convert.ToBase64String(hmac.ComputeHash(dataBytes));
    }
}


// ── Custom IResult for Minimal APIs: CSV download ─────────────────────────────
public static class CsvResults
{
    public static IResult Csv<T>(IEnumerable<T> data, string filename)
        => new CsvResult<T>(data, filename);
}

public sealed class CsvResult<T> : IResult
{
    private readonly IEnumerable<T> _data;
    private readonly string _filename;

    public CsvResult(IEnumerable<T> data, string filename)
    {
        _data = data;
        _filename = filename;
    }

    public async Task ExecuteAsync(HttpContext httpContext)
    {
        var response = httpContext.Response;
        response.ContentType = "text/csv";
        response.Headers.ContentDisposition = $"attachment; filename=\"{_filename}\"";
        response.StatusCode = StatusCodes.Status200OK;

        await using var writer = new StreamWriter(response.Body, leaveOpen: true);

        // Write header from property names
        var properties = typeof(T).GetProperties(BindingFlags.Public | BindingFlags.Instance);
        await writer.WriteLineAsync(string.Join(",", properties.Select(p => p.Name)));

        // Write rows
        foreach (var item in _data)
        {
            var values = properties.Select(p =>
            {
                var val = p.GetValue(item)?.ToString() ?? "";
                // Escape commas and quotes in CSV
                return val.Contains(',') || val.Contains('"')
                    ? $"\"{val.Replace("\"", "\"\"")}\""
                    : val;
            });
            await writer.WriteLineAsync(string.Join(",", values));
        }
    }
}

// Usage
app.MapGet("/api/orders/export", async (ISender mediator, CancellationToken ct) =>
{
    var orders = await mediator.Send(new GetAllOrdersQuery(), ct);
    return CsvResults.Csv(orders, $"orders-{DateTime.UtcNow:yyyyMMdd}.csv");
}).RequireAuthorization("AdminOnly");


// ── TypedResults — preferred for Minimal APIs (.NET 7+) ───────────────────────
// TypedResults enables OpenAPI schema inference without attributes
app.MapGet("/api/products/{id:int}", async (int id, IProductRepo repo, CancellationToken ct) =>
{
    var product = await repo.GetByIdAsync(id, ct);
    // Return type is Results<Ok<ProductDto>, NotFound>
    // Swagger generates the correct response schemas for both 200 and 404
    return product is null
        ? TypedResults.NotFound()
        : TypedResults.Ok(product);
});
```

## 5. DATA PROTECTION API

### What It Actually Is
The Data Protection API (`IDataProtector`) provides cryptographic operations for protecting data that you need to unprotect later — think: protecting cookie values, anti-forgery tokens, email confirmation links, password reset tokens. It manages key rotation, storage, encryption algorithms, and data lifetime — things you should **not** implement yourself. It is *not* for hashing passwords (use `PasswordHasher`) or general-purpose encryption.

### How It Works — Mechanically

1. You create a `protector = provider.CreateProtector("purpose")`. The purpose string is part of the key derivation — data protected with one purpose cannot be unprotected with another, even using the same master key.
2. `protector.Protect(plaintext)` → encrypts + authenticates, returns a base64 opaque payload.
3. `protector.Unprotect(ciphertext)` → authenticates + decrypts, returns plaintext. Throws `CryptographicException` if tampered, wrong purpose, or expired.
4. Keys are stored (file system, Azure Key Vault, Redis, DB) and rotated every 90 days by default.

### The Real-World Code

```csharp
// ── Registration ──────────────────────────────────────────────────────────────
builder.Services.AddDataProtection()
    .PersistKeysToDbContext<AppDbContext>()   // Store keys in database
    .SetApplicationName("MyApp")              // Must match across all instances
    .SetDefaultKeyLifetime(TimeSpan.FromDays(90))
    .ProtectKeysWithAzureKeyVault(
        new Uri(configuration["KeyVault:DataProtectionKeyId"]!),
        new DefaultAzureCredential());        // Encrypt the keys at rest


// ── Email confirmation token ──────────────────────────────────────────────────
public class EmailConfirmationService
{
    private readonly ITimeLimitedDataProtector _protector;

    public EmailConfirmationService(IDataProtectionProvider provider)
    {
        // Time-limited protector: tokens expire after specified duration
        _protector = provider
            .CreateProtector("EmailConfirmation.v1")
            .ToTimeLimitedDataProtector();
    }

    public string GenerateConfirmationToken(int userId, string email)
    {
        // Protect with a 48-hour expiry
        var payload = $"{userId}:{email}";
        return _protector.Protect(payload, TimeSpan.FromHours(48));
    }

    public (int UserId, string Email)? ValidateToken(string token)
    {
        try
        {
            // Throws CryptographicException if expired or tampered
            var payload = _protector.Unprotect(token, out var expiry);

            if (expiry <= DateTimeOffset.UtcNow)
                return null; // Expired (belt and suspenders check)

            var parts = payload.Split(':');
            return (int.Parse(parts[0]), parts[1]);
        }
        catch (CryptographicException)
        {
            return null; // Invalid or expired token
        }
    }
}


// ── Protecting query string values (prevent tampering) ───────────────────────
public class OrderDownloadController : ControllerBase
{
    private readonly IDataProtector _protector;

    public OrderDownloadController(IDataProtectionProvider provider)
        => _protector = provider.CreateProtector("OrderDownload");

    // Generate a protected download link
    [HttpGet("{id:int}/download-link")]
    public IActionResult GetDownloadLink(int id)
    {
        var token = _protector.Protect($"order:{id}:{DateTime.UtcNow:O}");
        var url = Url.Action("Download", new { token })!;
        return Ok(new { url });
    }

    // Validate the token before serving the file
    [HttpGet("download")]
    [AllowAnonymous]
    public IActionResult Download(string token)
    {
        try
        {
            var payload = _protector.Unprotect(token);
            var parts = payload.Split(':');
            if (parts[0] != "order") return BadRequest("Invalid download token");
            var orderId = int.Parse(parts[1]);
            // Serve the file...
            return File(Array.Empty<byte>(), "application/pdf", $"order-{orderId}.pdf");
        }
        catch (CryptographicException)
        {
            return BadRequest("Invalid or expired download token");
        }
    }
}
```

## 6. ANTIFORGERY (CSRF)

### What It Actually Is
Cross-Site Request Forgery (CSRF) is an attack where a malicious site makes authenticated requests to your API using the victim's browser cookies. Antiforgery protection breaks this by requiring a secret token that the malicious site cannot read (due to Same-Origin Policy). It matters for cookie-based auth. **JWT bearer tokens in Authorization headers are immune to CSRF** — browsers don't automatically send them, so CSRF attacks can't leverage them.

### How It Works — Mechanically

1. Server generates two tokens per session: a cookie token and a request (form/header) token. They're cryptographically linked.
2. On form submit/POST, both are sent: cookie auto-attaches (browser behavior), request token is in the form or header (your code must send it).
3. Server validates: both tokens present? Do they match? If not → 400.
4. The attacker's site gets the cookie (browser sends it automatically) but cannot read the request token (SOP blocks reading cross-origin responses).

### The Real-World Code

```csharp
// ── Registration ──────────────────────────────────────────────────────────────
builder.Services.AddAntiforgery(options =>
{
    options.HeaderName = "X-XSRF-TOKEN";        // SPA reads this header name
    options.Cookie.Name = "XSRF-TOKEN";         // Angular/Axios read this cookie
    options.Cookie.SameSite = SameSiteMode.Strict;
    options.Cookie.SecurePolicy = CookieSecurePolicy.Always;
    options.Cookie.HttpOnly = false;            // SPA JavaScript must READ this cookie
});                                             // NOTE: the cookie is not sensitive — the token value IS

// Endpoint to get a fresh token (SPAs call this on startup)
app.MapGet("/api/antiforgery/token", (IAntiforgery antiforgery, HttpContext ctx) =>
{
    var tokens = antiforgery.GetAndStoreTokens(ctx);
    return Results.Ok(new { token = tokens.RequestToken });
}).RequireAuthorization().AllowAnonymous();


// ── Validating antiforgery in middleware (for global protection) ──────────────
app.Use(async (ctx, next) =>
{
    // Only validate state-changing methods, only for cookie-authenticated requests
    if (HttpMethods.IsPost(ctx.Request.Method) ||
        HttpMethods.IsPut(ctx.Request.Method) ||
        HttpMethods.IsDelete(ctx.Request.Method) ||
        HttpMethods.IsPatch(ctx.Request.Method))
    {
        // Skip for API key or JWT bearer auth (they're CSRF-immune)
        if (ctx.User.Identity?.IsAuthenticated == true &&
            ctx.User.FindFirstValue("auth_method") == "cookie")
        {
            var antiforgery = ctx.RequestServices.GetRequiredService<IAntiforgery>();
            await antiforgery.ValidateRequestAsync(ctx); // Throws AntiforgeryValidationException on failure
        }
    }

    await next(ctx);
});


// ── Per-controller with attribute ─────────────────────────────────────────────
[ApiController]
[AutoValidateAntiforgeryToken]  // Validates for all non-GET methods
public class AccountController : ControllerBase
{
    [HttpPost("change-password")]
    public async Task<IActionResult> ChangePassword(ChangePasswordRequest request) { /* ... */ return Ok(); }

    [HttpPost("logout")]
    [ValidateAntiForgeryToken]  // Explicit per-action
    public async Task<IActionResult> Logout() { /* ... */ return Ok(); }
}
```

## 7. GRPC IN ASP.NET CORE

### What It Actually Is
gRPC is a high-performance, binary RPC framework using HTTP/2 and Protocol Buffers. Where REST sends JSON over HTTP/1.1 (text, stateless, request-response), gRPC sends Protobuf over HTTP/2 (binary, streaming, multiplexed). In ASP.NET Core, `Grpc.AspNetCore` hosts gRPC services. Use gRPC for internal service-to-service communication where throughput and latency matter. Use REST for public-facing APIs.

### How It Works — Mechanically

1. You define services and messages in `.proto` files.
2. The `Grpc.Tools` build package generates C# server base classes and client stubs from the `.proto`.
3. Your service class implements the generated abstract base, overriding the RPC methods.
4. `MapGrpcService<T>()` registers the service with HTTP/2 endpoint routing.
5. gRPC frame format: each message is length-prefixed and compressed. HTTP/2 trailers carry status codes.

**Four call types:**
```
Unary:           client sends 1 message, server replies 1 message
Server streaming: client sends 1 message, server streams N messages
Client streaming: client streams N messages, server replies 1 message
Bidirectional:   both sides stream simultaneously
```

### The Real-World Code

```proto
// ── orders.proto ─────────────────────────────────────────────────────────────
syntax = "proto3";
option csharp_namespace = "MyApp.Grpc";

package orders;

service OrderService {
  rpc GetOrder (GetOrderRequest) returns (OrderResponse);
  rpc StreamOrderUpdates (OrderStreamRequest) returns (stream OrderStatusUpdate);
  rpc BatchCreateOrders (stream CreateOrderRequest) returns (BatchCreateResponse);
}

message GetOrderRequest  { int32 order_id = 1; }
message OrderResponse    { int32 id = 1; string status = 2; double total = 3; }
message OrderStreamRequest { int32 customer_id = 1; }
message OrderStatusUpdate  { int32 order_id = 1; string status = 2; string timestamp = 3; }
message CreateOrderRequest { int32 customer_id = 1; repeated OrderItem items = 2; }
message OrderItem          { int32 product_id = 1; int32 quantity = 2; }
```

## 8. SERVER-SENT EVENTS & RESPONSE STREAMING

### What It Actually Is
Server-Sent Events (SSE) is a simple HTTP-based protocol for server-to-client one-way streaming. The server keeps the response body open and writes `data: ...\n\n` lines. The browser's `EventSource` API reconnects automatically on disconnection. SSE is simpler than WebSockets for one-way push (notifications, progress, live feeds). It works through proxies and uses regular HTTP.

### The Real-World Code

```csharp
// ── SSE endpoint — live job progress ─────────────────────────────────────────
app.MapGet("/api/jobs/{jobId}/progress", async (
    string jobId,
    IJobProgressService progress,
    HttpContext ctx,
    CancellationToken ct) =>
{
    ctx.Response.ContentType = "text/event-stream";
    ctx.Response.Headers.CacheControl = "no-cache";
    ctx.Response.Headers.Connection = "keep-alive";
    // Disable response buffering — bytes must go out immediately
    ctx.Response.Headers["X-Accel-Buffering"] = "no"; // For Nginx

    await ctx.Response.Body.FlushAsync(ct);

    await foreach (var update in progress.WatchAsync(jobId, ct))
    {
        // SSE format: "data: {json}\n\n"
        var json = JsonSerializer.Serialize(update);
        await ctx.Response.WriteAsync($"data: {json}\n\n", ct);
        await ctx.Response.Body.FlushAsync(ct); // Must flush each event immediately

        if (update.IsComplete) break;
    }
}).RequireAuthorization();


// ── Streaming JSON response (.NET 8: IAsyncEnumerable support) ───────────────
// Returns NDJSON (newline-delimited JSON) — each line is a complete JSON object
app.MapGet("/api/orders/stream", async IAsyncEnumerable<OrderSummaryDto> (
    IOrderRepository repo,
    CancellationToken ct) =>
{
    await foreach (var order in repo.StreamAllAsync(ct))
    {
        yield return order;
    }
}).RequireAuthorization();
// ASP.NET Core serializes each yielded item as a separate JSON line
// Client reads line-by-line with ReadLineAsync


// ── Low-level response streaming ──────────────────────────────────────────────
app.MapGet("/api/reports/large", async (
    IReportService reports,
    HttpContext ctx,
    CancellationToken ct) =>
{
    ctx.Response.ContentType = "application/json";
    ctx.Response.StatusCode = StatusCodes.Status200OK;

    // Start writing immediately — don't wait for all data before sending headers
    await using var writer = new Utf8JsonWriter(ctx.Response.Body);
    writer.WriteStartArray();

    await foreach (var record in reports.StreamAsync(ct))
    {
        JsonSerializer.Serialize(writer, record);
        // Flush periodically — don't buffer the entire response
        if (writer.BytesCommitted > 64 * 1024) // Every 64 KB
            await ctx.Response.Body.FlushAsync(ct);
    }

    writer.WriteEndArray();
    await writer.FlushAsync(ct);
});
```

## 9. RAW WEBSOCKETS

### What It Actually Is
Raw WebSocket support in ASP.NET Core (below SignalR) gives you a bidirectional, persistent TCP connection. You handle the framing yourself. Use raw WebSockets when you need full control over the wire protocol (binary frames, custom subprotocol) or when you're building a game server, relay, or proxy. For most apps, SignalR is the right abstraction.

### The Real-World Code

```csharp
// ── WebSocket handler ─────────────────────────────────────────────────────────
app.UseWebSockets(new WebSocketOptions
{
    KeepAliveInterval = TimeSpan.FromSeconds(30),
    AllowedOrigins = { "https://app.example.com" }
});

app.Map("/ws", async (HttpContext ctx) =>
{
    if (!ctx.WebSockets.IsWebSocketRequest)
    {
        ctx.Response.StatusCode = StatusCodes.Status400BadRequest;
        await ctx.Response.WriteAsync("WebSocket upgrade required");
        return;
    }

    // Negotiate subprotocol
    var subProtocol = ctx.WebSockets.WebSocketRequestedProtocols
        .FirstOrDefault(p => p == "myapp-v1");

    using var ws = await ctx.WebSockets.AcceptWebSocketAsync(subProtocol);

    await HandleWebSocketAsync(ws, ctx.RequestAborted);
});

async Task HandleWebSocketAsync(WebSocket ws, CancellationToken ct)
{
    // 4 KB receive buffer — reused across receives
    var buffer = new byte[4 * 1024];

    while (ws.State == WebSocketState.Open)
    {
        WebSocketReceiveResult result;
        using var messageBuffer = new MemoryStream();

        // Receive a complete message (may arrive in multiple frames)
        do
        {
            result = await ws.ReceiveAsync(buffer, ct);

            if (result.MessageType == WebSocketMessageType.Close)
            {
                await ws.CloseAsync(
                    WebSocketCloseStatus.NormalClosure,
                    "Closing",
                    ct);
                return;
            }

            await messageBuffer.WriteAsync(buffer.AsMemory(0, result.Count), ct);
        }
        while (!result.EndOfMessage);

        messageBuffer.Position = 0;
        var message = JsonSerializer.Deserialize<WsMessage>(messageBuffer);

        // Process message and send response
        var response = ProcessMessage(message!);
        var responseBytes = JsonSerializer.SerializeToUtf8Bytes(response);

        await ws.SendAsync(
            responseBytes.AsMemory(),
            WebSocketMessageType.Text,
            endOfMessage: true,
            cancellationToken: ct);
    }
}
```

## 10. MULTI-TENANCY PATTERNS

### What It Actually Is
Multi-tenancy means a single application instance serves multiple tenants, with complete data isolation between them. The two main patterns: **shared database** (all tenants in one DB, isolated by `tenant_id` column — simpler, harder to isolate) and **database per tenant** (separate connection string per tenant — strong isolation, complex routing). ASP.NET Core doesn't have built-in multi-tenancy; you build it via middleware + DI + EF Core query filters.

### The Real-World Code

```csharp
// ── Tenant resolution ─────────────────────────────────────────────────────────

// 1. From subdomain: tenant-a.app.com → TenantId = "tenant-a"
// 2. From JWT claim: "tid" claim → TenantId
// 3. From header: X-Tenant-Id header
// 4. From route: /api/{tenantId}/orders

public interface ITenantContext
{
    string TenantId { get; }
    TenantInfo Info { get; }
}

public class TenantContext : ITenantContext
{
    public string TenantId { get; set; } = null!;
    public TenantInfo Info { get; set; } = null!;
}

public class TenantResolutionMiddleware
{
    private readonly RequestDelegate _next;

    public TenantResolutionMiddleware(RequestDelegate next) => _next = next;

    public async Task InvokeAsync(
        HttpContext context,
        ITenantRepository tenantRepo,
        ITenantContext tenantContext) // Scoped — safe to inject here
    {
        // Strategy: JWT claim → subdomain → header
        string? tenantId =
            context.User.FindFirstValue("tid") ??
            ExtractFromSubdomain(context.Request.Host.Host) ??
            context.Request.Headers["X-Tenant-Id"].FirstOrDefault();

        if (string.IsNullOrEmpty(tenantId))
        {
            context.Response.StatusCode = StatusCodes.Status400BadRequest;
            await context.Response.WriteAsJsonAsync(
                new { error = "Tenant could not be identified" });
            return;
        }

        var tenant = await tenantRepo.GetByIdAsync(tenantId, context.RequestAborted);
        if (tenant is null || !tenant.IsActive)
        {
            context.Response.StatusCode = StatusCodes.Status401Unauthorized;
            await context.Response.WriteAsJsonAsync(
                new { error = "Tenant not found or suspended" });
            return;
        }

        // Populate the scoped tenant context
        ((TenantContext)tenantContext).TenantId = tenantId;
        ((TenantContext)tenantContext).Info = tenant;

        await _next(context);
    }

    private static string? ExtractFromSubdomain(string host)
    {
        var parts = host.Split('.');
        return parts.Length >= 3 ? parts[0] : null; // tenant-a.app.com
    }
}

builder.Services.AddScoped<ITenantContext, TenantContext>();
builder.Services.AddScoped<ITenantRepository, TenantRepository>();


// ── EF Core global query filter for tenant isolation ─────────────────────────
public class AppDbContext : DbContext
{
    private readonly ITenantContext _tenant;

    public AppDbContext(DbContextOptions<AppDbContext> options, ITenantContext tenant)
        : base(options)
    {
        _tenant = tenant;
    }

    public DbSet<Order> Orders => Set<Order>();
    public DbSet<Customer> Customers => Set<Customer>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        // Global filter on all ITenantEntity implementations
        // Every query is automatically scoped to the current tenant
        // HasQueryFilter always applies — even in .Include() chains
        foreach (var entityType in modelBuilder.Model.GetEntityTypes()
            .Where(e => typeof(ITenantEntity).IsAssignableFrom(e.ClrType)))
        {
            var method = typeof(AppDbContext)
                .GetMethod(nameof(SetTenantFilter), BindingFlags.NonPublic | BindingFlags.Instance)!
                .MakeGenericMethod(entityType.ClrType);
            method.Invoke(this, new object[] { modelBuilder });
        }
    }

    private void SetTenantFilter<TEntity>(ModelBuilder modelBuilder)
        where TEntity : class, ITenantEntity
    {
        modelBuilder.Entity<TEntity>()
            .HasQueryFilter(e => e.TenantId == _tenant.TenantId);
    }
}

// CRITICAL: When a singleton needs to query the DB for a different tenant,
// use IgnoreQueryFilters() explicitly — never assume filters don't apply
var allTenantOrders = await db.Orders.IgnoreQueryFilters()
    .Where(o => o.TenantId == specificTenantId)
    .ToListAsync(ct);

    }

    public AppDbContext CreateDbContext()
    {
        var connectionString = _tenant.Info.DatabaseConnectionString;

            .UseSqlServer(connectionString)
            .Options;
        return new AppDbContext(options, _tenant);
    }
}
```

## 11. MEDIATR INTEGRATION (CQRS IN ASP.NET CORE)

### What It Actually Is
MediatR is a library implementing the Mediator pattern — it decouples callers (controllers, endpoints) from handlers (business logic). Instead of injecting `IOrderService` with 20 methods, you send a `CreateOrderCommand` through the mediator and MediatR routes it to `CreateOrderCommandHandler`. This enables CQRS (separate Command and Query models), pipeline behaviors (cross-cutting concerns), and keeps controllers thin. The handler finds its dependencies, not the controller.

### The Real-World Code

```csharp
// ── NuGet: MediatR, MediatR.Extensions.Microsoft.DependencyInjection ──────────

builder.Services.AddMediatR(cfg =>
{
    cfg.RegisterServicesFromAssembly(typeof(Program).Assembly);
    // Pipeline behaviors run in registration order (like middleware)
    cfg.AddBehavior(typeof(IPipelineBehavior<,>), typeof(LoggingBehavior<,>));
    cfg.AddBehavior(typeof(IPipelineBehavior<,>), typeof(ValidationBehavior<,>));
    cfg.AddBehavior(typeof(IPipelineBehavior<,>), typeof(TransactionBehavior<,>));
});


// ── Command + Handler ─────────────────────────────────────────────────────────
public record CreateOrderCommand(
    int CustomerId,
    List<OrderItemRequest> Items,
    AddressDto ShippingAddress) : IRequest<OrderDto>;

public class CreateOrderCommandHandler : IRequestHandler<CreateOrderCommand, OrderDto>
{
    private readonly AppDbContext _db;
    private readonly IMapper _mapper;
    private readonly IPublisher _publisher; // MediatR notifications

    public CreateOrderCommandHandler(AppDbContext db, IMapper mapper, IPublisher publisher)
    {
        _db = db;
        _mapper = mapper;
        _publisher = publisher;
    }

    {

        _db.Orders.Add(order);
        await _db.SaveChangesAsync(ct);

        // Publish domain event via MediatR notification
        await _publisher.Publish(new OrderCreatedNotification(order.Id), ct);

    }

// ── Pipeline behavior: validation ─────────────────────────────────────────────
public class ValidationBehavior<TRequest, TResponse>
    : IPipelineBehavior<TRequest, TResponse>
    where TRequest : IRequest<TResponse>
{
    private readonly IEnumerable<IValidator<TRequest>> _validators;

    public ValidationBehavior(IEnumerable<IValidator<TRequest>> validators)
        => _validators = validators;

    public async Task<TResponse> Handle(
        TRequest request,
        RequestHandlerDelegate<TResponse> next,
        CancellationToken ct)
    {
        if (!_validators.Any())
            return await next();

        var context = new ValidationContext<TRequest>(request);
        var results = await Task.WhenAll(
            _validators.Select(v => v.ValidateAsync(context, ct)));

        var failures = results
            .SelectMany(r => r.Errors)
            .Where(e => e is not null)
            .ToList();

        if (failures.Count > 0)
            throw new FluentValidation.ValidationException(failures);

        return await next();
    }
}


// ── Pipeline behavior: logging with timing ────────────────────────────────────
public class LoggingBehavior<TRequest, TResponse>
    : IPipelineBehavior<TRequest, TResponse>
    where TRequest : IRequest<TResponse>
{
    private readonly ILogger<LoggingBehavior<TRequest, TResponse>> _logger;

    public LoggingBehavior(ILogger<LoggingBehavior<TRequest, TResponse>> logger)
        => _logger = logger;

    public async Task<TResponse> Handle(
        TRequest request,
        RequestHandlerDelegate<TResponse> next,
        CancellationToken ct)
    {
        var requestName = typeof(TRequest).Name;
        _logger.LogInformation("Handling {RequestName}", requestName);

        var sw = Stopwatch.StartNew();
        try
        {
            var response = await next();
            sw.Stop();

            if (sw.ElapsedMilliseconds > 500) // Warn on slow handlers
                _logger.LogWarning(
                    "Slow handler: {RequestName} took {ElapsedMs}ms",
                    requestName, sw.ElapsedMilliseconds);
            else
                _logger.LogInformation(
                    "Handled {RequestName} in {ElapsedMs}ms",
                    requestName, sw.ElapsedMilliseconds);

            return response;
        }
        catch (Exception ex)
        {
            sw.Stop();
            _logger.LogError(ex,
                "Handler failed: {RequestName} after {ElapsedMs}ms",
                requestName, sw.ElapsedMilliseconds);
            throw;
        }
    }
}


// ── Thin controller using MediatR ─────────────────────────────────────────────
[ApiController]
[Route("api/v1/orders")]
public class OrdersController : ControllerBase
{

    // ISender (not IMediator) — only needs Send, not Publish
```

## 12. RESILIENCE WITH POLLY V8

### What It Actually Is
Polly is the .NET resilience library. Version 8 introduced `ResiliencePipeline` — a composable, fluent API replacing the old `.WrapWith()` approach. You compose strategies: Retry, Circuit Breaker, Timeout, Rate Limiter, Hedging, Fallback. The pipeline applies them in declaration order (outer to inner). In .NET 8, `AddStandardResilienceHandler()` gives you a sensible default pipeline with one line.

### How It Works — Mechanically

```
Request → [Timeout] → [Circuit Breaker] → [Retry] → actual HTTP call
                                               ↑
                               Retry is innermost — retries within the breaker's window.
                               Circuit Breaker is outer — counts retried failures too.
                               Timeout is outermost — total time budget for all retries.
```

### The Real-World Code

```csharp
// ── HttpClient with resilience (.NET 8 built-in) ──────────────────────────────
builder.Services.AddHttpClient<IInventoryApiClient, InventoryApiClient>()
    .AddStandardResilienceHandler(options =>
    {
        // Default pipeline: retry + circuit breaker + timeout
        // All configurable:
        options.Retry.MaxRetryAttempts = 3;
        options.Retry.BackoffType = DelayBackoffType.Exponential;
        options.Retry.UseJitter = true;     // Avoids thundering herd

        options.CircuitBreaker.FailureRatio = 0.5; // Open at 50% failure
        options.CircuitBreaker.MinimumThroughput = 20;
        options.CircuitBreaker.SamplingDuration = TimeSpan.FromSeconds(30);
        options.CircuitBreaker.BreakDuration = TimeSpan.FromSeconds(60);

        options.TotalRequestTimeout.Timeout = TimeSpan.FromSeconds(30);
        options.AttemptTimeout.Timeout = TimeSpan.FromSeconds(10);
    });


// ── Custom pipeline for non-HTTP operations ───────────────────────────────────
builder.Services.AddResiliencePipeline("redis-operations", (builder, context) =>
{
    builder
        .AddRetry(new RetryStrategyOptions
        {
            ShouldHandle = new PredicateBuilder()
                .Handle<RedisConnectionException>()
                .Handle<RedisTimeoutException>(),
            MaxRetryAttempts = 3,
            Delay = TimeSpan.FromMilliseconds(100),
            BackoffType = DelayBackoffType.Exponential,
            UseJitter = true,
            OnRetry = args =>
            {
                var logger = context.ServiceProvider
                    .GetRequiredService<ILogger<ResiliencePipeline>>();
                logger.LogWarning("Redis retry attempt {Attempt}", args.AttemptNumber);
                return ValueTask.CompletedTask;
            }
        })
        .AddCircuitBreaker(new CircuitBreakerStrategyOptions
        {
            FailureRatio = 0.5,
            MinimumThroughput = 5,
            SamplingDuration = TimeSpan.FromSeconds(10),
            BreakDuration = TimeSpan.FromSeconds(30),
            OnOpened = args =>
            {
                var logger = context.ServiceProvider
                    .GetRequiredService<ILogger<ResiliencePipeline>>();
                logger.LogError("Redis circuit breaker opened!");
                return ValueTask.CompletedTask;
            }
        })
        .AddTimeout(TimeSpan.FromSeconds(5));
});


// ── Using the pipeline ────────────────────────────────────────────────────────
public class RedisCacheService
{

    private readonly IDatabase _redis;

    public RedisCacheService(
```

## 13. SCRUTOR — DECORATOR & ASSEMBLY SCANNING

### What It Actually Is
Scrutor extends `IServiceCollection` with two capabilities: `Scan()` for bulk-registering services by convention (no need to add each service manually), and `Decorate()` for the decorator pattern (wrap an existing registration with another implementation). Both eliminate the boilerplate of registering dozens of services one-by-one.

### The Real-World Code

```csharp
// ── NuGet: Scrutor ────────────────────────────────────────────────────────────

// ── Assembly scanning ─────────────────────────────────────────────────────────
builder.Services.Scan(scan => scan
    .FromAssemblyOf<Program>()   // Scan the main assembly

    // Register all IQueryHandler<,> implementations as scoped
    .AddClasses(c => c.AssignableTo(typeof(IQueryHandler<,>)))
    .AsImplementedInterfaces()
    .WithScopedLifetime()

    // Register all ICommandHandler<> implementations
    .AddClasses(c => c.AssignableTo(typeof(ICommandHandler<>)))
    .AsImplementedInterfaces()
    .WithScopedLifetime()

    // Register all validators
    .AddClasses(c => c.AssignableTo(typeof(IValidator<>)))
    .AsImplementedInterfaces()
    .WithTransientLifetime()

    // Register all classes named *Repository as their interfaces
    .AddClasses(c => c.Where(t => t.Name.EndsWith("Repository")))
    .AsImplementedInterfaces()
    .WithScopedLifetime());


// ── Decorator pattern ─────────────────────────────────────────────────────────
// Register the concrete implementation
builder.Services.AddScoped<IProductRepository, EfProductRepository>();

// Wrap it: CachingProductRepository(EfProductRepository)
builder.Services.Decorate<IProductRepository, CachingProductRepository>();

// Wrap again: LoggingProductRepository(CachingProductRepository(EfProductRepository))
builder.Services.Decorate<IProductRepository, LoggingProductRepository>();

// Resolution order: LoggingProductRepository → CachingProductRepository → EfProductRepository
// Each decorator receives the next in the chain via constructor injection

    // Invalidate on write-through
    public async Task UpdateAsync(Product product, CancellationToken ct)

        _cache.Remove($"product:{product.Id}");
    }
}
```

## 14. OPENTELEMETRY & DISTRIBUTED TRACING

### What It Actually Is
OpenTelemetry is the vendor-neutral observability framework. It captures three signal types: **Traces** (how a request flows through your services), **Metrics** (counters, histograms, gauges), and **Logs** (structured events). In a distributed system, a single user action can span 10 services — distributed tracing lets you see the full picture. ASP.NET Core's `Activity` class is the .NET implementation of an OpenTelemetry span.

### How It Works — Mechanically

```
HTTP Request arrives
     │
     ASP.NET Core creates Activity (span) for the request
     Activity.TraceId = extracted from "traceparent" header (if present)
                        or generated fresh (if first service in chain)

```

## 15. REQUEST/RESPONSE BUFFERING & BODY READING

### What It Actually Is
By default, the request body is a forward-only, non-seekable stream — you can read it once. If you need to read it multiple times (e.g., logging middleware reads the body, then your action reads it), you must buffer it first. `EnableBuffering()` replaces the body stream with a `FileBufferingReadStream` that writes to memory (under 30 KB) or a temp file (over), and is seekable. **Never enable buffering globally** — it defeats streaming and increases memory pressure for large uploads.

### The Real-World Code

```csharp
// ── Request body logging middleware ──────────────────────────────────────────
public class RequestBodyLoggingMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<RequestBodyLoggingMiddleware> _logger;
    private const int MaxLoggableBodySize = 32 * 1024; // 32 KB

    public RequestBodyLoggingMiddleware(RequestDelegate next,
        ILogger<RequestBodyLoggingMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        // Only log JSON bodies for specific paths
        if (!context.Request.Path.StartsWithSegments("/api") ||
            context.Request.ContentLength > MaxLoggableBodySize)
        {
            await _next(context);
            return;
        }

        // Enable buffering BEFORE reading — makes the stream seekable
        context.Request.EnableBuffering();

        string body = string.Empty;
        if (context.Request.ContentLength > 0)
        {
            using var reader = new StreamReader(
                context.Request.Body,
                Encoding.UTF8,
                leaveOpen: true); // CRITICAL: leaveOpen or body is disposed

            body = await reader.ReadToEndAsync(context.RequestAborted);
            context.Request.Body.Position = 0; // Rewind for the next middleware
        }

        _logger.LogDebug(
            "Request {Method} {Path}: {Body}",
            context.Request.Method,
            context.Request.Path,
            body);

        await _next(context);
    }
}


// ── Reading body directly in Minimal API (without model binding) ──────────────
app.MapPost("/webhooks/raw", async (HttpRequest request, CancellationToken ct) =>
{
    // Read raw body for webhook signature validation
    using var reader = new StreamReader(request.Body);
    var rawBody = await reader.ReadToEndAsync(ct);

    // Verify webhook signature
    var signature = request.Headers["X-Signature-256"].FirstOrDefault();
    if (!WebhookSignatureValidator.IsValid(rawBody, signature))
        return Results.Problem(statusCode: 401, title: "Invalid webhook signature");

    var payload = JsonSerializer.Deserialize<WebhookPayload>(rawBody);
    // Process...
    return Results.Ok();
}).AllowAnonymous();
```

## 16. MEMORY OPTIMIZATION

### What It Actually Is
High-throughput ASP.NET Core services allocate aggressively if you're not careful — every `string.Format`, `new byte[]`, LINQ query materializes objects on the heap, triggering GC pauses. `ArrayPool<T>` gives you a shared pool of arrays to rent and return. `System.IO.Pipelines` eliminates buffer copies. `Span<T>` and `Memory<T>` let you work with slices of buffers without allocation. `RecyclableMemoryStream` replaces `MemoryStream` for large stream operations.

### The Real-World Code

```csharp
// ── ArrayPool: reuse byte arrays for serialization ────────────────────────────
public async Task WriteJsonResponseAsync<T>(HttpResponse response, T value, CancellationToken ct)
{
    // Rent a buffer from the shared pool instead of allocating
    var buffer = ArrayPool<byte>.Shared.Rent(64 * 1024); // Request 64 KB
    try
    {
        int bytesWritten;
        while (true)
        {
            // Try to serialize into the rented buffer
            if (JsonSerializer.TrySerializeToUtf8(buffer, value, out bytesWritten))
                break;

            // Buffer too small — return and rent a larger one
            ArrayPool<byte>.Shared.Return(buffer);
            buffer = ArrayPool<byte>.Shared.Rent(buffer.Length * 2);
        }

        response.ContentType = "application/json";
        response.ContentLength = bytesWritten;
        await response.Body.WriteAsync(buffer.AsMemory(0, bytesWritten), ct);
    }
    finally
    {
        // ALWAYS return to pool — even on exception
        ArrayPool<byte>.Shared.Return(buffer, clearArray: false);
    }
}


// ── System.IO.Pipelines: zero-copy body reading ───────────────────────────────
public static async Task<string> ReadBodyWithPipelineAsync(
    HttpRequest request,
    CancellationToken ct)
{
    var reader = request.BodyReader; // PipeReader — no extra allocation
    StringBuilder result = new();

    while (true)
    {
        var readResult = await reader.ReadAsync(ct);
        var buffer = readResult.Buffer;

        foreach (var segment in buffer)
        {
            // Span<byte> — no allocation, direct memory view
            result.Append(Encoding.UTF8.GetString(segment.Span));
        }

        reader.AdvanceTo(buffer.End); // Tell the pipe we consumed all data

        if (readResult.IsCompleted) break;
    }

    return result.ToString();
}


// ── Span<T> for parsing without allocation ────────────────────────────────────
public static bool TryParseCorrelationId(string header, out Guid correlationId)
{
    // No substring allocation — work with a span slice
    ReadOnlySpan<char> span = header.AsSpan();

    if (span.StartsWith("Bearer ", StringComparison.OrdinalIgnoreCase))
        span = span.Slice("Bearer ".Length);

    return Guid.TryParse(span, out correlationId);
}


// ── RecyclableMemoryStream for large in-memory buffers ────────────────────────
// NuGet: Microsoft.IO.RecyclableMemoryStream
private static readonly RecyclableMemoryStreamManager _streamManager =
    new RecyclableMemoryStreamManager();

public async Task<byte[]> SerializeReportAsync(LargeReport report, CancellationToken ct)
{
    // RecyclableMemoryStream reuses buffers across requests — avoids LOH pressure
    using var stream = _streamManager.GetStream("report-serialization");
    await JsonSerializer.SerializeAsync(stream, report, cancellationToken: ct);
    return stream.ToArray();
}
```

## 17. ISTARTUPFILTER & IHOSTINGSTARTUP

### What It Actually Is
`IStartupFilter` intercepts the `Configure(IApplicationBuilder)` pipeline and lets libraries inject middleware automatically — without requiring the consumer to call anything in their `Configure` method. `IHostingStartup` (via `[assembly: HostingStartup]` attribute) automatically runs registration code when the host starts, without any reference in the application's code. These are library/framework extension mechanisms — your application code rarely uses them, but understanding them demystifies how packages like HealthChecks UI auto-wire themselves.

### The Real-World Code

```csharp
// ── IStartupFilter: auto-inject middleware ────────────────────────────────────
// Library code that ensures its middleware is always added:
public class SecurityHeadersStartupFilter : IStartupFilter
{
    public Action<IApplicationBuilder> Configure(Action<IApplicationBuilder> next)
    {
        return app =>
        {
            // Our middleware runs FIRST — before any app middleware
            app.Use(async (ctx, nextMiddleware) =>
            {
                ctx.Response.Headers.Append("X-Content-Type-Options", "nosniff");
                ctx.Response.Headers.Append("X-Frame-Options", "DENY");
                ctx.Response.Headers.Append("X-XSS-Protection", "1; mode=block");
                ctx.Response.Headers.Append("Referrer-Policy", "strict-origin-when-cross-origin");
                await nextMiddleware(ctx);
            });

            // Then the rest of the app's middleware runs
            next(app);
        };
    }
}

// Registration — any library can do this without the app knowing:
builder.Services.AddTransient<IStartupFilter, SecurityHeadersStartupFilter>();


// ── IHostingStartup: automatic host setup ─────────────────────────────────────
// In a separate library project:
[assembly: HostingStartup(typeof(MyLibrary.HostingStartup))]

namespace MyLibrary
{
    public class HostingStartup : IHostingStartup
    {
        public void Configure(IWebHostBuilder builder)
        {
            // Auto-register services when this assembly is referenced
            builder.ConfigureServices((context, services) =>
            {
                services.AddTransient<IStartupFilter, SecurityHeadersStartupFilter>();
                services.AddHealthChecks()
                    .AddCheck<MyLibraryHealthCheck>("my-library");
            });
        }
    }
}

// The consuming application doesn't need to call ANYTHING.
// Just referencing the NuGet package activates the hosting startup.
// (Can be disabled via ASPNETCORE_PREVENTHOSTINGSTARTUP=true env var)
```

## 18. THE REQUEST LIFECYCLE — COMPLETE END-TO-END

### What It Actually Is
A complete trace of every step from TCP bytes arriving at the NIC to JSON bytes leaving the socket. This is the mental model that makes debugging and performance optimization systematic — knowing exactly where time is spent and which layer owns each responsibility.

### The Complete Flow

```
[NETWORK LAYER]
 1. TCP SYN/SYN-ACK/ACK (TCP handshake) — or QUIC for HTTP/3
 2. TLS ClientHello → ServerHello → Certificate → KeyExchange (for HTTPS)

[KESTREL LAYER]
 3. Accept connection → create ConnectionContext
 4. HTTP parsing: read request line (GET /api/orders HTTP/1.1)
 5. Header parsing: Host, Content-Type, Authorization, etc.
 6. Request body available as PipeReader (not read yet)
 7. Populate IFeatureCollection with IHttpRequestFeature, IHttpConnectionFeature
 8. Wrap features in DefaultHttpContext
 9. Call first RequestDelegate in the pipeline

[MIDDLEWARE LAYER — IN ORDER]
10. ExceptionHandler middleware: wrap in try/catch
11. HSTS middleware: add Strict-Transport-Security header (response phase)
12. HttpsRedirection: check scheme, 301 if HTTP
13. StaticFiles: check if path maps to a file → serve and short-circuit if yes
14. UseRouting: run route matching engine against all registered endpoints
    - Evaluate route templates, constraints
    - Set IEndpointFeature.Endpoint on HttpContext
15. UseCors: read endpoint's CORS metadata, validate Origin header
    - Preflight: respond with CORS headers and 204
    - Simple: set Access-Control-Allow-Origin on response
16. UseAuthentication:
    - Read Authorization header (JWT) or Cookie
    - Call JwtBearerHandler.AuthenticateAsync
    - Validate token signature, expiry, issuer, audience
    - Build ClaimsPrincipal → set HttpContext.User
17. UseAuthorization:
    - Read endpoint's [Authorize] metadata
    - Evaluate policy against HttpContext.User
    - 401 if unauthenticated, 403 if insufficient permissions
18. UseOutputCache: check cache for this request key → serve cached response if hit
19. UseRateLimiter: evaluate partition limits → 429 if exceeded

[ENDPOINT LAYER — CONTROLLER]
20. ControllerActivator: resolve controller from DI (scoped)
21. Model binding:
    - Bind [FromRoute] params from RouteValueDictionary
    - Bind [FromQuery] params from QueryString
    - Read and deserialize [FromBody] from request body stream (JSON deserialization)
22. [ApiController] model state validation:
    - DataAnnotations evaluated against bound model
    - If invalid → 400 ProblemDetails returned, action never runs
23. Action filter pipeline (OnActionExecuting):
    - Auth filters → Resource filters → Action filters
24. Action method runs:
    - Your code executes
    - DI services resolved from HttpContext.RequestServices (scoped container)
    - Async: await downstream I/O (DB, HTTP, Redis)
25. Action filter pipeline (OnActionExecuted):
    - Filters can modify the IActionResult
26. Result execution:
    - IActionResult.ExecuteResultAsync called
    - For ObjectResult: content negotiation (JSON formatter selected)
    - JsonSerializer serializes the object to UTF-8 bytes
    - Response headers set (Content-Type, Content-Length)
    - Bytes written to response body pipe

[MIDDLEWARE UNWIND — REVERSE ORDER]
27. CorrelationId middleware: append X-Correlation-Id header (OnStarting callback fires)
28. ResponseCompression: compress body bytes if Accept-Encoding matches
29. RequestTiming middleware: log elapsed time after downstream completes

[KESTREL OUTPUT]
30. HTTP response writer: serialize status line + headers as ASCII
31. Flush output pipe to socket send buffer
32. OS sends bytes over TCP (TLS encrypts them first)

[CLIENT]
33. Browser receives bytes
34. TLS decrypts
35. HTTP parser reconstructs response
36. JSON.parse() on body
37. React state update / DOM render

Total time budget (aspirational):
 - Steps 1-9  (connection):    1-50ms (TCP + TLS — mostly network latency)
 - Steps 10-19 (middleware):   <1ms
 - Steps 20-22 (binding):      <1ms
 - Steps 24   (your code + DB): 5-200ms (dominant cost — optimize here)
 - Steps 26-30 (serialization + output): <2ms
```
