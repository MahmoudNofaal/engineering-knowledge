## 1. ASPNET CORE IDENTITY — COMPLETE REFERENCE

### What It Actually Is
ASP.NET Core Identity is a membership system: it manages users, passwords, roles, claims, tokens, and external logins. It stores everything in a database via EF Core (`UserManager`, `RoleManager`) and provides the middleware pipeline for signin/signout. It is *not* an authentication protocol — it's a user store that sits beneath authentication schemes. You can use Identity with cookie auth, JWT, or both.

### How It Works — Mechanically

```
UserManager<TUser>     ← CRUD for users: create, find, update password, lock out
RoleManager<TRole>     ← CRUD for roles: create, assign, check
SignInManager<TUser>   ← orchestrates authentication: password check, 2FA, cookie signin
IPasswordHasher<TUser> ← PBKDF2 by default (100,000 iterations in .NET 8)
IUserStore<TUser>      ← EF Core implementation: reads/writes AspNetUsers table
IUserClaimsPrincipalFactory ← builds ClaimsPrincipal from User entity
```

### The Real-World Code

```csharp
// ── Registration ──────────────────────────────────────────────────────────────
builder.Services.AddIdentity<ApplicationUser, ApplicationRole>(options =>
{
    // Password policy
    options.Password.RequireDigit = true;
    options.Password.RequiredLength = 12;
    options.Password.RequireUppercase = true;
    options.Password.RequireLowercase = true;
    options.Password.RequireNonAlphanumeric = true;
    options.Password.RequiredUniqueChars = 4;

    // Lockout policy — critical for brute force prevention
    options.Lockout.DefaultLockoutTimeSpan = TimeSpan.FromMinutes(15);
    options.Lockout.MaxFailedAccessAttempts = 5;
    options.Lockout.AllowedForNewUsers = true;

    // User settings
    options.User.RequireUniqueEmail = true;
    options.SignIn.RequireConfirmedEmail = true;
    options.SignIn.RequireConfirmedAccount = true;

.AddEntityFrameworkStores<AppDbContext>()
.AddDefaultTokenProviders()             // Email/phone confirmation, password reset

// ── Auth service: registration + login ───────────────────────────────────────
public class AuthService : IAuthService
{
    private readonly UserManager<ApplicationUser> _userManager;
    private readonly SignInManager<ApplicationUser> _signInManager;
    private readonly IJwtTokenService _jwtService;
    private readonly ILogger<AuthService> _logger;

    public AuthService(
        UserManager<ApplicationUser> userManager,
        SignInManager<ApplicationUser> signInManager,
        IJwtTokenService jwtService,
        ILogger<AuthService> logger)
    {
        _userManager = userManager;
        _signInManager = signInManager;
        _jwtService = jwtService;
        _logger = logger;
    }

    public async Task<AuthResult> RegisterAsync(RegisterRequest request, CancellationToken ct)
    {
        // Check if email is taken
        var existing = await _userManager.FindByEmailAsync(request.Email);
        if (existing is not null)
            return AuthResult.Failure("Email is already registered");

        var user = new ApplicationUser
        {
            UserName = request.Email,
            Email = request.Email,
            FirstName = request.FirstName,
            LastName = request.LastName,
            CreatedAt = DateTime.UtcNow
        };

        // CreateAsync: hashes password, runs validators, saves to DB
        var result = await _userManager.CreateAsync(user, request.Password);
        if (!result.Succeeded)
            return AuthResult.Failure(result.Errors.Select(e => e.Description));

        await _userManager.AddToRoleAsync(user, "User"); // Default role

        // Generate email confirmation token
        var token = await _userManager.GenerateEmailConfirmationTokenAsync(user);
        // Send confirmation email (fire and forget via background service)

        _logger.LogInformation("User {Email} registered successfully", request.Email);
        return AuthResult.Success(user.Id);
    }

    public async Task<TokenPair?> LoginAsync(LoginRequest request, CancellationToken ct)
    {
        var user = await _userManager.FindByEmailAsync(request.Email);
        if (user is null || !user.IsActive)
        {
            // Never reveal whether email exists — timing attack mitigation
            await Task.Delay(Random.Shared.Next(50, 200), ct); // Constant-time delay
            return null;
        }

        // CheckPasswordSignInAsync: validates password + lockout + 2FA
        var result = await _signInManager.CheckPasswordSignInAsync(
            user, request.Password, lockoutOnFailure: true);

        if (result.IsLockedOut)
        {
            _logger.LogWarning("User {Email} is locked out", request.Email);
            throw new AccountLockedException(
                $"Account locked for {_userManager.Options.Lockout.DefaultLockoutTimeSpan.Minutes} minutes");
        }

        if (result.RequiresTwoFactor)
            throw new TwoFactorRequiredException(user.Id);

        if (!result.Succeeded)
        {
            _logger.LogWarning("Failed login attempt for {Email}", request.Email);
            return null;
        }

        // Update last login
        user.LastLoginAt = DateTime.UtcNow;
        await _userManager.UpdateAsync(user);

        var roles = await _userManager.GetRolesAsync(user);
        var claims = await _userManager.GetClaimsAsync(user);

        return _jwtService.GenerateTokenPair(user, roles, claims);
    }

    public async Task<bool> ConfirmEmailAsync(int userId, string token, CancellationToken ct)
    {
        var user = await _userManager.FindByIdAsync(userId.ToString());
        if (user is null) return false;

        var result = await _userManager.ConfirmEmailAsync(user, token);
        return result.Succeeded;
    }

    public async Task<bool> ResetPasswordAsync(
        string email, string token, string newPassword, CancellationToken ct)
    {
        var user = await _userManager.FindByEmailAsync(email);
        if (user is null) return true; // Don't reveal if user exists

        var result = await _userManager.ResetPasswordAsync(user, token, newPassword);
        if (!result.Succeeded)
        {
            _logger.LogWarning("Password reset failed for {Email}: {Errors}",
                email, string.Join(", ", result.Errors.Select(e => e.Description)));
        }
        return result.Succeeded;
    }
}


// ── Custom password validator: reject common passwords ────────────────────────
public class CommonPasswordValidator : IPasswordValidator<ApplicationUser>
{
    private static readonly HashSet<string> CommonPasswords = new(StringComparer.OrdinalIgnoreCase)
    {
        "Password1!", "Password123!", "Qwerty123!", "Admin1234!", "Welcome1!"
    };

    public Task<IdentityResult> ValidateAsync(
        UserManager<ApplicationUser> manager,
        ApplicationUser user,
        string? password)
    {
        if (password is null) return Task.FromResult(IdentityResult.Success);

        if (CommonPasswords.Contains(password))
            return Task.FromResult(IdentityResult.Failed(new IdentityError
            {
                Code = "CommonPassword",
                Description = "This password is too commonly used. Please choose a more unique password."
            }));

        // Don't allow password to contain the email username
        var emailUser = user.Email?.Split('@').FirstOrDefault();
        if (emailUser is not null &&
            password.Contains(emailUser, StringComparison.OrdinalIgnoreCase))
            return Task.FromResult(IdentityResult.Failed(new IdentityError
            {
                Code = "PasswordContainsEmail",
                Description = "Password cannot contain parts of your email address."
            }));

        return Task.FromResult(IdentityResult.Success);
    }
}
```

## 2. COMPLETE JWT + REFRESH TOKEN SYSTEM

### What It Actually Is
A complete, production-grade JWT system consists of: short-lived access tokens (15–60 minutes), long-lived refresh tokens (7–30 days) stored in the database, refresh token rotation (each use issues a new refresh token and revokes the old one), and a token family concept to detect refresh token theft (if a revoked token is used, revoke the entire family).

### How It Works — Mechanically

```
[Login]
  → Validate credentials
  → Generate: AccessToken (JWT, 15 min) + RefreshToken (random, 30 days)
  → Store RefreshToken in DB: {Token, UserId, FamilyId, ExpiresAt, IsRevoked}
  → Return both to client

[API Call]
  → Client sends: Authorization: Bearer <AccessToken>
  → JwtBearerHandler validates signature + expiry

[Token Refresh]
  → Client sends: {refreshToken}
  → Look up in DB → validate not revoked, not expired
  → Issue NEW AccessToken + NEW RefreshToken
  → Revoke OLD RefreshToken
  → Return new pair

[Theft Detection]
  → If a revoked RefreshToken is used: revoke ENTIRE family
  → All sessions for this user/family are invalidated
  → User must re-login
```

### The Real-World Code

```csharp
// ── RefreshToken entity ───────────────────────────────────────────────────────
public class RefreshToken
{
    public int Id { get; private set; }
    public string Token { get; private set; } = null!;      // Opaque random string
    public string FamilyId { get; private set; } = null!;   // All tokens from same login
    public int UserId { get; private set; }
    public DateTime CreatedAt { get; private set; }
    public DateTime ExpiresAt { get; private set; }
    public bool IsRevoked { get; private set; }
    public string? ReplacedByToken { get; private set; }    // Audit trail
    public string? CreatedByIp { get; private set; }
    public string? RevokedByIp { get; private set; }
    public string? RevokedReason { get; private set; }

    public bool IsExpired => DateTime.UtcNow >= ExpiresAt;
    public bool IsActive => !IsRevoked && !IsExpired;

    public static RefreshToken Create(int userId, string ip, string? familyId = null)
    {
        return new RefreshToken
        {
            Token = Convert.ToBase64String(RandomNumberGenerator.GetBytes(64)),
            FamilyId = familyId ?? Guid.NewGuid().ToString("N"),
            UserId = userId,
            CreatedAt = DateTime.UtcNow,
            ExpiresAt = DateTime.UtcNow.AddDays(30),
            CreatedByIp = ip
        };
    }

    public RefreshToken Rotate(string newToken, string ip)
    {
        ReplacedByToken = newToken;
        return new RefreshToken
        {
            Token = newToken,
            FamilyId = FamilyId, // Same family — theft detection works across rotations
            UserId = UserId,
            CreatedAt = DateTime.UtcNow,
            ExpiresAt = DateTime.UtcNow.AddDays(30),
            CreatedByIp = ip
        };
    }

    public void Revoke(string ip, string reason = "Replaced by rotation")
    {
        IsRevoked = true;
        RevokedByIp = ip;
        RevokedReason = reason;
    }
}


// ── Token service ─────────────────────────────────────────────────────────────
public class TokenService : ITokenService
{
    private readonly JwtOptions _jwt;
    private readonly AppDbContext _db;
    private readonly ILogger<TokenService> _logger;

    public TokenService(IOptions<JwtOptions> jwt, AppDbContext db, ILogger<TokenService> logger)
    {
        _jwt = jwt.Value;
        _db = db;
        _logger = logger;
    }

    public async Task<TokenPair> CreateTokenPairAsync(
        ApplicationUser user,
        IList<string> roles,
        string clientIp,
        CancellationToken ct)
    {
        var accessToken = CreateAccessToken(user, roles);
        var refreshToken = RefreshToken.Create(user.Id, clientIp);

        _db.RefreshTokens.Add(refreshToken);
        await _db.SaveChangesAsync(ct);

        return new TokenPair(accessToken, refreshToken.Token,
            DateTime.UtcNow.AddMinutes(_jwt.AccessTokenExpiryMinutes),
            refreshToken.ExpiresAt);
    }

    public async Task<TokenPair?> RefreshAsync(
        string token,
        string clientIp,
        CancellationToken ct)
    {
        var stored = await _db.RefreshTokens
            .Include(t => t.User)
            .FirstOrDefaultAsync(t => t.Token == token, ct);

        if (stored is null)
        {
            _logger.LogWarning("Refresh attempt with unknown token from {Ip}", clientIp);
            return null;
        }

        // THEFT DETECTION: if a revoked token from this family is used,
        // an attacker may have stolen the token. Revoke the entire family.
        if (stored.IsRevoked)
        {
            _logger.LogCritical(
                "Revoked refresh token reuse detected for user {UserId} family {Family}. " +
                "Revoking entire family. Possible theft.",
                stored.UserId, stored.FamilyId);

            var family = await _db.RefreshTokens
                .Where(t => t.FamilyId == stored.FamilyId && !t.IsRevoked)
                .ToListAsync(ct);

            foreach (var familyToken in family)
                familyToken.Revoke(clientIp, "Family revoked due to reuse of revoked token");

            await _db.SaveChangesAsync(ct);
            return null;
        }

        if (stored.IsExpired)
        {
            _logger.LogInformation("Expired refresh token used by user {UserId}", stored.UserId);
            return null;
        }

        // Rotate: revoke old, issue new
        var newTokenValue = Convert.ToBase64String(RandomNumberGenerator.GetBytes(64));
        var newRefreshToken = stored.Rotate(newTokenValue, clientIp);
        stored.Revoke(clientIp);

        _db.RefreshTokens.Add(newRefreshToken);
        await _db.SaveChangesAsync(ct);

        var roles = await _db.UserRoles
            .Where(ur => ur.UserId == stored.UserId)
            .Select(ur => ur.Role.Name!)
            .ToListAsync(ct);

        var accessToken = CreateAccessToken(stored.User, roles);

        return new TokenPair(accessToken, newRefreshToken.Token,
            DateTime.UtcNow.AddMinutes(_jwt.AccessTokenExpiryMinutes),
            newRefreshToken.ExpiresAt);
    }

    public async Task RevokeAsync(string token, string clientIp, CancellationToken ct)
    {
        var stored = await _db.RefreshTokens
            .FirstOrDefaultAsync(t => t.Token == token, ct);

        if (stored is null || !stored.IsActive) return;

        stored.Revoke(clientIp, "User logout");
        await _db.SaveChangesAsync(ct);
    }

    private string CreateAccessToken(ApplicationUser user, IList<string> roles)
    {
        var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(_jwt.Secret));
        var credentials = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);

        var claims = new List<Claim>
        {
            new(JwtRegisteredClaimNames.Sub, user.Id.ToString()),
            new(JwtRegisteredClaimNames.Email, user.Email!),
            new(JwtRegisteredClaimNames.Jti, Guid.NewGuid().ToString("N")),
            new(JwtRegisteredClaimNames.Iat,
                DateTimeOffset.UtcNow.ToUnixTimeSeconds().ToString(),
                ClaimValueTypes.Integer64),
            ..roles.Select(r => new Claim(ClaimTypes.Role, r))
        };

        if (user.TenantId is not null)
            claims.Add(new Claim("tid", user.TenantId));

        var token = new JwtSecurityToken(
            issuer: _jwt.Issuer,
            audience: _jwt.Audience,
            claims: claims,
            notBefore: DateTime.UtcNow,
            expires: DateTime.UtcNow.AddMinutes(_jwt.AccessTokenExpiryMinutes),
            signingCredentials: credentials);

        return new JwtSecurityTokenHandler().WriteToken(token);
    }
}


// ── Auth endpoints ────────────────────────────────────────────────────────────
public static class AuthEndpoints
{
    public static void MapAuthEndpoints(this WebApplication app)
    {
        var group = app.MapGroup("/api/auth")
            .WithTags("Auth")
            .AllowAnonymous();

        group.MapPost("/register", Register);
        group.MapPost("/login", Login);
        group.MapPost("/refresh", Refresh);
        group.MapPost("/logout", Logout).RequireAuthorization();
        group.MapPost("/confirm-email", ConfirmEmail);
        group.MapPost("/forgot-password", ForgotPassword);
        group.MapPost("/reset-password", ResetPassword);
    }

    static async Task<Results<Ok<TokenPair>, UnauthorizedHttpResult, ProblemHttpResult>> Login(
        LoginRequest request,
        IAuthService auth,
        HttpContext ctx,
        CancellationToken ct)
    {
        try
        {
            var ip = ctx.Connection.RemoteIpAddress?.ToString() ?? "unknown";
            var tokens = await auth.LoginAsync(request, ip, ct);
            return tokens is null
                ? TypedResults.Unauthorized()
                : TypedResults.Ok(tokens);
        }
        catch (AccountLockedException ex)
        {
            return TypedResults.Problem(ex.Message, statusCode: 429, title: "Account Locked");
        }
    }

    static async Task<Results<Ok<TokenPair>, UnauthorizedHttpResult>> Refresh(
        RefreshTokenRequest request,
        ITokenService tokenService,
        HttpContext ctx,
        CancellationToken ct)
    {
        var ip = ctx.Connection.RemoteIpAddress?.ToString() ?? "unknown";
        var tokens = await tokenService.RefreshAsync(request.RefreshToken, ip, ct);
        return tokens is null
            ? TypedResults.Unauthorized()
            : TypedResults.Ok(tokens);
    }

    static async Task<NoContent> Logout(
        LogoutRequest request,
        ITokenService tokenService,
        HttpContext ctx,
        CancellationToken ct)
    {
        var ip = ctx.Connection.RemoteIpAddress?.ToString() ?? "unknown";
        await tokenService.RevokeAsync(request.RefreshToken, ip, ct);
        return TypedResults.NoContent();
    }
}
```

## 3. OAUTH2 / OPENID CONNECT (EXTERNAL PROVIDERS)

### What It Actually Is
OAuth2 is the authorization protocol; OpenID Connect (OIDC) adds an identity layer on top. When a user clicks "Sign in with Google", your app redirects them to Google (the Identity Provider), Google authenticates them and sends back an `id_token` (JWT) containing claims, and your app exchanges this for a local session. `Microsoft.AspNetCore.Authentication.Google/Microsoft` handles the protocol dance; `SignInManager` links the external identity to your local user.

### The Real-World Code

```csharp
// ── Registration ──────────────────────────────────────────────────────────────
builder.Services.AddAuthentication(options =>
{
    options.DefaultScheme = CookieAuthenticationDefaults.AuthenticationScheme;
    options.DefaultChallengeScheme = GoogleDefaults.AuthenticationScheme;
})
.AddCookie(options =>
{
    options.Cookie.Name = "myapp.session";
    options.Cookie.HttpOnly = true;
    options.Cookie.SameSite = SameSiteMode.Lax; // Lax (not Strict) for OIDC redirect
    options.ExpireTimeSpan = TimeSpan.FromHours(8);
    options.SlidingExpiration = true;
})
.AddGoogle(options =>
{
    options.ClientId = builder.Configuration["Auth:Google:ClientId"]!;
    options.ClientSecret = builder.Configuration["Auth:Google:ClientSecret"]!;
    options.Scope.Add("profile");
    options.Scope.Add("email");
    // Map Google claims to standard .NET claims
    options.ClaimActions.MapJsonKey(ClaimTypes.Name, "name");
    options.ClaimActions.MapJsonKey("picture", "picture");
    options.SaveTokens = true; // Saves access_token for calling Google APIs
})
.AddMicrosoftAccount(options =>
{
    options.ClientId = builder.Configuration["Auth:Microsoft:ClientId"]!;
    options.ClientSecret = builder.Configuration["Auth:Microsoft:ClientSecret"]!;
});


// ── External login callback ───────────────────────────────────────────────────
[HttpGet("signin-google")]  // Callback URL registered in Google Console
public async Task<IActionResult> ExternalLoginCallback(
    string? returnUrl = null,
    string? remoteError = null)
{
    if (remoteError is not null)
        return BadRequest($"External login failed: {remoteError}");

    // Get the external login info from the authentication cookie set by the OIDC dance
    var info = await _signInManager.GetExternalLoginInfoAsync();
    if (info is null)
        return Redirect("/login?error=external_login_failed");

    // Try to sign in with existing external login record
    var result = await _signInManager.ExternalLoginSignInAsync(
        info.LoginProvider, info.ProviderKey, isPersistent: false);

    if (result.Succeeded)
        return LocalRedirect(returnUrl ?? "/");

    // No existing link — create a new user or link to existing account
    var email = info.Principal.FindFirstValue(ClaimTypes.Email)!;
    var existingUser = await _userManager.FindByEmailAsync(email);

    if (existingUser is not null)
    {
        // Link external login to existing account
        await _userManager.AddLoginAsync(existingUser, info);
        await _signInManager.SignInAsync(existingUser, isPersistent: false);
        return LocalRedirect(returnUrl ?? "/");
    }

    // Create brand new user from external claims
    var newUser = new ApplicationUser
    {
        UserName = email,
        Email = email,
        FirstName = info.Principal.FindFirstValue(ClaimTypes.GivenName) ?? "",
        LastName = info.Principal.FindFirstValue(ClaimTypes.Surname) ?? "",
        EmailConfirmed = true // Email confirmed via external provider
    };

    var createResult = await _userManager.CreateAsync(newUser);
    if (!createResult.Succeeded)
        return BadRequest(createResult.Errors.Select(e => e.Description));

    await _userManager.AddLoginAsync(newUser, info);
    await _userManager.AddToRoleAsync(newUser, "User");
    await _signInManager.SignInAsync(newUser, isPersistent: false);

    return LocalRedirect(returnUrl ?? "/");
}
```

## 4. THE COMPLETE IRESULT HIERARCHY

### What It Actually Is
`IResult` is the Minimal API response abstraction. `TypedResults` (static class) returns typed variants that enable OpenAPI schema inference at compile time. `Results` (static class) returns untyped variants. Always prefer `TypedResults` — it makes Swagger auto-generate correct response schemas without `[ProducesResponseType]` attributes.

### The Complete Reference

```csharp
// ══════════════════════════════════════════════════════════════════════════════
// TYPED RESULTS — preferred for Minimal APIs (OpenAPI schema inferred automatically)
// Return type hint: Results<Ok<T>, NotFound> tells Swagger exactly what schemas to expect
// ══════════════════════════════════════════════════════════════════════════════

// 2xx Success
TypedResults.Ok(value)                    // 200 Ok<T>
TypedResults.Ok()                         // 200 (no body)
TypedResults.Created(uri, value)          // 201 Created<T>
TypedResults.CreatedAtRoute(name, routeValues, value)  // 201 with Location header
TypedResults.Accepted(uri, value)         // 202 Accepted<T>
TypedResults.AcceptedAtRoute(name, routeValues, value) // 202
TypedResults.NoContent()                  // 204

// 3xx Redirects
TypedResults.Redirect(url)                // 302 (temporary)
TypedResults.RedirectToRoute(name)        // 302 by route name
TypedResults.LocalRedirect(path)          // 302, must be local URL (prevents open redirect)

// 4xx Client Errors
TypedResults.BadRequest(error)            // 400 BadRequest<T>
TypedResults.BadRequest()                 // 400 (no body)
TypedResults.Unauthorized()              // 401
TypedResults.Forbid()                    // 403
TypedResults.NotFound(value)             // 404 NotFound<T>
TypedResults.NotFound()                  // 404 (no body)
TypedResults.Conflict(value)             // 409
TypedResults.UnprocessableEntity(value)  // 422
TypedResults.ValidationProblem(errors)   // 400 HttpValidationProblemDetails

// Problem Details (RFC 7807)
TypedResults.Problem(
    detail: "...",
    statusCode: 500,
    title: "Error",
    type: "https://...",
    instance: "/api/orders/42",
    extensions: new Dictionary<string, object?> { ["traceId"] = "abc" })

// Files
TypedResults.File(fileContents, contentType, fileDownloadName) // Binary file
TypedResults.File(stream, contentType, fileDownloadName)       // Stream
TypedResults.PhysicalFile(physicalPath, contentType)           // Disk file
TypedResults.VirtualFile(path, contentType)                    // wwwroot file

// Content
TypedResults.Content(content, contentType, encoding, statusCode)
TypedResults.Json(data, options, contentType, statusCode)
TypedResults.Text(text, contentType, encoding, statusCode)
TypedResults.Bytes(bytes, contentType, fileDownloadName)
TypedResults.Stream(stream, contentType)

// Special
TypedResults.Empty                        // 200, no body
TypedResults.StatusCode(code)            // Arbitrary status code


// ── Using union return types for full Swagger schema inference ────────────────
app.MapGet("/api/orders/{id:int}",
    async (int id, ISender mediator, CancellationToken ct)
        // Return type declared → Swagger shows 200 and 404 schemas
        // No [ProducesResponseType] attributes needed
        : Task<Results<Ok<OrderDto>, NotFound>>
    =>
{
    var order = await mediator.Send(new GetOrderByIdQuery(id), ct);
    return order is null
        ? TypedResults.NotFound()
        : TypedResults.Ok(order);
});

// Three possible responses → union of three
```

## 5. YARP — REVERSE PROXY IN ASPNET CORE

### What It Actually Is
YARP (Yet Another Reverse Proxy) is Microsoft's high-performance, extensible reverse proxy library built on ASP.NET Core. Instead of putting Nginx or Traefik in front of your services, YARP lets you write your proxy in C# — giving you the full power of .NET middleware, DI, and auth in your routing/forwarding logic. Use cases: API gateway, BFF (Backend for Frontend), gradual migration from monolith to microservices.

### How It Works — Mechanically

```
Client Request
      │
      ▼
YARP Middleware
  ├── Route matching (matches request to a "route" config entry)
  ├── Cluster selection (picks a "cluster" = group of backend servers)
  ├── Load balancing (round-robin, least requests, random, power of two)
  ├── Request transformation (add/remove headers, rewrite path)
  ├── Forward request to selected backend
  ├── Response transformation (modify response headers)
  └── Return response to client
```

### The Real-World Code

## 6. OUTBOX PATTERN — COMPLETE IMPLEMENTATION

### What It Actually Is
The Outbox pattern solves the dual-write problem: when you need to both persist a database change AND publish an event to a message bus, both operations must succeed atomically. You can't have a transaction that spans a DB and a message broker. The solution: write the event to an `OutboxMessages` table *within the same DB transaction* as your domain change, then have a background service read and publish these events separately.

### How It Works — Mechanically

```
HTTP Request → Handler
                │
                DB Transaction {
                  INSERT INTO Orders (...)
                  INSERT INTO OutboxMessages (event_type, payload, created_at)
                } ← Single atomic commit. Both or neither.
                │
                Response returned to client

[Background Service — every 5 seconds]
  SELECT TOP 50 FROM OutboxMessages WHERE ProcessedAt IS NULL ORDER BY CreatedAt
  FOR EACH message:
    Publish to message bus (RabbitMQ / ServiceBus / Kafka)
    UPDATE OutboxMessages SET ProcessedAt = NOW() WHERE Id = @id
```

### The Real-World Code

```csharp
// ── OutboxMessage entity ──────────────────────────────────────────────────────
public class OutboxMessage
{
    public Guid Id { get; private set; } = Guid.NewGuid();
    public string EventType { get; private set; } = null!;
    public string Payload { get; private set; } = null!;  // JSON-serialized event
    public DateTime CreatedAt { get; private set; } = DateTime.UtcNow;
    public DateTime? ProcessedAt { get; set; }
    public int RetryCount { get; set; }
    public string? LastError { get; set; }

    public static OutboxMessage From<TEvent>(TEvent @event)
        where TEvent : IDomainEvent
    {
        return new OutboxMessage
        {
            EventType = @event.GetType().AssemblyQualifiedName!,
            Payload = JsonSerializer.Serialize(@event, @event.GetType())
        };
    }
}

// ── Domain event publishing inside a command handler ─────────────────────────
public class CreateOrderCommandHandler : IRequestHandler<CreateOrderCommand, OrderDto>
{
    private readonly AppDbContext _db;

    public CreateOrderCommandHandler(AppDbContext db) => _db = db;

    public async Task<OrderDto> Handle(CreateOrderCommand cmd, CancellationToken ct)
    {
        var order = Order.Create(cmd.CustomerId, cmd.Items);

        _db.Orders.Add(order);

        // Write event to outbox IN THE SAME TRANSACTION as the order
        _db.OutboxMessages.Add(OutboxMessage.From(new OrderCreatedEvent(
            OrderId: order.Id,
            CustomerId: order.CustomerId,
            Total: order.Total,
            CreatedAt: order.CreatedAt)));

        // Single SaveChangesAsync = single transaction = atomic
        await _db.SaveChangesAsync(ct);

        return new OrderDto(order);
    }
}


// ── Outbox processor background service ──────────────────────────────────────
public sealed class OutboxProcessorService : BackgroundService
{
    private readonly IServiceScopeFactory _scopeFactory;
    private readonly ILogger<OutboxProcessorService> _logger;

    public OutboxProcessorService(
        IServiceScopeFactory scopeFactory,
        ILogger<OutboxProcessorService> logger)
    {
        _scopeFactory = scopeFactory;
        _logger = logger;
    }

    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        using var timer = new PeriodicTimer(TimeSpan.FromSeconds(5));

        while (await timer.WaitForNextTickAsync(stoppingToken))
        {
            await ProcessBatchAsync(stoppingToken);
        }
    }

    private async Task ProcessBatchAsync(CancellationToken ct)
    {
        await using var scope = _scopeFactory.CreateAsyncScope();
        var db = scope.ServiceProvider.GetRequiredService<AppDbContext>();
        var bus = scope.ServiceProvider.GetRequiredService<IMessageBus>();

        // Only fetch unprocessed, with max retries not exceeded
        var messages = await db.OutboxMessages
            .Where(m => m.ProcessedAt == null && m.RetryCount < 5)
            .OrderBy(m => m.CreatedAt)
            .Take(50)
            .ToListAsync(ct);

        if (messages.Count == 0) return;

        _logger.LogDebug("Processing {Count} outbox messages", messages.Count);

        foreach (var message in messages)
        {
            try
            {
                // Deserialize back to the correct event type
                var eventType = Type.GetType(message.EventType)!;
                var @event = (IDomainEvent)JsonSerializer.Deserialize(
                    message.Payload, eventType)!;

                await bus.PublishAsync(@event, ct);
                message.ProcessedAt = DateTime.UtcNow;
            }
            catch (Exception ex)
            {
                message.RetryCount++;
                message.LastError = ex.Message;
                _logger.LogWarning(ex,
                    "Failed to process outbox message {Id} (attempt {Attempt})",
                    message.Id, message.RetryCount);
            }
        }

        await db.SaveChangesAsync(ct);
    }
}

// ── EF Core configuration ─────────────────────────────────────────────────────
// In OnModelCreating:
modelBuilder.Entity<OutboxMessage>(b =>
{
    b.HasKey(m => m.Id);
    b.Property(m => m.EventType).HasMaxLength(500).IsRequired();
    b.Property(m => m.Payload).HasColumnType("nvarchar(max)").IsRequired();
    b.HasIndex(m => m.ProcessedAt); // Index for WHERE ProcessedAt IS NULL query
    b.HasIndex(m => new { m.ProcessedAt, m.RetryCount }); // Composite for filtered query
```

```csharp
public class RequiredNotNullableSchemaFilter : ISchemaFilter
{
    public void Apply(OpenApiSchema schema, SchemaFilterContext context)
    {
        if (schema.Properties is null) return;

        var notNullableProperties = schema.Properties
            .Where(p => !p.Value.Nullable &&
                        p.Value.Default is null &&
                        !schema.Required.Contains(p.Key))
            .Select(p => p.Key)
            .ToList();

        foreach (var prop in notNullableProperties)
            schema.Required.Add(prop);
    }
}
```

```csharp
// appsettings.Development.json:
// { "Features": { "NewCheckout": true, "BetaAnalytics": true } }

// appsettings.Production.json:
// { "Features": { "NewCheckout": false, "BetaAnalytics": false } }

builder.Services.AddOptions<FeatureFlags>()
    .BindConfiguration("Features");

// Use in code
public class CheckoutService
{
    private readonly FeatureFlags _features;
    public CheckoutService(IOptionsSnapshot<FeatureFlags> features)
        => _features = features.Value;

    public async Task ProcessAsync(Order order, CancellationToken ct)
    {
        if (_features.NewCheckout)
            await ProcessWithNewCheckoutAsync(order, ct);
        else
            await ProcessWithLegacyCheckoutAsync(order, ct);
    }
}
```

## 12. COMMON INTERVIEW DEEP-DIVES

These are the questions that expose whether someone truly understands ASP.NET Core or just uses it.

---

**Q: What's the difference between `UseRouting()` and `MapControllers()`?**

`UseRouting()` runs the route matching engine — it inspects the request URL and HTTP method, matches it against all registered endpoint templates, and stores the matched `Endpoint` on `HttpContext`. It does NOT execute the endpoint. `MapControllers()` (or `MapGet`, `MapGroup`) *registers* endpoint descriptors into the route table. The actual execution happens in the endpoint middleware. The split matters because authorization middleware can run *between* `UseRouting` (match) and endpoint execution — it reads the matched endpoint's `[Authorize]` metadata before running the action.

---

**Q: Why does `IOptionsSnapshot<T>` not work in a singleton?**

`IOptionsSnapshot<T>` is a scoped service — it's created once per DI scope (per HTTP request). Singletons don't have a request scope. If you inject `IOptionsSnapshot<T>` into a singleton, it would need to be resolved from the root scope, which doesn't have a request context. ASP.NET Core's scope validation (enabled in Development) throws an `InvalidOperationException` if you attempt this. Use `IOptionsMonitor<T>` in singletons — it's a singleton itself and supports live reload via `OnChange`.

---

**Q: What is the captive dependency problem and how do you detect it?**

A captive dependency occurs when a longer-lived service (Singleton) holds a reference to a shorter-lived service (Scoped). The scoped service is captured at the time of injection into the singleton and never released — it lives for the application's lifetime, defeating its scoped purpose. If the scoped service holds a `DbContext`, all requests share the same context — data corruption and thread-safety violations result. Detection: enable `ServiceProviderOptions.ValidateScopes = true` (default in Development). Prevention: inject `IServiceScopeFactory` into singletons and create scopes manually.

---

**Q: How does JWT validation work without calling the identity server?**

JWT validation is cryptographic, not a network call. The token's signature is verified using the public key (for RS256) or the shared secret (for HS256) configured in `TokenValidationParameters.IssuerSigningKey`. If the signature is valid, the claims are trustworthy — no one could have tampered with them. The `exp`, `iss`, and `aud` claims are then checked against `TokenValidationParameters` values. The entire validation happens in-process in microseconds. The tradeoff: tokens cannot be revoked before expiry (you need a token blacklist/Redis lookup for revocation).

---

**Q: What happens when you don't pass `CancellationToken` to `SaveChangesAsync`?**

`SaveChangesAsync(CancellationToken.None)` always runs to completion regardless of client disconnect or host shutdown. This is actually **correct for write operations** — if a client disconnects after sending a POST, you still want the database write to complete. The client just won't receive the response. The mistake is using the request's `CancellationToken` for `SaveChangesAsync` on write paths — then a client disconnect aborts the commit, leaving data inconsistently unsaved. Correct pattern: use `CancellationToken` for reads (can be abandoned), use `CancellationToken.None` for writes.

---

**Q: Explain middleware vs filter — when would you use each?**

Middleware runs for every request regardless of the endpoint. Filters run only within the MVC/Minimal API execution pipeline, after route matching and model binding. Use middleware for cross-cutting concerns that apply globally (logging, CORS, authentication, timing, correlation IDs). Use filters for concerns specific to MVC actions (validation, transaction management, response transformation, per-controller caching). A key difference: filters have access to `ActionContext` (the controller, action name, model state); middleware does not. Filters can also short-circuit before model binding (resource filters), which middleware cannot do at that granularity.

---

**Q: How does output caching differ from response caching?**

`UseResponseCaching()` sets HTTP cache headers (`Cache-Control`, `ETag`) and instructs clients and CDNs to cache responses. The server stores nothing — it's delegated to the client/proxy. It requires careful handling of `Vary` headers and can serve stale content if headers are misconfigured. `UseOutputCache()` is server-side — ASP.NET Core stores the response in memory (or distributed store). The endpoint handler doesn't run for cache hits. Output cache supports tag-based invalidation (evict all `/api/products` responses by tag "products"). They're complementary: output cache eliminates server load; response cache headers let CDNs serve from the edge.
