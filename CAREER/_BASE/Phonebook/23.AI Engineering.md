## 23.01 — OWASP Top 10 (2021)

**A01 — Broken Access Control**
Failure to enforce that users can only perform actions/access data they're authorized for.

Common vulnerabilities:
- IDOR: `GET /api/orders/42` — user can access any order by changing the ID
- Privilege escalation: user can access admin functions
- Missing authorization checks on certain endpoints

Prevention in .NET:
```csharp
// WRONG — trusts the ID from the URL
[HttpGet("{orderId}")]
public async Task<OrderDto> GetOrder(int orderId) =>
    await _orders.GetByIdAsync(orderId);

// RIGHT — verify ownership
[HttpGet("{orderId}")]
public async Task<OrderDto> GetOrder(int orderId)
{
    var order = await _orders.GetByIdAsync(orderId);
    if (order is null) throw new NotFoundException();
    if (order.CustomerId != _currentUser.CustomerId)
        throw new ForbiddenException();  // 403, not 404 (don't reveal existence)
    return order.ToDto();
}

// Resource-based authorization
[Authorize]
public async Task<IActionResult> DeleteOrder(int orderId)
{
    var order = await _orders.GetByIdAsync(orderId);
    var authResult = await _authorizationService.AuthorizeAsync(
        User, order, OrderOperations.Delete);
    
    if (!authResult.Succeeded) return Forbid();
    
    await _orders.DeleteAsync(orderId);
    return NoContent();
}
```

**A02 — Cryptographic Failures**
Weak or missing encryption for sensitive data.

Prevention:
- Never store passwords as plaintext — use bcrypt, Argon2, or PBKDF2
- Encrypt sensitive data at rest (using ASP.NET Core Data Protection)
- Use TLS 1.2+ for all network communication
- Don't use MD5 or SHA-1 for security purposes

```csharp
// Password hashing with ASP.NET Core Identity
var hasher = new PasswordHasher<User>();
string hashed = hasher.HashPassword(user, plainTextPassword);
var result = hasher.VerifyHashedPassword(user, hashed, inputPassword);
```

**A03 — Injection**
User input is interpreted as part of a command or query.

```csharp
// SQL injection — NEVER do this
var sql = $"SELECT * FROM Users WHERE Email = '{userInput}'";
// Input: ' OR '1'='1 → returns all users

// Safe with parameterized queries
var users = await _db.Users
    .Where(u => u.Email == userInput)  // EF Core parameterizes automatically
    .ToListAsync();

// Raw SQL — safe:
var users = await _db.Users
    .FromSqlInterpolated($"SELECT * FROM Users WHERE Email = {userInput}")
    .ToListAsync();

// Raw SQL — UNSAFE (string interpolation before parameterization):
var users = await _db.Users
    .FromSqlRaw($"SELECT * FROM Users WHERE Email = {userInput}")
    .ToListAsync();
```

**A07 — Authentication Failures**
```csharp
// JWT configuration — secure defaults
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer = true,
            ValidateAudience = true,
            ValidateLifetime = true,
            ValidateIssuerSigningKey = true,
            ValidIssuer = builder.Configuration["Jwt:Issuer"],
            ValidAudience = builder.Configuration["Jwt:Audience"],
            IssuerSigningKey = new SymmetricSecurityKey(
                Encoding.UTF8.GetBytes(builder.Configuration["Jwt:Key"]!)),
            ClockSkew = TimeSpan.Zero  // Don't allow any clock drift
        };
    });
```

**A10 — SSRF (Server-Side Request Forgery)**
The server is tricked into making requests to internal services.

```csharp
// VULNERABLE — fetches any URL provided by user
app.MapPost("/proxy", async (ProxyRequest request, HttpClient client) =>
    await client.GetStringAsync(request.Url));

// SAFE — validate the URL
app.MapPost("/proxy", async (ProxyRequest request, HttpClient client) =>
{
    if (!Uri.TryCreate(request.Url, UriKind.Absolute, out var uri))
        return Results.BadRequest("Invalid URL");
    
    // Only allow external HTTP/HTTPS
    if (uri.Scheme != "https" && uri.Scheme != "http")
        return Results.BadRequest("Only HTTP/HTTPS allowed");
    
    // Block internal IP ranges
    if (IsInternalAddress(uri.Host))
        return Results.BadRequest("Internal URLs not allowed");
    
    return Results.Ok(await client.GetStringAsync(uri));
});
```

## 23.02 — Authentication — JWT and OAuth2

**JWT anatomy**
```
Header.Payload.Signature

Header:  {"alg": "HS256", "typ": "JWT"}
Payload: {"sub": "user-42", "name": "Alice", "iat": 1700000000, "exp": 1700086400}
Signature: HMACSHA256(base64url(header) + "." + base64url(payload), secret)
```

**Generating JWT in .NET**
```csharp
public string GenerateToken(User user)
{
    var key = new SymmetricSecurityKey(
        Encoding.UTF8.GetBytes(_config["Jwt:Key"]!));
    var credentials = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);
    
    var claims = new[]
    {
        new Claim(JwtRegisteredClaimNames.Sub, user.Id.ToString()),
        new Claim(JwtRegisteredClaimNames.Email, user.Email),
        new Claim(ClaimTypes.Role, user.Role),
        new Claim(JwtRegisteredClaimNames.Jti, Guid.NewGuid().ToString())
    };
    
    var token = new JwtSecurityToken(
        issuer: _config["Jwt:Issuer"],
        audience: _config["Jwt:Audience"],
        claims: claims,
        expires: DateTime.UtcNow.AddMinutes(15),  // Short-lived access token
        signingCredentials: credentials);
    
    return new JwtSecurityTokenHandler().WriteToken(token);
}
```

**Refresh token rotation**
```csharp
public async Task<TokenPair> RefreshAsync(string refreshToken)
{
    var stored = await _db.RefreshTokens
        .Include(rt => rt.User)
        .FirstOrDefaultAsync(rt => rt.Token == refreshToken);
    
    if (stored is null || stored.IsRevoked || stored.ExpiresAt < DateTime.UtcNow)
        throw new SecurityException("Invalid refresh token");
    
    // Rotation: revoke old token, issue new pair
    stored.IsRevoked = true;
    stored.RevokedAt = DateTime.UtcNow;
    
    var newRefreshToken = new RefreshToken
    {
        Token = GenerateSecureToken(),
        UserId = stored.UserId,
        ExpiresAt = DateTime.UtcNow.AddDays(30),
        Family = stored.Family  // Track family for reuse detection
    };
    
    _db.RefreshTokens.Add(newRefreshToken);
    await _db.SaveChangesAsync();
    
    return new TokenPair(
        GenerateJwt(stored.User),
        newRefreshToken.Token);
}
```

**OAuth 2.0 flows**
- **Authorization Code + PKCE**: For user-facing apps (web, mobile). Most secure.
- **Client Credentials**: Machine-to-machine (service accounts, background services).
- **Device Code**: IoT devices, CLI tools with limited display.
- **Implicit**: Deprecated — don't use.

## 23.03 — Authorization in ASP.NET Core

**Policy-based authorization**
```csharp
// Define policies
builder.Services.AddAuthorization(options =>
{
    options.AddPolicy("AdminOnly", policy =>
        policy.RequireRole("Admin"));
    
    options.AddPolicy("VerifiedUser", policy =>
        policy.RequireClaim("EmailVerified", "true")
              .RequireAuthenticatedUser());
    
    options.AddPolicy("MinimumAge", policy =>
        policy.AddRequirements(new MinimumAgeRequirement(18)));
    
    options.AddPolicy("OwnsOrder", policy =>
        policy.AddRequirements(new OrderOwnerRequirement()));
});

// Requirement + Handler
public class MinimumAgeRequirement : IAuthorizationRequirement
{
    public int MinimumAge { get; }
    public MinimumAgeRequirement(int minimumAge) => MinimumAge = minimumAge;
}

public class MinimumAgeHandler : AuthorizationHandler<MinimumAgeRequirement>
{
    protected override Task HandleRequirementAsync(
        AuthorizationHandlerContext context,
        MinimumAgeRequirement requirement)
    {
        var birthDateClaim = context.User.FindFirst("BirthDate");
        if (birthDateClaim is null) return Task.CompletedTask;
        
        var birthDate = DateOnly.Parse(birthDateClaim.Value);
        var age = DateOnly.FromDateTime(DateTime.Today).Year - birthDate.Year;
        
        if (age >= requirement.MinimumAge)
            context.Succeed(requirement);
        
        return Task.CompletedTask;
    }
}

// Resource-based authorization handler
public class OrderOwnerHandler : AuthorizationHandler<OrderOwnerRequirement, Order>
{
    protected override Task HandleRequirementAsync(
        AuthorizationHandlerContext context,
        OrderOwnerRequirement requirement,
        Order resource)
    {
        var userId = context.User.FindFirstValue(ClaimTypes.NameIdentifier);
        if (resource.CustomerId.ToString() == userId)
            context.Succeed(requirement);
        
        return Task.CompletedTask;
    }
}
```
