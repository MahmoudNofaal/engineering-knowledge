## 19.01 — Structured Logging with Serilog

**What it is**
Logging where each log entry is structured data (not just a string), enabling filtering, querying, and alerting in log management systems.

**Setup**
```csharp
// Program.cs
builder.Host.UseSerilog((context, services, configuration) =>
    configuration
        .ReadFrom.Configuration(context.Configuration)
        .ReadFrom.Services(services)
        .Enrich.FromLogContext()
        .Enrich.WithMachineName()
        .Enrich.WithEnvironmentName()
        .WriteTo.Console(new JsonFormatter())
        .WriteTo.Seq("http://localhost:5341")  // Dev
        .WriteTo.ApplicationInsights(
            services.GetRequiredService<TelemetryConfiguration>(),
            TelemetryConverter.Traces));  // Prod
```

**Correlation ID middleware**
```csharp
public class CorrelationIdMiddleware
{
    private readonly RequestDelegate _next;
    private const string CorrelationIdHeader = "X-Correlation-ID";
    
    public CorrelationIdMiddleware(RequestDelegate next) => _next = next;
    
    public async Task InvokeAsync(HttpContext context)
    {
        var correlationId = context.Request.Headers[CorrelationIdHeader].FirstOrDefault()
            ?? Guid.NewGuid().ToString();
        
        context.Response.Headers.TryAdd(CorrelationIdHeader, correlationId);
        
        using (LogContext.PushProperty("CorrelationId", correlationId))
        using (LogContext.PushProperty("RequestPath", context.Request.Path))
        {
            await _next(context);
        }
    }
}
```

**Structured logging patterns**
```csharp
// Good: structured properties
_logger.LogInformation(
    "Order {OrderId} created for customer {CustomerId} with total {Total:C}",
    order.Id, order.CustomerId, order.Total);

// Logs as JSON:
// {"Level":"Information","Message":"Order 42 created...","OrderId":42,"CustomerId":7,"Total":150.00}

// Bad: string concatenation destroys structure
_logger.LogInformation("Order " + order.Id + " created for customer " + order.CustomerId);

// Log levels:
_logger.LogDebug("Checking cache for key {Key}", key);          // Dev only
_logger.LogInformation("Request started: {Method} {Path}", ...); // Normal operations
_logger.LogWarning("Retry attempt {Attempt} for {Operation}", attempt, op);
_logger.LogError(ex, "Failed to process order {OrderId}", orderId);
_logger.LogCritical("Database connection lost — all requests failing");
```

## 19.02 — OpenTelemetry and Distributed Tracing

**What it is**
Distributed tracing tracks a request as it flows through multiple services, showing timing and errors at each step.

**Setup in ASP.NET Core**
```csharp
builder.Services.AddOpenTelemetry()
    .WithTracing(tracing =>
    {
        tracing
            .AddAspNetCoreInstrumentation()
            .AddHttpClientInstrumentation()
            .AddEntityFrameworkCoreInstrumentation()
            .AddSource("MyApp.*")  // Custom activity sources
            .AddJaegerExporter(jaeger =>
            {
                jaeger.AgentHost = "localhost";
                jaeger.AgentPort = 6831;
            });
    })
    .WithMetrics(metrics =>
    {
        metrics
            .AddAspNetCoreInstrumentation()
            .AddHttpClientInstrumentation()
            .AddRuntimeInstrumentation()
            .AddPrometheusExporter();
    });

// Custom tracing
private static readonly ActivitySource Source = new("MyApp.OrderService");

public async Task ProcessOrderAsync(int orderId)
{
```

**The three pillars of observability**

| Pillar | What it answers | Tool |
|---|---|---|
| Logs | What happened? | Serilog + Seq/Elasticsearch |
| Metrics | How is it performing? | Prometheus + Grafana |
| Traces | Where is the slowness? | Jaeger/Zipkin/Application Insights |

**SLI/SLO/SLA**
- **SLI** (Service Level Indicator): A metric measuring performance. E.g., "95th percentile latency"
- **SLO** (Service Level Objective): A target for an SLI. E.g., "95th percentile latency < 200ms for 99.9% of requests"
- **SLA** (Service Level Agreement): A contractual commitment with consequences. E.g., "99.9% uptime or credit issued"

## 19.03 — Application Insights in .NET

```csharp
// Registration
builder.Services.AddApplicationInsightsTelemetry(
    builder.Configuration["ApplicationInsights:InstrumentationKey"]);

// Custom telemetry
public class OrderService
{
    private readonly TelemetryClient _telemetry;
    
    public async Task CreateOrderAsync(CreateOrderCommand cmd)
    {
        var sw = Stopwatch.StartNew();
        try
        {
            var result = await ProcessAsync(cmd);
            
            // Track custom event
            _telemetry.TrackEvent("OrderCreated", new Dictionary<string, string>
            {
                ["CustomerId"] = cmd.CustomerId.ToString(),
                ["ItemCount"] = cmd.Items.Count.ToString()
            });
            
            // Track custom metric
            _telemetry.TrackMetric("OrderValue", (double)result.Total);
        }
        catch (Exception ex)
        {
            _telemetry.TrackException(ex, new Dictionary<string, string>
            {
                ["CustomerId"] = cmd.CustomerId.ToString()
            });
            throw;
        }
        finally
        {
            // Track dependency (external call timing)
            _telemetry.TrackDependency("Database", "CreateOrder", 
                sw.Elapsed, success: true);
        }
    }
}
```
