## 88.01 — Dockerfile for .NET

**Multi-stage Dockerfile (production)**
```dockerfile
# Stage 1: Build
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src

# Copy csproj files first (layer caching — only rebuilt when dependencies change)
COPY ["src/MyApp.Api/MyApp.Api.csproj", "src/MyApp.Api/"]
COPY ["src/MyApp.Application/MyApp.Application.csproj", "src/MyApp.Application/"]
COPY ["src/MyApp.Domain/MyApp.Domain.csproj", "src/MyApp.Domain/"]
COPY ["src/MyApp.Infrastructure/MyApp.Infrastructure.csproj", "src/MyApp.Infrastructure/"]

# Restore packages (cached unless .csproj changes)
RUN dotnet restore "src/MyApp.Api/MyApp.Api.csproj"

# Copy everything else
COPY . .

# Build and publish
RUN dotnet publish "src/MyApp.Api/MyApp.Api.csproj" \
    -c Release \
    -o /app/publish \
    --no-restore \
    /p:UseAppHost=false

# Stage 2: Runtime (much smaller)
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS final
WORKDIR /app

# Security: run as non-root user
RUN adduser --disabled-password --gecos '' appuser
USER appuser

# Copy only the published output
COPY --from=build --chown=appuser:appuser /app/publish .

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \
    CMD curl -f http://localhost:8080/health/live || exit 1

EXPOSE 8080
ENTRYPOINT ["dotnet", "MyApp.Api.dll"]
```

**Docker layer optimization**
```dockerfile
# BAD: Every code change rebuilds packages
FROM mcr.microsoft.com/dotnet/sdk:8.0
WORKDIR /app
COPY . .                          # Copy everything
RUN dotnet restore                 # Restore (rebuilds on any file change)
RUN dotnet publish -c Release

# GOOD: Restore cached separately
FROM mcr.microsoft.com/dotnet/sdk:8.0
WORKDIR /app
COPY *.csproj .                   # Copy only project files
RUN dotnet restore                 # Cached unless .csproj changes!
COPY . .                          # Copy remaining files
RUN dotnet publish -c Release -o publish
```

**Docker Compose for full local stack**
```yaml
version: '3.8'

services:
  api:
    build:
      context: .
      dockerfile: src/MyApp.Api/Dockerfile
    ports:
      - "8080:8080"

      - ASPNETCORE_ENVIRONMENT=Development
      - ConnectionStrings__Default=Server=sqlserver;Database=MyApp;User Id=sa;Password=Dev@1234;TrustServerCertificate=true
      - ConnectionStrings__Redis=redis:6379
    depends_on:
      sqlserver:
        condition: service_healthy
      redis:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health/live"]
      interval: 10s
      timeout: 5s
      retries: 3

    image: mcr.microsoft.com/mssql/server:2022-latest
    environment:
      ACCEPT_EULA: Y
      MSSQL_SA_PASSWORD: Dev@1234

      - "1433:1433"
    volumes:
      - sqlserver-data:/var/opt/mssql

      test: /opt/mssql-tools/bin/sqlcmd -S localhost -U sa -P 'Dev@1234' -Q 'SELECT 1' -b
      interval: 10s
      timeout: 3s

  redis:
    image: redis:7-alpine
```

## 88.02 — Container Security

**Minimal, hardened images**
```dockerfile
# Distroless images — no shell, no package manager, minimal attack surface
FROM mcr.microsoft.com/dotnet/aspnet:8.0-jammy-chiseled AS final
# jammy-chiseled: Ubuntu-based, ultra-minimal, ~50% smaller, non-root by default
# No bash, no curl, no package manager → dramatically reduced attack surface

# OR: Alpine-based (smallest but less compatible)
FROM mcr.microsoft.com/dotnet/aspnet:8.0-alpine AS final
```

**Image scanning**
```bash
# Trivy — scan for vulnerabilities
docker run --rm aquasec/trivy image myapp:latest

# Or in CI/CD:
trivy image --exit-code 1 --severity HIGH,CRITICAL myapp:latest
# Fails CI if HIGH or CRITICAL vulnerabilities found
```

**Secrets — never in image**
```dockerfile
# WRONG: Secret in Dockerfile
ENV DB_PASSWORD=MySecretPassword  # IN IMAGE HISTORY FOREVER

# WRONG: Secret in COPY
COPY appsettings.Production.json .  # Contains secrets

# RIGHT: Environment variables from secrets manager at runtime
# K8s: secretKeyRef, AWS: SSM Parameter Store, Azure: Key Vault
```
