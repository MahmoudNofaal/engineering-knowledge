# CATEGORY 71 — MONOLITH TO MICROSERVICES

---

## 71.01 — The Strangler Fig Pattern

**What it is**
Incrementally replace a monolith by building new features as microservices and gradually migrating old features.

**Process**
```
Phase 1: New requests, new code
  All requests → Monolith
  
  Build: New "Orders" microservice (new code only)
  Route: All ORDER requests → Orders service
         All other requests → Monolith
  
Phase 2: Extract existing functionality
  Build: Extract "Customers" from Monolith to Customers microservice
  Migrate: Customer data moved to Customers service database
  Route: Customer requests → Customers service
         Order requests → Orders service (calls Customers service)
         All others → Monolith
  
Phase 3: Monolith shrinks until gone
  Eventually: Monolith has no responsibilities left → decomission
```

**The facade/proxy**
```csharp
// API Gateway or reverse proxy routes based on path
// NGINX config:
location /api/orders/ {
    proxy_pass http://orders-service:8080/;
}

location /api/customers/ {
    proxy_pass http://customers-service:8080/;
}

location / {
    proxy_pass http://monolith:8080/;  // Everything else still goes to monolith
}
```

## 71.02 — Event-Driven Data Migration

**Problem**: Moving data from monolith to microservice without downtime.

**Dual-write approach**
```csharp
// Phase 1: Write to both (monolith DB + new service DB)
// Phase 2: Read from new service, verify against monolith
// Phase 3: Stop writing to monolith, stop reading from it

public class OrderService
{
    private readonly MonolithDbContext _monolithDb;
    private readonly INewOrdersServiceClient _newOrdersService;
    private readonly IFeatureFlags _flags;
    
    public async Task CreateOrderAsync(CreateOrderCommand cmd, CancellationToken ct)
    {
        // Always write to monolith (during migration)
        var order = await _monolithDb.CreateOrderAsync(cmd, ct);
        
        if (_flags.IsEnabled("DualWriteOrders"))
        {
            // Also write to new service (can fail without affecting primary)
            try
            {
                await _newOrdersService.CreateAsync(order, ct);
            }
            catch (Exception ex)
            {
                _logger.LogWarning(ex, "Dual-write to new Orders service failed");
                // Don't throw — primary (monolith) succeeded
            }
        }
        
        return order;
    }
    
    public async Task<Order?> GetOrderAsync(int id, CancellationToken ct)
    {
        if (_flags.IsEnabled("ReadFromNewOrdersService"))
        {
            var order = await _newOrdersService.GetAsync(id, ct);
            
            if (_flags.IsEnabled("VerifyAgainstMonolith"))
            
            {
                var monolithOrder = await _monolithDb.GetOrderAsync(id, ct);
                if (!order.Matches(monolithOrder))
                    _logger.LogWarning("Data mismatch for order {Id}", id);
            }
            
            return order;
        }
        
        return await _monolithDb.GetOrderAsync(id, ct);
    }
}
```
