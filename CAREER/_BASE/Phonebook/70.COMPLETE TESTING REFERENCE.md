## 70.01 — Testing Pyramid and Strategy

**The testing pyramid**
```
              ╱╲
             ╱  ╲       E2E Tests (Playwright, Selenium)
            ╱────╲      Few, slow, expensive, high confidence in integration
           ╱      ╲
          ╱────────╲    Integration Tests (TestContainers, WebApplicationFactory)
         ╱          ╲   Some, moderate speed, test components together
        ╱────────────╲
       ╱              ╲  Unit Tests (xUnit, NUnit)
      ╱────────────────╲ Many, fast, cheap, test one thing in isolation
```

**Distribution guideline**
- Unit: 70% — test all business logic
- Integration: 25% — test database queries, API endpoints
- E2E: 5% — test critical user journeys

**Test naming convention**
```csharp
// Pattern: MethodName_Scenario_ExpectedBehavior
[Fact] void CreateOrder_WithValidData_ReturnsOrderId()
[Fact] void CreateOrder_WithEmptyItems_ThrowsValidationException()
[Fact] void CreateOrder_WhenCustomerNotFound_ThrowsNotFoundException()

// Or: Given_When_Then (BDD style)
[Fact] void GivenExpiredToken_WhenRefreshCalled_ThenNewTokenIssuedAndOldRevoked()
```

## 70.02 — Test Doubles — Types and When to Use

```csharp
// STUB: Returns hardcoded values, doesn't verify calls
public class StubOrderRepository : IOrderRepository
{
    public Task<Order?> GetByIdAsync(int id, CancellationToken ct)
        => Task.FromResult<Order?>(new Order { Id = id, Status = "Pending" });
}

// MOCK: Verifies that specific calls were made
// (NSubstitute creates mocks automatically)
var repo = Substitute.For<IOrderRepository>();
repo.GetByIdAsync(42, Arg.Any<CancellationToken>())
    .Returns(new Order { Id = 42 });

// After SUT execution:
await repo.Received(1).SaveAsync(Arg.Is<Order>(o => o.Status == "Confirmed"), Arg.Any<CancellationToken>());
// Verifies SaveAsync was called exactly once with a Confirmed order

// FAKE: Simplified real implementation
public class InMemoryOrderRepository : IOrderRepository
{
    private readonly Dictionary<int, Order> _store = new();
    private int _nextId = 1;
    
    public Task<Order?> GetByIdAsync(int id, CancellationToken ct)
        => Task.FromResult(_store.TryGetValue(id, out var order) ? order : null);
    
    public Task SaveAsync(Order order, CancellationToken ct)
    {
        if (order.Id == 0) order.Id = _nextId++;
        _store[order.Id] = order;
        return Task.CompletedTask;
    }
}

// SPY: Records calls for later inspection (NSubstitute does this too)
var spy = Substitute.For<IEmailService>();
await sut.ProcessAsync(command);
var receivedCalls = spy.ReceivedCalls().ToList();
Assert.Equal(1, receivedCalls.Count);

// When to use which:
// Unit tests: Mocks and Stubs (fast, isolated)
// Integration tests: Fakes (real behavior without external dependencies)
// System tests: Real implementations with test database (TestContainers)
```

## 70.03 — Testing Async Code

```csharp
// Testing async methods — just await them
[Fact]
public async Task GetOrderAsync_WithValidId_ReturnsOrder()
{
    var result = await _sut.GetOrderAsync(42, CancellationToken.None);
    result.Should().NotBeNull();
    result!.Id.Should().Be(42);
}

// Testing CancellationToken propagation
[Fact]
public async Task GetOrderAsync_WhenCancelled_ThrowsOperationCancelledException()
{
    using var cts = new CancellationTokenSource();
    cts.Cancel();  // Already cancelled
    
    await _sut.Invoking(s => s.GetOrderAsync(42, cts.Token))
        .Should().ThrowAsync<OperationCanceledException>();
}

// Testing timeouts
[Fact]
public async Task GetOrderAsync_WithShortTimeout_ThrowsOperationCancelledException()
{
    using var cts = new CancellationTokenSource(TimeSpan.FromMilliseconds(1));
    
    // Make the mock slow
    _repo.GetByIdAsync(Arg.Any<int>(), Arg.Any<CancellationToken>())
        .Returns(async (CallInfo _) =>
        {
            await Task.Delay(TimeSpan.FromSeconds(5));
            return new Order();
        });
    
    await _sut.Invoking(s => s.GetOrderAsync(42, cts.Token))
        .Should().ThrowAsync<OperationCanceledException>();
}

// Testing IAsyncEnumerable
[Fact]
public async Task GetOrdersStreamAsync_ReturnsAllOrders()
{
    var orders = new List<Order>();
    await foreach (var order in _sut.GetOrdersStreamAsync(CancellationToken.None))
        orders.Add(order);
    
    orders.Should().HaveCountGreaterThan(0);
}
```

## 70.04 — Snapshot Testing

**What it is**
On first run, capture the output as a "snapshot". On subsequent runs, compare output to snapshot. Fail if different. Good for: API response shapes, serialization output, HTML rendering.

```csharp
// NuGet: Verify.Xunit or Snapshooter

[Fact]
public async Task GetOrder_ResponseMatchesSnapshot()
{
    var client = _factory.CreateClient();
    var response = await client.GetAsync("/api/orders/1");
    var content = await response.Content.ReadAsStringAsync();
    
    await Verify(content);  // First run: creates .verified.txt snapshot file
                             // Subsequent runs: compares to snapshot, fails if different
}
```
