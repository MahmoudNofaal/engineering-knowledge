## 58.01 — Hangfire Complete Reference

**What it is**
A background job processing library for .NET. Jobs survive application restarts (stored in SQL Server, Redis, etc.).

**Setup**
```csharp
// NuGet: Hangfire, Hangfire.AspNetCore, Hangfire.SqlServer
builder.Services.AddHangfire(config =>
    config
        .SetDataCompatibilityLevel(CompatibilityLevel.Version_180)
        .UseSimpleAssemblyNameTypeSerializer()
        .UseRecommendedSerializerSettings()
        .UseSqlServerStorage(builder.Configuration.GetConnectionString("Hangfire"),
            new SqlServerStorageOptions
            {
                CommandBatchMaxTimeout = TimeSpan.FromMinutes(5),
                SlidingInvisibilityTimeout = TimeSpan.FromMinutes(5),
                QueuePollInterval = TimeSpan.FromSeconds(5),
                UseRecommendedIsolationLevel = true,
                DisableGlobalLocks = true
            }));

builder.Services.AddHangfireServer(options =>
{
    options.WorkerCount = Environment.ProcessorCount * 2;  // CPU-bound: *1, IO-bound: *5
    options.Queues = new[] { "critical", "default", "low" };  // Priority queues
});

// Dashboard (protect in production!)
app.UseHangfireDashboard("/hangfire", new DashboardOptions
{
    Authorization = new[] { new HangfireAuthorizationFilter() }
});
```

**Job types**
```csharp
// 1. Fire-and-forget
var jobId = BackgroundJob.Enqueue<IOrderService>(
    svc => svc.ProcessOrderAsync(orderId, CancellationToken.None));

// 2. Delayed job
BackgroundJob.Schedule<IEmailService>(
    svc => svc.SendReminderAsync(userId, CancellationToken.None),
    TimeSpan.FromHours(24));

// 3. Recurring job (cron)
RecurringJob.AddOrUpdate<IReportService>(
    "generate-daily-report",
    svc => svc.GenerateDailyReportAsync(CancellationToken.None),
    Cron.Daily(8, 0),  // 8:00 AM daily
    new RecurringJobOptions { TimeZone = TimeZoneInfo.Utc });

// 4. Continuation (run after another job)
var parentId = BackgroundJob.Enqueue<IFileService>(
    svc => svc.UploadAsync(fileId, CancellationToken.None));

BackgroundJob.ContinueJobWith<INotificationService>(
    parentId,
    svc => svc.NotifyUploadCompleteAsync(fileId, CancellationToken.None));

// 5. Batch (Hangfire Pro only)
var batchId = BatchJob.StartNew(batch =>
{
    batch.Enqueue<IOrderProcessor>(p => p.ProcessAsync(1));
    batch.Enqueue<IOrderProcessor>(p => p.ProcessAsync(2));
    batch.Enqueue<IOrderProcessor>(p => p.ProcessAsync(3));
});

BatchJob.ContinueJobWith(batchId, batch =>
{
    batch.Enqueue<IReportService>(r => r.GenerateSummaryAsync());
});
```

**Priority queues**
```csharp
// Enqueue to specific queue
[Queue("critical")]
public class CriticalOrderProcessor
{
    public void Process(int orderId) { }
}

// Or specify at enqueue time:
BackgroundJob.Enqueue<IOrderProcessor>(
    p => p.ProcessAsync(orderId),
    new EnqueuedState("critical"));

// Worker processes "critical" before "default" before "low"
options.Queues = new[] { "critical", "default", "low" };
```

**Retry configuration**
```csharp
// Global retry policy
GlobalJobFilters.Filters.Add(new AutomaticRetryAttribute
{
    Attempts = 5,
    DelaysInSeconds = new[] { 60, 300, 600, 3600, 86400 },  // 1min, 5min, 10min, 1hr, 1day
    OnAttemptsExceeded = AttemptsExceededAction.Fail
});

// Per-job retry
[AutomaticRetry(Attempts = 3, DelaysInSeconds = new[] { 10, 30, 60 })]
public async Task SendEmailAsync(string to, string subject, CancellationToken ct)
{
    // If this throws, Hangfire retries automatically
    await _emailClient.SendAsync(to, subject, ct);
}

// Disable retry for specific job
[AutomaticRetry(Attempts = 0)]
public async Task OneTimeSetupAsync(CancellationToken ct) { }
```

**Distributed lock (one instance at a time)**
```csharp
[DisableConcurrentExecution(timeoutInSeconds: 300)]
public async Task GenerateDailyReportAsync(CancellationToken ct)
{
    // Only one instance runs at a time across all servers
    // If another instance is running, this one waits up to 5 minutes
}

// Skip if already running (don't queue another)
// Implement via distributed lock in Redis:
public async Task ScheduleIfNotRunning()
{
    var lockKey = "recurring:daily-report:lock";
    var acquired = await _redis.StringSetAsync(lockKey, "1", TimeSpan.FromHours(2), When.NotExists);
    if (!acquired) return;  // Already running
    
    BackgroundJob.Schedule<IReportService>(
        svc => svc.GenerateDailyReportAsync(CancellationToken.None),
        TimeSpan.Zero);
}
```

**Job argument serialization**
```csharp
// Hangfire serializes job arguments to JSON in the database
// Keep arguments small — don't pass large objects

// WRONG: Large object as argument
BackgroundJob.Enqueue<IProcessor>(p => p.Process(largeOrder));  // Serializes entire order!

// RIGHT: Pass only the ID, load from DB in the job
BackgroundJob.Enqueue<IProcessor>(p => p.Process(orderId));

public async Task Process(int orderId, CancellationToken ct)

    var order = await _repo.GetByIdAsync(orderId, ct);  // Fresh data from DB
```

**Job dashboard security**
```csharp
public class HangfireAuthorizationFilter : IDashboardAuthorizationFilter
{
    public bool Authorize(DashboardContext context)
    {
        var httpContext = context.GetHttpContext();
        
        // Require authentication
        if (!httpContext.User.Identity?.IsAuthenticated ?? true)
            return false;
        
        // Require specific role
        return httpContext.User.IsInRole("Admin");
    }
}
```
