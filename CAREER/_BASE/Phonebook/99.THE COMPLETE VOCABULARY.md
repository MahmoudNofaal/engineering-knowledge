## 99.01 — Terms Every Software Engineer Must Know

**Fundamental terms**
```
Idempotent:      Applying multiple times has same effect as applying once.
                 GET /users/42 is idempotent — same result each call.
                 POST /orders is NOT idempotent — creates new order each call.

Pure function:   Given same inputs, always returns same output. No side effects.
                 string.ToUpper() is pure.
                 DateTime.Now is NOT pure.

Side effect:     Any observable effect beyond returning a value.
                 Writing to a file, database, network, modifying shared state.

Deterministic:   Same inputs always produce same outputs.
                 Random.Next() is non-deterministic.

Referential      An expression can be replaced by its value without changing behavior.
transparency:    Only possible with pure functions.

Cohesion:        How closely related the responsibilities of a single module are.
                 High cohesion = focused, single-purpose. GOOD.
                 Low cohesion = does many unrelated things. BAD.

Coupling:        How dependent modules are on each other.
                 Low coupling = modules can change independently. GOOD.
                 High coupling = change one, must change all. BAD.
                 Goal: high cohesion, low coupling.

Abstraction:     Hiding implementation details, exposing only the interface.
                 IRepository<T> is an abstraction over database access.

Encapsulation:   Hiding internal state, requiring access through defined interface.
                 Private fields + public methods = encapsulation.
```

**Concurrency terms**
```
Race condition:  Behavior depends on the timing of uncontrollable events.
                 Two threads read count (100), both increment, both write 101. Should be 102.

Deadlock:        Two or more threads wait for each other forever.
                 Thread A holds Lock1, waits for Lock2.
                 Thread B holds Lock2, waits for Lock1. → Stuck.

Livelock:        Threads change state in response to each other but make no progress.
                 Like two people repeatedly stepping aside for each other in a hallway.

Starvation:      Thread is indefinitely denied necessary resources.
                 Low-priority thread never runs because high-priority threads always preempt it.

Thread safety:   Code that behaves correctly under concurrent execution.

Atomic operation: Executes completely or not at all. Not interruptible.
                  Interlocked.Increment is atomic.
                  i++ is NOT atomic (read-modify-write, three operations).

Memory barrier:  Prevents CPU/compiler from reordering memory operations.
                 volatile keyword adds memory barriers in C#.

ABA problem:     Thread A reads value A. Thread B changes to B then back to A.
                 Thread A's compare-and-swap succeeds but state has changed.
```

**Distributed systems terms**
```
CAP Theorem:     A distributed system can guarantee at most 2 of:
                 Consistency, Availability, Partition tolerance.
                 Networks always partition → choose C or A.
                 
                 CP systems: consistent but may be unavailable during partition.
                 AP systems: always available but may return stale data.

Eventual        Given no new updates, all replicas will converge to same value.
consistency:    Not strong consistency (reads may see stale data temporarily).

Strong          Every read receives the most recent write. Like a single machine.
consistency:    Requires coordination → higher latency.

Linearizable:   Operations appear to occur at a single point in time.
                Strongest consistency model. Required for: counters, locks.

Serializability: Transaction results as if transactions ran serially (one after another).
                 Database isolation level: SERIALIZABLE.

ACID:           Atomicity, Consistency, Isolation, Durability.
                Properties of reliable database transactions.

BASE:           Basically Available, Soft state, Eventually consistent.
                Alternative to ACID for distributed databases.

Idempotency key: Unique identifier sent with request. Server deduplicates.
                 Safe to retry POST /payments with same idempotency key.

At-least-once   Message delivered at least once. May be delivered multiple times.
delivery:       Consumer must be idempotent.

Exactly-once    Message delivered exactly once. Requires deduplication.
delivery:       Expensive to guarantee. Often emulated with idempotent consumers.

SAGA:           Sequence of local transactions with compensating actions for rollback.
                Distributed alternative to 2-phase commit.

Backpressure:   Signal from consumer to producer to slow down.
                System.Threading.Channels.BoundedChannel with FullMode.Wait.
```

**Performance terms**
```
Throughput:     Operations per unit time. req/sec, msg/sec.
Latency:        Time for one operation. P50, P95, P99, P999.
Percentile:     P99 = 99% of requests complete within this time.
                1% of requests (the slowest 1%) take longer than P99.

Jitter:         Variance in latency. High jitter = unpredictable.

Throughput vs.  Increasing throughput often increases latency.
latency trade-off: Batching improves throughput but adds latency.

HOL blocking:   Head-of-Line blocking. First request in queue blocks all behind it.
                HTTP/1.1 has HOL blocking. HTTP/2 eliminates it.

Cold start:     First invocation after idle period is slow.
                JIT compilation, connection pool warming, cache population.

Warm-up:        Period after start where performance improves as caches fill.

Cache miss:     Requested item not in cache. Must fetch from slower source.
Cache stampede: Many requests miss cache simultaneously, all hit origin.
                Solved by: mutex coalescing, probabilistic early expiry.

GC pause:       Time where application is paused for garbage collection.
                Stop-the-world vs. concurrent/background collection.

Memory leak:    Memory allocated but never freed. Grows unboundedly.
                In .NET: objects held in long-lived collections, event handler subscriptions.
```
