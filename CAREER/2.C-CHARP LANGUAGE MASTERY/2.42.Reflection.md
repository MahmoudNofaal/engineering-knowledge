# C# Reflection

> The ability to inspect and invoke types, methods, properties, and attributes at runtime — examine the structure of a type without knowing it at compile time, create instances dynamically, and call methods by name.

## When To Use It

Use reflection when the types you need to work with are genuinely unknown at compile time — serialisers, dependency injection containers, ORMs, plugin architectures, test frameworks, and attribute-driven frameworks. These are infrastructure tools that must be generic across all user-defined types.

Don't use reflection in hot paths or application-level code where the types are known. The overhead (10–1000× slower than direct calls), the loss of compile-time safety, and the `ArgumentException`/`NullReferenceException` mines make it painful to maintain and debug. The right pattern: reflect once at startup, compile what you discover into delegates, then call those delegates at speed.

**The correct alternatives by scenario:**

- Compile-time polymorphism → generics with constraints
- Type-safe dynamic dispatch → pattern matching + sealed hierarchies
- Attribute-driven frameworks where performance matters → source generators
- Mapping between known types → hand-written or source-generated mappers

## Core Concept

At runtime, every type in .NET has a corresponding `Type` object in a metadata table. `typeof(Order)` and `new Order().GetType()` both return a reference to that same shared `Type` object. From it, you can enumerate members, get `MethodInfo`/`PropertyInfo` objects, and invoke them.

The cost model is non-uniform:

- `GetType()` is fast — reads the type handle from the object header
- `GetMethod` / `GetProperty` is moderate — scans metadata tables with caching
- `MethodInfo.Invoke()` is expensive — boxes arguments into an `object[]`, does a security check, unboxes the result; roughly 10–100× slower than a direct call

The key performance pattern: **reflect once, compile to a delegate, invoke the delegate many times.** Use `Expression.Lambda<T>` or `Delegate.CreateDelegate` to turn a `MethodInfo` into a strongly-typed callable after discovery. Cache the result in a `static readonly` field or `ConcurrentDictionary`. The compilation cost is paid once at startup; subsequent invocations are indistinguishable from a normal call.

## Performance

|Operation|Cost|Notes|
|---|---|---|
|`GetType()`|O(1) — ~1 ns|Reads type handle from object header|
|`typeof(T)`|O(1) — compile-time|Resolved by compiler; no runtime lookup|
|`GetMethod(name)` / `GetProperty(name)`|O(n members) first call, cached|CLR caches metadata after first scan|
|`MethodInfo.Invoke()`|~100–1000 ns per call|Boxes args, security check, unboxes result|
|`PropertyInfo.GetValue()` / `SetValue()`|~100–500 ns per call|Boxes value type results|
|`Activator.CreateInstance(type)`|~500 ns|Finds + calls parameterless constructor|
|Compiled delegate from `Expression.Lambda`|~1–5 ns per call|Same as direct virtual call after compilation|
|`Delegate.CreateDelegate`|~1–3 ns per call|Direct method pointer after compilation|
|`UnsafeAccessor` (.NET 8)|~1–2 ns per call|Direct field/method access via IL trick|

**Allocation behaviour:** `GetType()` returns a shared `Type` object — no allocation. `MethodInfo.Invoke()` allocates an `object[]` for the argument list plus boxes every value type argument and the return value. For a method accepting two `int` parameters and returning `int`, one `Invoke` call can allocate 3–5 objects. Compiling to a typed delegate eliminates all of this.

**Benchmark notes:** The inflection point where reflection becomes a bottleneck is roughly when a reflective call is in any loop that runs more than a few hundred iterations, or when it's on the hot path of a web request handler. Measure with BenchmarkDotNet's `[MemoryDiagnoser]` before optimising — the compilation overhead of `Expression.Lambda` is measurable at startup, so it's only worth it if the delegate is called frequently.

## The Code

**Inspecting types and their members**

```csharp
Type t = typeof(Order);

Console.WriteLine(t.FullName);          // "MyApp.Models.Order"
Console.WriteLine(t.IsClass);           // true
Console.WriteLine(t.IsAbstract);        // false
Console.WriteLine(t.BaseType?.Name);    // "Object" (or whatever the base is)

// Enumerate public instance properties
foreach (PropertyInfo prop in t.GetProperties())
    Console.WriteLine($"  {prop.PropertyType.Name} {prop.Name} [get={prop.CanRead} set={prop.CanWrite}]");

// Get a specific method — overloads require parameter types
MethodInfo? method = t.GetMethod("CalculateTotal"); // unambiguous
MethodInfo? overload = t.GetMethod("Add", new[] { typeof(int), typeof(int) }); // disambiguate

// Non-public members need BindingFlags
FieldInfo? field = t.GetField("_internalState",
    BindingFlags.NonPublic | BindingFlags.Instance);
```

**Invoking methods dynamically**

```csharp
// Create instance and invoke — standard reflection (slow path)
Type calculatorType = typeof(Calculator);
object calc   = Activator.CreateInstance(calculatorType)!;
MethodInfo add = calculatorType.GetMethod("Add", new[] { typeof(int), typeof(int) })!;

// Invoke: args must be object[] — boxes value types, expensive
int result = (int)add.Invoke(calc, new object[] { 3, 4 })!; // 7

// Property read and write
object order = Activator.CreateInstance(typeof(Order))!;
PropertyInfo totalProp = typeof(Order).GetProperty("Total")!;

totalProp.SetValue(order, 99.99m);           // boxes decimal
decimal val = (decimal)totalProp.GetValue(order)!; // unboxes

// Null safety: always check for missing members before invoking
PropertyInfo? optional = typeof(Order).GetProperty("UnknownProp");
if (optional?.CanRead == true)
    Console.WriteLine(optional.GetValue(order));
```

**Compiling reflection to a delegate — cache the cost once**

```csharp
// Compile a MethodInfo to a strongly-typed Func — reflects once, calls fast
static Func<TTarget, TResult> CompileGetter<TTarget, TResult>(string propertyName)
{
    PropertyInfo prop = typeof(TTarget).GetProperty(propertyName)
        ?? throw new ArgumentException($"Property '{propertyName}' not found.");

    // Build expression: (TTarget target) => target.PropertyName
    var param = Expression.Parameter(typeof(TTarget), "target");
    var body  = Expression.Property(param, prop);
    return Expression.Lambda<Func<TTarget, TResult>>(body, param).Compile();
}

// Compile once at startup (or in a static constructor)
private static readonly Func<Order, decimal> GetTotal = CompileGetter<Order, decimal>("Total");

// Subsequent calls are direct delegate invocations — same speed as a property access
void ProcessOrders(IEnumerable<Order> orders)
{
    foreach (var order in orders)
        Console.WriteLine(GetTotal(order)); // no reflection overhead
}
```

**Reading custom attributes**

```csharp
[AttributeUsage(AttributeTargets.Property)]
public sealed class AuditAttribute : Attribute
{
    public string Reason { get; }
    public AuditAttribute(string reason) => Reason = reason;
}

public class Order
{
    [Audit("Changed when customer modifies order")]
    public string CustomerName { get; set; } = "";

    [Audit("Price approved by pricing engine")]
    public decimal Total { get; set; }

    public OrderStatus Status { get; set; } // not audited
}

// Discover audited properties — reflect and cache at startup
var auditedProperties = typeof(Order)
    .GetProperties()
    .Select(p => new
    {
        Property  = p,
        Attribute = p.GetCustomAttribute<AuditAttribute>()
    })
    .Where(x => x.Attribute is not null)
    .ToList();

foreach (var a in auditedProperties)
    Console.WriteLine($"{a.Property.Name}: {a.Attribute!.Reason}");
```

**Generic type reflection — open vs closed types**

```csharp
// Open generic type: List<> with no type argument
Type openList = typeof(List<>);
Console.WriteLine(openList.IsGenericTypeDefinition); // true

// Close it with a concrete type argument
Type closedList = openList.MakeGenericType(typeof(int)); // List<int>
object instance  = Activator.CreateInstance(closedList)!;

// Inspect generic arguments on a closed type
Type closedDict = typeof(Dictionary<string, int>);
Type[] typeArgs = closedDict.GetGenericArguments(); // [string, int]

// Invoke a generic method by closing it at runtime
MethodInfo openMethod = typeof(Enumerable)
    .GetMethods()
    .First(m => m.Name == "ToList" && m.GetParameters().Length == 1);

MethodInfo closedMethod = openMethod.MakeGenericMethod(typeof(Order));
var list = closedMethod.Invoke(null, new object[] { someEnumerable }); // ToList<Order>()
```

**`UnsafeAccessor` — fast private member access (.NET 8)**

```csharp
// Avoids full reflection overhead for accessing private members
// Useful for testing or when you need to call internal library methods
public class TargetClass
{
    private int _counter;
    private static int _staticCounter;
    private void IncrementInternal() => _counter++;
}

// Each accessor is a static extern method with [UnsafeAccessor]
[UnsafeAccessor(UnsafeAccessorKind.Field, Name = "_counter")]
extern static ref int GetCounter(TargetClass target);

[UnsafeAccessor(UnsafeAccessorKind.Method, Name = "IncrementInternal")]
extern static void CallIncrementInternal(TargetClass target);

// Usage: direct access without Invoke overhead
var obj = new TargetClass();
CallIncrementInternal(obj);
ref int counter = ref GetCounter(obj); // direct reference to the field
Console.WriteLine(counter); // 1
```

## Real World Example

A lightweight object mapper uses reflection to copy matching properties between types, but pre-compiles the mapping at startup so the hot path has zero reflection overhead.

```csharp
public static class TypeMapper
{
    private static readonly ConcurrentDictionary<(Type Src, Type Dst), Action<object, object>> _cache
        = new();

    public static TDest Map<TSource, TDest>(TSource source) where TDest : new()
    {
        var mapper = _cache.GetOrAdd(
            (typeof(TSource), typeof(TDest)),
            BuildMapper);

        var dest = new TDest();
        mapper(source!, dest);
        return dest;
    }

    private static Action<object, object> BuildMapper((Type Src, Type Dst) key)
    {
        // Reflect once — find properties with matching name and compatible type
        var srcProps = key.Src.GetProperties(BindingFlags.Public | BindingFlags.Instance)
            .Where(p => p.CanRead)
            .ToDictionary(p => p.Name);

        var assignments = key.Dst
            .GetProperties(BindingFlags.Public | BindingFlags.Instance)
            .Where(p => p.CanWrite && srcProps.TryGetValue(p.Name, out var sp)
                     && p.PropertyType.IsAssignableFrom(sp.PropertyType))
            .Select(dst =>
            {
                var src = srcProps[dst.Name];

                // Build typed expression: dst.Prop = (T)src.Prop
                var srcParam = Expression.Parameter(typeof(object), "src");
                var dstParam = Expression.Parameter(typeof(object), "dst");

                var assignment = Expression.Assign(
                    Expression.Property(Expression.Convert(dstParam, key.Dst), dst),
                    Expression.Convert(
                        Expression.Property(Expression.Convert(srcParam, key.Src), src),
                        dst.PropertyType));

                return (Expression)assignment;
            })
            .ToList();

        if (assignments.Count == 0)
            return static (_, _) => { }; // nothing to map

        // Compile to a single Action<object, object> — no reflection on subsequent calls
        var srcParam2 = Expression.Parameter(typeof(object), "src");
        var dstParam2 = Expression.Parameter(typeof(object), "dst");
        var body       = Expression.Block(assignments
            .Select(a =>
            {
                // Rewrite parameter references to use the lambda's parameters
                return new ParameterRewriter(
                    Expression.Parameter(typeof(object), "src"),
                    Expression.Parameter(typeof(object), "dst"),
                    srcParam2, dstParam2).Visit(a);
            }));

        return Expression.Lambda<Action<object, object>>(body, srcParam2, dstParam2).Compile();
    }

    // Helper to rewrite parameter references during expression tree reuse
    private sealed class ParameterRewriter : ExpressionVisitor
    {
        private readonly ParameterExpression _oldSrc, _oldDst, _newSrc, _newDst;

        public ParameterRewriter(
            ParameterExpression oldSrc, ParameterExpression oldDst,
            ParameterExpression newSrc, ParameterExpression newDst)
            => (_oldSrc, _oldDst, _newSrc, _newDst) = (oldSrc, oldDst, newSrc, newDst);

        protected override Expression VisitParameter(ParameterExpression node)
        {
            if (node == _oldSrc) return _newSrc;
            if (node == _oldDst) return _newDst;
            return base.VisitParameter(node);
        }
    }
}

// Usage — zero reflection on every call after the first for each type pair
public class OrderDto
{
    public int Id { get; set; }
    public string CustomerName { get; set; } = "";
    public decimal Total { get; set; }
}

public class OrderSummary
{
    public int Id { get; set; }
    public string CustomerName { get; set; } = "";
    public decimal Total { get; set; }
    public string? ExtraField { get; set; } // unmatched — ignored
}

var dto     = new OrderDto { Id = 1, CustomerName = "Alice", Total = 99.99m };
var summary = TypeMapper.Map<OrderDto, OrderSummary>(dto);
// First call: reflects + compiles expression tree (~500 µs)
// Every subsequent call: executes compiled Action<object,object> (~3 ns)
```

_The key insight: the mapper reflects on type pairs exactly once — when first encountered. The discovered property mappings are compiled into an expression tree that the JIT turns into direct field assignments. Adding 50 more `OrderDto` → `OrderSummary` mappings doesn't add reflection cost per call; it only adds one compilation at startup._

## Common Misconceptions

**"Reflection is always slow — avoid it completely"** Reflection's cost is front-loaded in discovery and invocation. `GetType()` is essentially free. `MethodInfo.Invoke()` is expensive per call — but if you compile the discovered `MethodInfo` to a typed delegate and cache it, the invocation cost drops to zero. Libraries like AutoMapper, EF Core, and System.Text.Json all use reflection internally — but pre-compile everything at startup so your hot paths never touch the reflection API.

**"Source generators replace reflection entirely"** Source generators generate code at compile time that would otherwise require runtime reflection. For many frameworks (serialisation, DI, mapping), they're strictly better. But they require the type structure to be known at compile time. A plugin system that loads assemblies at runtime, or a general-purpose scripting host that accepts arbitrary types, still needs runtime reflection. The two tools solve different problems.

**"`GetMethod("Foo")` is always safe to call"** `GetMethod` returns null when the method isn't found, throws `AmbiguousMatchException` when multiple overloads match without parameter types specified, and returns a method with the wrong visibility if you don't pass `BindingFlags`. Always null-check the result, always provide parameter types for overloaded methods, and always be explicit about `BindingFlags.Public | BindingFlags.Instance` vs `BindingFlags.NonPublic | BindingFlags.Static`.

## Gotchas

- **`GetMethod` with an ambiguous name throws `AmbiguousMatchException`.** If a method is overloaded, pass the exact parameter type array: `GetMethod("Add", new[] { typeof(int), typeof(int) })`. Without it, the runtime finds multiple matching methods and throws.
    
- **`MethodInfo.Invoke` wraps exceptions in `TargetInvocationException`.** If the reflected method throws, the exception is caught and re-thrown wrapped in `TargetInvocationException`. Your catch blocks must unwrap `.InnerException` — or use `ExceptionDispatchInfo.Capture(ex.InnerException!).Throw()` to preserve the original stack trace.
    
- **`Activator.CreateInstance` requires a public parameterless constructor.** Types with only DI constructor parameters can't be created this way. Use an overload that accepts constructor arguments, or build the type through the DI container. Forgetting this produces a misleading `MissingMethodException`.
    
- **Reflection doesn't see private members by default.** `GetProperty("_field")` returns null for private fields. Pass `BindingFlags.NonPublic | BindingFlags.Instance` to access private and protected members. This also means reflection-based code silently skips private backing fields when mapping properties — which can produce incomplete copies.
    
- **Trimming (AOT/Native AOT) removes unused members from the binary.** Reflection depends on metadata that the linker can delete when publishing with `PublishTrimmed=true` or Native AOT. Any type or member accessed only via reflection must be annotated with `[DynamicallyAccessedMembers]` or preserved via `TrimmerRootDescriptor`. Without this, the code compiles and runs fine in development but crashes at runtime in a trimmed publish.
    
- **`GetCustomAttributes` re-creates attribute instances on every call.** Attributes are not singletons — calling `GetCustomAttributes<RequiredAttribute>()` twice creates two separate instances. Cache the results if you call them repeatedly, especially in hot paths like per-request validation.

## Interview Angle

**What they're really testing:** Whether you understand the cost model and know when to cache or compile reflection discoveries — not just that you've used `GetType()` before.

**Common question forms:**

- "How does dependency injection work under the hood?"
- "When would you use reflection and what are the risks?"
- "How would you make reflection calls faster?"
- "How does EF Core or a serialiser know the structure of your types at runtime?"

**The depth signal:** A junior answers "you can inspect types and call methods at runtime using `GetType()` and `MethodInfo.Invoke()`." A senior gives the full cost model — discovery is moderate, `Invoke` is expensive due to boxing and security checks — and immediately describes the mitigation: reflect and compile to typed delegates at startup, cache them in a `ConcurrentDictionary`, invoke at near-zero cost in the hot path. They mention `Expression.Lambda` and `Delegate.CreateDelegate` as the two compilation tools. They also flag trimming compatibility and know that source generators eliminate the problem entirely for cases where types are known at compile time.

**Follow-up questions to expect:**

- "If you're calling a method via reflection 10 million times per second, what do you do?"
- "What is `UnsafeAccessor` and when would you reach for it over full reflection?"
- "What is `TargetInvocationException` and how do you handle it correctly?"
- "Why does reflection break in Native AOT?"
