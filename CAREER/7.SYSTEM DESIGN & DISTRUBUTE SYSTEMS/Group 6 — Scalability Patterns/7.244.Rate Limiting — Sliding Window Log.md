Perfect accuracy (no boundary spike). Stores timestamps in a sliding window. Memory = O(window × rate) per client. Use for low-rate precise limits only.

```csharp
// In-memory
public bool TryConsume() {
    var cutoff = DateTime.UtcNow - _window;
    while (_timestamps.TryPeek(out var t) && t < cutoff) _timestamps.TryDequeue(out _);
    if (_timestamps.Count >= _limit) return false;
    _timestamps.Enqueue(DateTime.UtcNow);
    return true;
}

// Redis (Lua atomic)
// ZREMRANGEBYSCORE key 0 {now-window}
// ZCARD key → if >= limit, reject
// ZADD key {now} {now}
```

**ADR:** Login rate limiting — 5 attempts/min/IP. ✅ Perfect accuracy; ✅ Low memory (5 entries/IP); ❌ Not for high-rate.
