## 43.01 — Polly V8 Resilience Pipelines

**The complete resilience pipeline for external service calls**
```csharp
// NuGet: Microsoft.Extensions.Http.Resilience (wraps Polly V8)
builder.Services.AddHttpClient<IExternalApiClient, ExternalApiHttpClient>()
    .AddResilienceHandler("external-api-pipeline", pipeline =>
    {
        // Order matters: outermost = first applied

        // 1. Total request timeout (including retries)
        pipeline.AddTimeout(new HttpTimeoutStrategyOptions
        {
            Timeout = TimeSpan.FromSeconds(30)
        });
        
        // 2. Retry with exponential backoff and jitter
        pipeline.AddRetry(new HttpRetryStrategyOptions
        {
            MaxRetryAttempts = 3,
            Delay = TimeSpan.FromSeconds(1),
            BackoffType = DelayBackoffType.Exponential,
            UseJitter = true,  // Randomize delay to spread load
            ShouldHandle = new PredicateBuilder<HttpResponseMessage>()
                .Handle<HttpRequestException>()
                .HandleResult(r => r.StatusCode is
                    HttpStatusCode.TooManyRequests or
                    HttpStatusCode.InternalServerError or
                    HttpStatusCode.ServiceUnavailable),
            OnRetry = args =>
            {
                logger.LogWarning(
                    "Retry {Attempt} after {Delay}ms due to {Outcome}",
                    args.AttemptNumber,
                    args.RetryDelay.TotalMilliseconds,
                    args.Outcome.Exception?.Message ?? args.Outcome.Result?.StatusCode.ToString());
                return default;
            }
        });
        
        // 3. Circuit breaker
        pipeline.AddCircuitBreaker(new HttpCircuitBreakerStrategyOptions
        {
            SamplingDuration = TimeSpan.FromSeconds(60),
            FailureRatio = 0.5,
            MinimumThroughput = 10,
            BreakDuration = TimeSpan.FromSeconds(30),
            OnOpened = args =>
            {
                logger.LogError("Circuit opened for {Duration}", args.BreakDuration);
                return default;
            },
            OnClosed = args =>
            {
                logger.LogInformation("Circuit closed — service recovered");
                return default;
            }
        });
        
        // 4. Per-attempt timeout
        pipeline.AddTimeout(new HttpTimeoutStrategyOptions
        {
            Timeout = TimeSpan.FromSeconds(5)
        });
    });
```

**Hedging (parallel requests for latency-sensitive paths)**
```csharp
// Send duplicate requests to multiple instances, use first response
// Useful when you have redundant backends and tail latency matters

pipeline.AddHedging(new HedgingStrategyOptions<HttpResponseMessage>
{
    MaxHedgedAttempts = 2,       // Try up to 2 extra parallel requests
    Delay = TimeSpan.FromMilliseconds(100),  // Wait 100ms before hedging
    ShouldHandle = new PredicateBuilder<HttpResponseMessage>()
        .Handle<TimeoutRejectedException>()
        .HandleResult(r => r.StatusCode >= HttpStatusCode.InternalServerError)
});
```

## 43.02 — Health Check Best Practices

**What a good health check does**
```csharp
// A health check should test what it claims to test
public class SqlServerHealthCheck : IHealthCheck
{
    private readonly string _connectionString;
    
    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context, CancellationToken ct)
    {
        using var connection = new SqlConnection(_connectionString);
        try
        {
            await connection.OpenAsync(ct);  // Test actual connection
            
            // Test actual query execution, not just connectivity
            using var command = new SqlCommand("SELECT 1", connection);
            var result = await command.ExecuteScalarAsync(ct);
            
            return result is 1
                ? HealthCheckResult.Healthy("Database is responsive")
                : HealthCheckResult.Degraded("Database returned unexpected result");
        }
        catch (SqlException ex) when (ex.Number == -2)  // Timeout
        {
            return HealthCheckResult.Degraded("Database is slow", ex);
        }
        catch (Exception ex)
        {
            return HealthCheckResult.Unhealthy("Database is unreachable", ex);
        }
    }
}

// Redis health check with timeout
public class RedisHealthCheck : IHealthCheck
{
    private readonly IConnectionMultiplexer _redis;
    
    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context, CancellationToken ct)
    {
        try
        {
            using var cts = CancellationTokenSource.CreateLinkedTokenSource(ct);
            cts.CancelAfter(TimeSpan.FromSeconds(2));  // 2-second timeout
            
            var db = _redis.GetDatabase();
            var pingResult = await db.PingAsync();
            
            return pingResult.TotalMilliseconds < 100
                ? HealthCheckResult.Healthy($"Redis responded in {pingResult.TotalMilliseconds}ms")
                : HealthCheckResult.Degraded($"Redis is slow: {pingResult.TotalMilliseconds}ms");
        }
        catch (Exception ex)
        {
            return HealthCheckResult.Unhealthy("Redis is unreachable", ex);
        }
    }
}
```

**Liveness vs. Readiness vs. Startup probes**
```csharp
// Liveness: Is the process alive? Should we restart it?
// Only return Unhealthy if the process is truly broken (deadlock, OOM, etc.)
// Do NOT check dependencies — if Redis is down, don't restart the app
app.MapHealthChecks("/health/live", new HealthCheckOptions
{
    Predicate = _ => false  // No checks — if we can respond, we're alive
});

// Readiness: Can we serve traffic? Should load balancer route to us?
// Check all dependencies — if DB is down, take us out of rotation
app.MapHealthChecks("/health/ready", new HealthCheckOptions
{
    Predicate = check => check.Tags.Contains("ready")
});

// Startup: Have we finished initializing?
// Used for slow-starting apps to prevent premature liveness checks
app.MapHealthChecks("/health/startup", new HealthCheckOptions
{
    Predicate = check => check.Tags.Contains("startup")
});
```
