## 28.01 — GitHub Profile

**What makes a strong profile**
- Pinned repositories: 6 that represent your best work and widest range
- Consistent green contribution graph: not vanity — signals sustained activity
- Profile README: who you are, what you build, what you're learning

**What makes a strong repository README**
```markdown
# InterVision — AI-Powered Interview Training Platform

## What it does
InterVision helps software engineers practice technical interviews using AI-powered
feedback on mock interview sessions.

## Architecture
[Diagram here]

- **Backend**: ASP.NET Core 8, Clean Architecture
- **AI**: Groq Llama3 + Gemini for session analysis via Hangfire async pipeline
- **Storage**: SQL Server (sessions) + Cloudflare R2 (recordings)
- **Auth**: JWT + refresh token rotation

## Key Technical Decisions

### Why Hangfire for AI processing?
LLM API calls take 3-8 seconds. Running this synchronously would make the API
unresponsive. Hangfire allows the response to return immediately while AI processing
runs in the background. Users see results when ready.

### Why Clean Architecture?
The team wanted to swap the AI provider without touching business logic.
Clean Architecture's dependency rule meant we could replace Groq with Gemini
by only changing the Infrastructure layer.

## Running locally
[Clear instructions with Docker Compose]

## What I would do differently
If starting over, I would implement the Outbox pattern for reliable domain event
publishing rather than the current direct Hangfire enqueuing.
```

## 28.02 — Technical Writing

**The story-driven format (most effective)**
```
Title: How I Debugged a CancellationToken Deadlock in Production

Hook: "At 2:47 AM, all API requests started hanging. Logs showed nothing.
The health check was green. 847 users were affected."

The investigation: Step by step, what I tried, what didn't work, the hypothesis.

The discovery: The smoking gun.

The explanation: Why this happens, the underlying mechanism.

The fix: What we did.

The lesson: What to watch for, how to prevent it.
```

## 28.03 — Salary and Negotiation

**Research the market first**
- levels.fyi: Tech companies specifically, with level breakdowns
- Glassdoor: Broad market, less precise
- LinkedIn Salary: Good for local markets
- Blind: Anonymous, tech-focused community data

**Principles**
1. Never state a number first. "What is the budgeted range for this role?"
2. Evaluate total compensation: base + equity + bonus + benefits + flexibility
3. Always negotiate — offers are rarely final
4. Use data: "Based on my research and similar roles at [company], I was expecting..."
5. Be specific: "I was hoping for X" is stronger than "can you do better?"
6. If you have competing offers, state them factually (not as threats)
