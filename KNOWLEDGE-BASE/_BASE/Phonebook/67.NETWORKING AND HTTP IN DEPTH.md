## 67.01 — HTTP/2 and HTTP/3

**HTTP/1.1 Problems**
- One request per connection at a time (head-of-line blocking)
- Many connections per domain (typically 6)
- Plain text headers (verbose, redundant)

**HTTP/2 Solutions**
- Multiplexing: multiple requests on one TCP connection, no blocking
- Header compression (HPACK): repeated headers compressed
- Server push: server can proactively send resources
- Binary protocol: more efficient than text

**HTTP/3 (QUIC)**
- Uses QUIC (UDP-based transport) instead of TCP
- No TCP head-of-line blocking (each stream independent)
- 0-RTT connection resumption (no handshake for returning clients)
- Better on lossy/mobile networks

**In ASP.NET Core**
```csharp
// HTTP/2 enabled by default over HTTPS with Kestrel
// HTTP/3 requires explicit configuration:
builder.WebHost.ConfigureKestrel(options =>
{
    options.ListenAnyIP(443, listenOptions =>
    {
        listenOptions.Protocols = HttpProtocols.Http1AndHttp2AndHttp3;
        listenOptions.UseHttps();
    });
});

// Check negotiated protocol
app.MapGet("/protocol", (HttpContext ctx) =>
    Results.Ok(ctx.Request.Protocol));  // "HTTP/1.1", "HTTP/2", "HTTP/3"
```

## 67.02 — WebSockets

**What it is**
Full-duplex communication over a single TCP connection. Starts as HTTP upgrade request, then becomes a persistent bi-directional channel.

**Server implementation**
```csharp
// In minimal API
app.MapGet("/ws", async (HttpContext ctx) =>
{
    if (!ctx.WebSockets.IsWebSocketRequest)
    {
        ctx.Response.StatusCode = StatusCodes.Status400BadRequest;
        return;
    }
    
    using var ws = await ctx.WebSockets.AcceptWebSocketAsync();
    
    var buffer = new byte[4096];
    var result = await ws.ReceiveAsync(new ArraySegment<byte>(buffer), CancellationToken.None);
    
    while (!result.CloseStatus.HasValue)
    {
        // Echo the message back
        await ws.SendAsync(
            new ArraySegment<byte>(buffer, 0, result.Count),
            result.MessageType,
            result.EndOfMessage,
            CancellationToken.None);
        
        result = await ws.ReceiveAsync(new ArraySegment<byte>(buffer), CancellationToken.None);
    }
    
    await ws.CloseAsync(result.CloseStatus.Value, result.CloseStatusDescription, CancellationToken.None);
});

app.UseWebSockets();
```

**SignalR — ASP.NET Core's WebSocket wrapper**
```csharp
// Automatic: handles WebSocket, Server-Sent Events, Long Polling fallbacks
// Groups, connection management, automatic reconnect

// Hub
public class ChatHub : Hub
{
    public async Task SendMessage(string user, string message)
    {
        // Broadcast to all connected clients
        await Clients.All.SendAsync("ReceiveMessage", user, message);
    }
    
    public async Task JoinRoom(string roomName)
    {
        await Groups.AddToGroupAsync(Context.ConnectionId, roomName);
        await Clients.Group(roomName).SendAsync("UserJoined", Context.User?.Identity?.Name);
    }
    
    public async Task SendToRoom(string roomName, string message)
    {
        await Clients.Group(roomName).SendAsync("ReceiveMessage",
            Context.User?.Identity?.Name, message);
    }
    
    public override async Task OnDisconnectedAsync(Exception? exception)
    {
        // Handle cleanup when client disconnects
        await base.OnDisconnectedAsync(exception);
    }
}

// Registration
builder.Services.AddSignalR(options =>
{
    options.EnableDetailedErrors = builder.Environment.IsDevelopment();
    options.KeepAliveInterval = TimeSpan.FromSeconds(15);
    options.ClientTimeoutInterval = TimeSpan.FromSeconds(30);
});

app.MapHub<ChatHub>("/hubs/chat");

// Inject and use hub outside of hub class (e.g., from a service)
public class NotificationService
{
    private readonly IHubContext<ChatHub> _hub;
    
    public async Task NotifyUserAsync(string userId, string message)
    {
        await _hub.Clients.User(userId).SendAsync("Notification", message);
    }
}
```

## 67.03 — Server-Sent Events (SSE)

**What it is**
One-way stream from server to client over HTTP. Client never sends data after initial request. Simpler than WebSocket for push scenarios (notifications, live feeds).

```csharp
// Server
app.MapGet("/api/events", async (HttpContext ctx, CancellationToken ct) =>
{
    ctx.Response.ContentType = "text/event-stream";
    ctx.Response.Headers.CacheControl = "no-cache";
    ctx.Response.Headers.Connection = "keep-alive";
    
    var writer = ctx.Response.BodyWriter;
    
    while (!ct.IsCancellationRequested)
    {
        var eventData = await GetNextEventAsync(ct);
        
        // SSE format: "data: {json}\n\n"
        var message = $"data: {JsonSerializer.Serialize(eventData)}\n\n";
        var bytes = Encoding.UTF8.GetBytes(message);
        
        await writer.WriteAsync(bytes, ct);
        await writer.FlushAsync(ct);
        
        await Task.Delay(TimeSpan.FromSeconds(1), ct);
    }
});

// Or using the new SSE support in .NET 8:
app.MapGet("/api/live-orders", (CancellationToken ct) =>
{
    async IAsyncEnumerable<OrderEvent> GetEvents([EnumeratorCancellation] CancellationToken token = default)
    {
        while (!token.IsCancellationRequested)
        {
            yield return new OrderEvent { OrderId = 42, Status = "Shipped" };
            await Task.Delay(1000, token);
        }
    }
    
    return TypedResults.ServerSentEvents(GetEvents(ct), eventType: "order-update");
});
```
