## 36.01 — The Twelve-Factor App

**I. Codebase** — One codebase, many deploys. Never different codebases for dev/staging/prod.

**II. Dependencies** — Explicitly declare all dependencies (`dotnet restore`, NuGet). No system-wide packages assumed.

**III. Config** — Store config in environment variables. Not in code. Not in config files committed to source control.
```csharp
// Good: reads from environment
builder.Configuration.AddEnvironmentVariables();
var connectionString = Environment.GetEnvironmentVariable("DATABASE_URL");

// Bad: hardcoded or committed secrets
var connectionString = "Server=prod.db;Password=secret123";
```

**IV. Backing services** — Treat databases, caches, message queues as attached resources. Swap them via config change, not code change.

**V. Build, release, run** — Strict separation. Build: compile code. Release: combine build with config. Run: execute.

**VI. Processes** — Execute as stateless processes. No sticky sessions. Session state in Redis, not in-process.

**VII. Port binding** — Export services by binding to a port. The app is self-contained (Kestrel), not dependent on an external web server.

**VIII. Concurrency** — Scale out via the process model. Add more instances, not bigger instances.

**IX. Disposability** — Fast startup, graceful shutdown. Handle SIGTERM properly.
```csharp
// Graceful shutdown in ASP.NET Core
app.Lifetime.ApplicationStopping.Register(() =>
{
    // Finish in-flight requests (Kestrel handles this automatically)
    // Close message consumers
    // Flush logs
    _logger.LogInformation("Application shutting down");
});
```

**X. Dev/prod parity** — Keep dev, staging, prod as similar as possible. Docker Compose makes this much easier.

**XI. Logs** — Treat logs as event streams. Don't manage log files. Write to stdout. Infrastructure routes logs to the log management system.
```csharp
// Serilog writing to console (stdout) — infrastructure routes it
.WriteTo.Console(new JsonFormatter())  // Structured JSON to stdout
```

**XII. Admin processes** — Run admin tasks as one-off processes.
```csharp
// Database migration as one-off task on deploy
if (args.Contains("--migrate"))
{
    using var scope = app.Services.CreateScope();
    var db = scope.ServiceProvider.GetRequiredService<AppDbContext>();
    await db.Database.MigrateAsync();
    return;
}
```

## 36.02 — Service Mesh (Istio / Linkerd)

**What it is**
A dedicated infrastructure layer for service-to-service communication in microservices. Handles: TLS between services, retries, circuit breaking, observability — without changing application code.

**How it works**
Every pod gets a sidecar proxy (Envoy). All traffic goes through the proxy. The proxy handles:
- mTLS (mutual TLS between all services)
- Retries and timeouts
- Circuit breaking
- Load balancing
- Metrics collection
- Distributed tracing

**In Kubernetes**
```yaml
# Istio sidecar injection — one label enables it for the namespace
apiVersion: v1
kind: Namespace
metadata:
  name: myapp
  labels:
    istio-injection: enabled  # All pods in this namespace get Envoy sidecar

# Traffic management — canary deployment
apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: orders-service
spec:
  http:
  - route:
    - destination:
        host: orders-service
        subset: v1
      weight: 90  # 90% to v1
    - destination:
        host: orders-service
        subset: v2
      weight: 10  # 10% to new version (canary)
```

## 36.03 — Dapr (Distributed Application Runtime)

**What it is**
A portable runtime for building distributed applications. Abstracts: state stores, pub/sub, service invocation, secrets, bindings — through a consistent API regardless of the underlying technology.

**Setup with ASP.NET Core**
```csharp
builder.Services.AddDaprClient();

// State management (Redis, Cosmos DB, SQL Server — configured separately)
public class CartService
{
    private readonly DaprClient _dapr;
    
    public async Task AddItemAsync(string userId, CartItem item, CancellationToken ct)
    {
        var cart = await _dapr.GetStateAsync<Cart>("statestore", userId, ct: ct)
            ?? new Cart();
        
        cart.AddItem(item);
        await _dapr.SaveStateAsync("statestore", userId, cart, ct: ct);
    }
}

// Pub/Sub (Azure Service Bus, RabbitMQ, Kafka — configured in Dapr)
public class OrderService
{
    private readonly DaprClient _dapr;
    
    public async Task CreateOrderAsync(Order order, CancellationToken ct)
    {
        await _repo.SaveAsync(order, ct);
        await _dapr.PublishEventAsync("pubsub", "order-created",
            new OrderCreatedEvent(order.Id), ct);
    }
}

// Subscribe to events
app.MapPost("/order-created", [Topic("pubsub", "order-created")]
    async (OrderCreatedEvent evt, IInventoryService inventory) =>
    {
        await inventory.ReserveAsync(evt.OrderId);
        return Results.Ok();
    });

// Service invocation
var product = await _dapr.InvokeMethodAsync<Product>(
    HttpMethod.Get,
    "product-service",  // Service name in Kubernetes
    $"api/products/{productId}",
    ct);
```
