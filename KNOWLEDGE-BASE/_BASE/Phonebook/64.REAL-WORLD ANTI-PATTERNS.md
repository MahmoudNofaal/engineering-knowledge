# CATEGORY 64 — REAL-WORLD ANTI-PATTERNS

---

## 64.01 — Common Mistakes and Their Fixes

**Anti-pattern 1: God class / service**
```csharp
// WRONG: One service doing everything
public class OrderService
{
    // 500 lines, 30 methods
    // Creates orders, sends emails, processes payments,
    // manages inventory, sends notifications, generates reports...
}

// RIGHT: Single Responsibility
public class OrderCreationService    { /* only creates orders */ }
public class OrderPaymentService     { /* only processes payments */ }
public class OrderEmailService       { /* only sends emails */ }
public class OrderInventoryService   { /* only manages inventory */ }
// Coordinated by: handlers in CQRS pattern
```

**Anti-pattern 2: Primitive obsession**
```csharp
// WRONG: Raw primitives everywhere
public void CreateOrder(int customerId, string email, decimal total, string currency)
{
    // What is customerId? An int? Which int? User? Customer? Anything?
    // What are valid currencies? "USD"? "usd"? "US Dollar"?
}

// RIGHT: Value objects
public void CreateOrder(CustomerId customerId, EmailAddress email, Money total)
{
    // CustomerId guarantees it's a valid customer ID
    // EmailAddress guarantees it's a valid email format
    // Money guarantees currency is valid and amount is non-negative
}

public record CustomerId(int Value)
{
    public CustomerId { if (Value <= 0) throw new ArgumentException("Invalid customer ID"); }
}

public record Money(decimal Amount, string Currency)
{
    public Money
    {
        if (Amount < 0) throw new ArgumentException("Amount cannot be negative");
        if (!new[] {"USD", "EUR", "GBP"}.Contains(Currency))
            throw new ArgumentException($"Invalid currency: {Currency}");
    }
}
```

**Anti-pattern 3: Shotgun surgery**
```csharp
// WRONG: One change requires modifying many files
// Example: Adding "LastUpdatedBy" to logging
// Must change: OrderService, ProductService, CustomerService,
//              all repositories, all controllers, all DTOs...

// RIGHT: Cross-cutting concerns via interceptors/middleware/decorators
// Add audit logging via a SaveChanges interceptor in EF Core
// One change in one place, applies everywhere
public class AuditInterceptor : SaveChangesInterceptor
{
    public override InterceptionResult<int> SavingChanges(DbContextEventData eventData, ...)
    {
        // Applied to ALL SaveChanges calls everywhere
        SetAuditFields(eventData.Context);
        return base.SavingChanges(eventData, result);
    }
}
```

**Anti-pattern 4: Magic numbers**
```csharp
// WRONG
if (order.Items.Count > 50)
    throw new InvalidOperationException("Too many items");

await Task.Delay(5000);

if (response.StatusCode == 429)
    await RetryAsync();

// RIGHT
private const int MaxOrderItems = 50;
private static readonly TimeSpan RetryDelay = TimeSpan.FromSeconds(5);
private const int TooManyRequestsStatusCode = 429;

if (order.Items.Count > MaxOrderItems)
    throw new InvalidOperationException($"Order cannot exceed {MaxOrderItems} items");

await Task.Delay(RetryDelay);

if ((int)response.StatusCode == TooManyRequestsStatusCode)
    await RetryAsync();
```

**Anti-pattern 5: Returning null from collections**
```csharp
// WRONG: caller must null-check before iterating
public List<Order>? GetOrders(int customerId)
{
    if (customerId <= 0) return null;
    return _db.Orders.Where(o => o.CustomerId == customerId).ToList();
}

// Caller:
var orders = _service.GetOrders(customerId);
if (orders != null)  // Easy to forget
    foreach (var order in orders) { }

// RIGHT: Return empty collection
public List<Order> GetOrders(int customerId)
{
    if (customerId <= 0) return new List<Order>();  // Or Enumerable.Empty<Order>().ToList()
    return _db.Orders.Where(o => o.CustomerId == customerId).ToList();
}

// Caller: always safe
foreach (var order in _service.GetOrders(customerId)) { }
```

**Anti-pattern 6: Boolean parameters**
```csharp
// WRONG: What does false mean here?
CreateUser(name, email, true, false, true);

// RIGHT: Enum or multiple named methods
public enum UserCreationOptions
{
    Standard,
    WithEmailVerification,
    WithWelcomeEmail,
    AsAdmin
}

CreateUser(name, email, UserCreationOptions.WithEmailVerification);
// Or: multiple focused methods
CreateStandardUser(name, email);
CreateAdminUser(name, email);
```

**Anti-pattern 7: Feature envy**
```csharp
// WRONG: OrderValidator knows too much about Order internals
public class OrderValidator
{
    public bool IsValid(Order order)
    {
        // Accessing order's items' product's category's discount rules
        return order.Items.All(item =>
            item.Product.Category.DiscountRules.Min.Quantity <= item.Quantity &&
            item.Quantity <= item.Product.Category.DiscountRules.Max.Quantity);
    }
}

// RIGHT: Move logic to where the data lives
public class Order
{
    public bool IsValid()
    {
        return Items.All(item => item.IsValidQuantity());  // Delegate to item
    }
}

public class OrderItem
{
    public bool IsValidQuantity()
    {
        return Product.Category.IsQuantityValid(Quantity);  // Delegate to category
    }
}
```
