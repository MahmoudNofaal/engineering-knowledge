## 74.01 — Starting a .NET Project From Scratch

**Step 1: Solution structure**
```bash
dotnet new sln -n YourProject
mkdir src tests

cd src
dotnet new classlib -n YourProject.Domain --framework net8.0
dotnet new classlib -n YourProject.Application --framework net8.0
dotnet new classlib -n YourProject.Infrastructure --framework net8.0
dotnet new webapi -n YourProject.Api --framework net8.0

cd ../tests
dotnet new xunit -n YourProject.UnitTests --framework net8.0
dotnet new xunit -n YourProject.IntegrationTests --framework net8.0

cd ..
# Add all projects to solution
dotnet sln add src/YourProject.Domain
dotnet sln add src/YourProject.Application
dotnet sln add src/YourProject.Infrastructure
dotnet sln add src/YourProject.Api
dotnet sln add tests/YourProject.UnitTests
dotnet sln add tests/YourProject.IntegrationTests

# Set up project references
dotnet add src/YourProject.Application reference src/YourProject.Domain
dotnet add src/YourProject.Infrastructure reference src/YourProject.Application
dotnet add src/YourProject.Api reference src/YourProject.Application
dotnet add src/YourProject.Api reference src/YourProject.Infrastructure
dotnet add tests/YourProject.UnitTests reference src/YourProject.Application
dotnet add tests/YourProject.IntegrationTests reference src/YourProject.Api
```

**Step 2: Core NuGet packages per project**
```bash
# Domain — no external dependencies
# (Only: pure C# — no NuGet packages)

# Application
dotnet add src/YourProject.Application package MediatR
dotnet add src/YourProject.Application package FluentValidation
dotnet add src/YourProject.Application package Microsoft.Extensions.Logging.Abstractions

# Infrastructure
dotnet add src/YourProject.Infrastructure package Microsoft.EntityFrameworkCore.SqlServer
dotnet add src/YourProject.Infrastructure package Microsoft.EntityFrameworkCore.Tools
dotnet add src/YourProject.Infrastructure package StackExchange.Redis
dotnet add src/YourProject.Infrastructure package Azure.Messaging.ServiceBus

# API
dotnet add src/YourProject.Api package MediatR.Extensions.Microsoft.DependencyInjection
dotnet add src/YourProject.Api package FluentValidation.AspNetCore
dotnet add src/YourProject.Api package Serilog.AspNetCore
dotnet add src/YourProject.Api package Swashbuckle.AspNetCore

# Unit Tests
dotnet add tests/YourProject.UnitTests package xunit
dotnet add tests/YourProject.UnitTests package FluentAssertions
dotnet add tests/YourProject.UnitTests package NSubstitute
dotnet add tests/YourProject.UnitTests package Bogus

# Integration Tests
dotnet add tests/YourProject.IntegrationTests package Microsoft.AspNetCore.Mvc.Testing
dotnet add tests/YourProject.IntegrationTests package Testcontainers.MsSql
```

**Step 7: Initial Program.cs**
```csharp
var builder = WebApplication.CreateBuilder(args);

// Serilog
builder.Host.UseSerilog((ctx, lc) => lc
    .ReadFrom.Configuration(ctx.Configuration)
    .Enrich.FromLogContext()
    .WriteTo.Console());

// Application layer
builder.Services.AddMediatR(cfg => cfg.RegisterServicesFromAssembly(ApplicationAssembly));
builder.Services.AddValidatorsFromAssembly(ApplicationAssembly);

// Infrastructure
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("Default")));

// API
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();
builder.Services.AddProblemDetails();
builder.Services.AddExceptionHandler<GlobalExceptionHandler>();

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseExceptionHandler();
app.UseHttpsRedirection();

// Map endpoints
app.MapOrderEndpoints();

app.Run();

public partial class Program { }  // For WebApplicationFactory in integration tests
```
