## 73.01 — Redis Data Structures — Advanced Patterns

**HyperLogLog — approximate cardinality counting**
```bash
# Count unique visitors (approximate, uses ~12KB regardless of cardinality)
PFADD visitors:2026-06-01 user1 user2 user3 user4
PFADD visitors:2026-06-02 user2 user3 user5 user6

PFCOUNT visitors:2026-06-01    # ~4
PFMERGE visitors:total visitors:2026-06-01 visitors:2026-06-02
PFCOUNT visitors:total          # ~6 (unique across both days)
```

```csharp
await db.HyperLogLogAddAsync("visitors:today", userId);
long uniqueVisitors = await db.HyperLogLogLengthAsync("visitors:today");
```

**Bloom Filter (Redis Stack required)**
```bash
# Test set membership — no false negatives, possible false positives
BF.ADD blocked-emails "spammer@evil.com"
BF.EXISTS blocked-emails "innocent@gmail.com"   # 0 (not in set)
BF.EXISTS blocked-emails "spammer@evil.com"      # 1 (in set)
# Uses < 1 byte per element regardless of element size
```

**Geospatial commands**
```bash
GEOADD stores -73.9857 40.7484 "New York Store"
GEOADD stores -118.2437 34.0522 "LA Store"
GEOADD stores 31.2357 30.0444 "Cairo Store"

# Find stores within 100km of a location
GEOSEARCH stores FROMLONLAT 31.0 30.0 BYRADIUS 100 km ASC COUNT 5
# Returns: Cairo Store (distance from given point)

GEODIST stores "New York Store" "LA Store" km
# Returns: ~3,940 km
```

```csharp
await db.GeoAddAsync("stores", -73.9857, 40.7484, "New York Store");

var nearbyStores = await db.GeoSearchAsync(
    "stores",
    GeoSearchShape.ByRadius(-73.9857, 40.7484, 50, GeoUnit.Kilometers),
    count: 5,
    order: Order.Ascending);
```

**Redis Streams — event log**
```csharp
// Append event to stream
var messageId = await db.StreamAddAsync("orders:events", new NameValueEntry[]
{
    new("event_type", "OrderCreated"),
    new("order_id", "42"),
    new("customer_id", "7"),
    new("total", "299.99"),
    new("timestamp", DateTime.UtcNow.ToString("O"))
});
// messageId: "1749123456789-0" (timestamp-sequence)

// Consumer group — reliable processing, acknowledgment required
await db.StreamCreateConsumerGroupAsync("orders:events", "notification-service", StreamPosition.NewMessages);

var messages = await db.StreamReadGroupAsync(
    "orders:events",
    "notification-service",
    "consumer-1",
    count: 10);

foreach (var message in messages)
{
    await ProcessEventAsync(message);
    await db.StreamAcknowledgeAsync("orders:events", "notification-service", message.Id);
}

// Recover pending (unacknowledged) messages
var pending = await db.StreamPendingMessagesAsync(
    "orders:events", "notification-service",
    count: 10, minId: StreamPosition.Beginning);
```

## 73.02 — Redis Cluster Architecture

**What is Redis Cluster?**
- Automatically shards data across multiple nodes
- Each node handles a subset of keyspace
- Built-in replication and failover

**16,384 hash slots**
```
Redis Cluster: 16,384 slots distributed across nodes
Node 1: slots 0 – 5,460
Node 2: slots 5,461 – 10,922
Node 3: slots 10,923 – 16,383

Key → CRC16(key) % 16384 → which slot → which node

// Hash tags force same slot for related keys:
{order:42}:data  → CRC16("order:42") → same slot
{order:42}:items → CRC16("order:42") → same slot (hash computed on {})
// Now both keys are on same node → MGET works without CROSSSLOT error
```

**StackExchange.Redis cluster configuration**
```csharp
var redis = await ConnectionMultiplexer.ConnectAsync(new ConfigurationOptions
{
    EndPoints = { "node1:6379", "node2:6379", "node3:6379" },
    // StackExchange.Redis discovers all nodes automatically via CLUSTER NODES
    AllowAdmin = false,
    ConnectTimeout = 5000,
    SyncTimeout = 5000,
    ReconnectRetryPolicy = new ExponentialRetry(
        deltaBackoffMilliseconds: 5000,
        maxDeltaBackoffMilliseconds: 20000)
});
```
