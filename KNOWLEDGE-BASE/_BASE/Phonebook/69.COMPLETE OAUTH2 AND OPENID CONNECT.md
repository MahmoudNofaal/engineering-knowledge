## 69.01 — OAuth 2.0 In Depth

**The four parties**
1. **Resource Owner**: The user who owns the data
2. **Client**: The application wanting access (your app)
3. **Authorization Server**: Issues tokens (Google, Azure AD, Auth0, Keycloak)
4. **Resource Server**: The API being accessed

**Authorization Code + PKCE Flow (most common)**
```
User → Client (your web/mobile app):
  "I want to log in with Google"

Client → User's Browser:
  Redirect to: https://accounts.google.com/oauth/authorize
    ?client_id=your_client_id
    &redirect_uri=https://yourapp.com/callback
    &response_type=code
    &scope=openid profile email
    &state=random_csrf_token        ← Prevent CSRF
    &code_challenge=sha256_of_verifier  ← PKCE
    &code_challenge_method=S256

User → Authorization Server:
  Login, see consent screen, approve

Authorization Server → Client (via redirect):
  https://yourapp.com/callback?code=AUTH_CODE&state=csrf_token

Client → Authorization Server (server-to-server, not browser):
  POST https://accounts.google.com/oauth/token
    grant_type=authorization_code
    code=AUTH_CODE
    redirect_uri=https://yourapp.com/callback
    client_id=your_client_id
    code_verifier=original_verifier  ← PKCE proof

Authorization Server → Client:
  {
    "access_token": "eyJ...",
    "token_type": "Bearer",
    "expires_in": 3600,
    "refresh_token": "...",
    "id_token": "eyJ..."  ← OpenID Connect: contains user info
  }
```

**Why PKCE?**
Authorization code could be intercepted (in mobile apps, the redirect URI could be hijacked). PKCE (Proof Key for Code Exchange) requires proof that the same client that generated the code challenge is exchanging the code.

```
1. Client generates: random_verifier (secret, only in memory)
2. Client sends: hash of verifier (code_challenge) to auth server
3. Auth server stores the hash with the code
4. Client exchanges code + verifier → auth server verifies hash(verifier) == stored hash
5. Intercepted code is useless without the original verifier
```

**Client Credentials Flow (service-to-service)**
```
POST https://auth.example.com/oauth/token
  grant_type=client_credentials
  client_id=your_service_client_id
  client_secret=your_service_client_secret
  scope=api.orders.read api.orders.write

Response: { "access_token": "...", "expires_in": 3600 }

Use: M2M (microservice A calling microservice B)
```

**In ASP.NET Core — validating incoming JWT**
```csharp
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        // For OAuth2/OIDC tokens from external identity provider:
        options.Authority = "https://your-identity-provider.com";  // OIDC discovery
        options.Audience = "your-api";  // aud claim in token
        
        // Token validation
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer = true,
            ValidateAudience = true,
            ValidateLifetime = true,
            ValidateIssuerSigningKey = true,
            ClockSkew = TimeSpan.Zero
        };
        
        // For development: allow HTTP
        options.RequireHttpsMetadata = !builder.Environment.IsDevelopment();
    });

// The discovery endpoint (https://authority/.well-known/openid-configuration)
// automatically fetches signing keys — no manual key configuration needed
```

**Inspecting tokens in .NET**
```csharp
// Access claims in handlers/controllers
var userId = User.FindFirstValue(ClaimTypes.NameIdentifier);
var email = User.FindFirstValue(ClaimTypes.Email);
var roles = User.FindAll(ClaimTypes.Role).Select(c => c.Value).ToList();

// Custom claims from token
var tenantId = User.FindFirstValue("tenant_id");
var subscriptionLevel = User.FindFirstValue("subscription");

// Check if token is expired
var expiryUnix = long.Parse(User.FindFirstValue(JwtRegisteredClaimNames.Exp)!);
var expiry = DateTimeOffset.FromUnixTimeSeconds(expiryUnix);
var isExpired = expiry < DateTimeOffset.UtcNow;
```

## 69.02 — Implementing Your Own Identity Provider

**When you build your own auth (not use external IdP)**
```csharp
// Complete auth service
public class AuthService
{
    private readonly IUserRepository _users;
    private readonly IRefreshTokenRepository _tokens;
    private readonly IConfiguration _config;
    private readonly IPasswordHasher<User> _hasher;
    
    public async Task<AuthResult> LoginAsync(LoginRequest request, CancellationToken ct)
    {
        var user = await _users.GetByEmailAsync(request.Email, ct);
        if (user is null) return AuthResult.Failure("Invalid credentials");
        
        var result = _hasher.VerifyHashedPassword(user, user.PasswordHash, request.Password);
        if (result == PasswordVerificationResult.Failed)
        {
            await _users.RecordFailedLoginAsync(user.Id, ct);
            return AuthResult.Failure("Invalid credentials");
        }
        
        if (result == PasswordVerificationResult.SuccessRehashNeeded)
        {
            user.PasswordHash = _hasher.HashPassword(user, request.Password);
            await _users.UpdateAsync(user, ct);
        }
        
        if (!user.IsEmailVerified) return AuthResult.Failure("Email not verified");
        if (user.IsLocked) return AuthResult.Failure("Account is locked");
        
        var accessToken = GenerateAccessToken(user);
        var refreshToken = await CreateRefreshTokenAsync(user.Id, ct);
        
        await _users.RecordSuccessfulLoginAsync(user.Id, ct);
        
        return AuthResult.Success(accessToken, refreshToken.Token);
    }
    
    private string GenerateAccessToken(User user)
    {
        var key = new SymmetricSecurityKey(
            Encoding.UTF8.GetBytes(_config["Jwt:Key"]!));
        
        var claims = new List<Claim>
        {
            new(JwtRegisteredClaimNames.Sub, user.Id.ToString()),
            new(JwtRegisteredClaimNames.Email, user.Email),
            new(JwtRegisteredClaimNames.Jti, Guid.NewGuid().ToString()),  // Unique token ID
            new(JwtRegisteredClaimNames.Iat,
                DateTimeOffset.UtcNow.ToUnixTimeSeconds().ToString(),
                ClaimValueTypes.Integer64),
            new(ClaimTypes.Role, user.Role.ToString()),
            // Custom claims
            new("tenant_id", user.TenantId.ToString()),
            new("subscription", user.SubscriptionLevel)
        };
        
        var token = new JwtSecurityToken(
            issuer: _config["Jwt:Issuer"],
            audience: _config["Jwt:Audience"],
            claims: claims,
            notBefore: DateTime.UtcNow,
            expires: DateTime.UtcNow.AddMinutes(15),  // Short! Refresh handles longer sessions
            signingCredentials: new SigningCredentials(key, SecurityAlgorithms.HmacSha256));
        
        return new JwtSecurityTokenHandler().WriteToken(token);
    }
    
    private async Task<RefreshToken> CreateRefreshTokenAsync(int userId, CancellationToken ct)
    {
        // Revoke all existing tokens for this user (optional: allow multiple devices)
        await _tokens.RevokeAllForUserAsync(userId, ct);
        
        var token = new RefreshToken
        {
            Token = Convert.ToBase64String(RandomNumberGenerator.GetBytes(64)),
            UserId = userId,
            ExpiresAt = DateTime.UtcNow.AddDays(30),
            CreatedAt = DateTime.UtcNow,
            CreatedByIp = _httpContext.Connection.RemoteIpAddress?.ToString()
        };
        
        await _tokens.CreateAsync(token, ct);
        return token;
    }
}
```
