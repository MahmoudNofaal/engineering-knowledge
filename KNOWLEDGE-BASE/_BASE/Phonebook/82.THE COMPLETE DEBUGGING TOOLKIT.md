## 82.01 — Systematic Debugging

**The rubber duck method**
Explain the problem out loud to a rubber duck (or anyone). Often, articulating the problem reveals the solution. The act of explaining forces your brain to be precise.

**The binary search debug method**
```
Bug: X doesn't work
Step 1: Find the midpoint of the code path
Step 2: Log/breakpoint at the midpoint
Step 3: Is the state correct at the midpoint?
  YES → Bug is in the second half. Repeat with second half.
  NO  → Bug is in the first half. Repeat with first half.
Step 4: Repeat until the bug is found

O(log n) instead of O(n) debugging
```

**Change-based debugging**
```
"It worked on Wednesday. Now it doesn't."

1. git log --since="Wednesday" --until="now"
2. Find the commit where it broke (git bisect automates this)
3. Read the diff carefully

git bisect start
git bisect bad HEAD         # Current is bad
git bisect good abc123      # This commit was good
# Git checks out middle commit
# Test manually or run test suite
git bisect good  # or bad
# Repeat until: "<hash> is the first bad commit"
git bisect reset
```

**Hypothesis-driven debugging**
```
1. Observe symptoms exactly
2. Form ONE specific hypothesis
3. Make a prediction: "If my hypothesis is correct, then X will happen"
4. Test the prediction
5. If prediction holds → hypothesis likely correct, fix it
   If prediction fails → hypothesis wrong, form new one
6. Never change two things at once — you can't determine which fixed it
```

## 82.02 — Debugging Concurrency Issues

**Data race detection**
```csharp
// Detect shared mutable state
// WRONG: Static mutable state accessed from multiple threads
private static int _counter = 0;

void IncrementAsync()
{
    _counter++;  // Not atomic! Read-Modify-Write is three operations
    // Thread A reads 5, Thread B reads 5, both write 6, result should be 7
}

// FIX: Interlocked
private static int _counter = 0;

void IncrementSafe()
{
    Interlocked.Increment(ref _counter);  // Atomic
}

// FIX: Lock
private static readonly object _lock = new();
private static int _counter = 0;

void IncrementSafe()
{
    lock (_lock) { _counter++; }
}
```

**Deadlock detection in .NET**
```bash
# Capture dump when process appears hung
dotnet dump collect --process-id <PID>
dotnet dump analyze <dump_file>

# In analysis:
> threads            # All managed threads
> clrstack           # Current thread stack
> syncblk            # Show all sync blocks (locks held)
```

```csharp
// Code to detect potential deadlocks: use CancellationToken with lock timeout
using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(30));
bool acquired = false;

try
{
    acquired = await _semaphore.WaitAsync(30000);  // 30-second timeout
    if (!acquired)
    {
        _logger.LogError("Could not acquire lock within 30 seconds — possible deadlock");
        throw new TimeoutException("Failed to acquire lock");
    }
    
    // Critical section
}
finally
{
    if (acquired) _semaphore.Release();
}
```
