# CATEGORY 81 — COMPLETE DIAGNOSTIC COMMANDS

---

## 81.01 — Linux/Container Diagnostics

**Process inspection**
```bash
# All running processes
ps aux | grep dotnet

# Process memory usage
cat /proc/<PID>/status | grep -E "VmRSS|VmSize|VmPeak"

# Top processes by CPU
top -p <PID>

# Thread count
cat /proc/<PID>/status | grep Threads

# Open file descriptors (connections, sockets, files)
ls -la /proc/<PID>/fd | wc -l
lsof -p <PID> | head -50

# Network connections
ss -tulpn | grep <PORT>          # TCP/UDP listeners
ss -tnp state established        # Established TCP connections
netstat -an | grep ESTABLISHED | wc -l  # Count established connections
```

**Memory diagnostics**
```bash
# Memory usage overview
free -h

# Memory by process
ps aux --sort=-%mem | head -10

# OOM kills in last boot
dmesg | grep -i "oom\|kill"
journalctl -k | grep -i oom

# Swap usage
swapon -s
cat /proc/swaps

# Page faults (high = memory pressure)
cat /proc/<PID>/status | grep -E "VmPFC|VmSwap"
```

**Disk diagnostics**
```bash
# Disk usage
df -h

# Directory sizes
du -sh /var/log/*
du -sh * | sort -h | tail -20

# Disk I/O
iostat -xz 1 10    # 1-second intervals, 10 times
iotop -p <PID>     # I/O by process

# Find large files
find / -size +1G -type f 2>/dev/null | head -10
```

**Docker container diagnostics**
```bash
# Container stats
docker stats --no-stream
docker stats <container_id> --format "{{.CPUPerc}} {{.MemUsage}}"

# Container logs
docker logs <container_id> --tail=100 --follow
docker logs <container_id> --since="2026-06-01T12:00:00"

# Execute command in container
docker exec -it <container_id> /bin/bash
docker exec <container_id> cat /proc/meminfo

# Inspect container
docker inspect <container_id>

# Container health
docker inspect <container_id> --format='{{.State.Health.Status}}'
docker inspect <container_id> --format='{{json .State.Health}}'
```

**Kubernetes diagnostics**
```bash
# Pod status
kubectl get pods -n <namespace>
kubectl get pods -o wide  # Include node placement

# Pod details (events, volumes, containers)
kubectl describe pod <pod-name> -n <namespace>

# Pod logs
kubectl logs <pod-name> -n <namespace> --tail=100
kubectl logs <pod-name> -n <namespace> --previous  # Previous crashed container
kubectl logs -f <pod-name>  # Follow

# Multi-container pod logs
kubectl logs <pod-name> -c <container-name> -n <namespace>

# Events (great for debugging)
kubectl get events -n <namespace> --sort-by='.lastTimestamp'

# Execute in pod
kubectl exec -it <pod-name> -n <namespace> -- /bin/bash

# Resource usage
kubectl top pods -n <namespace>
kubectl top nodes

# Port-forward for local debugging
kubectl port-forward pod/<pod-name> 8080:80 -n <namespace>
kubectl port-forward service/<svc-name> 8080:80 -n <namespace>

# Get all resources
kubectl get all -n <namespace>

# Delete stuck terminating pod
kubectl delete pod <pod-name> --grace-period=0 --force -n <namespace>
```
