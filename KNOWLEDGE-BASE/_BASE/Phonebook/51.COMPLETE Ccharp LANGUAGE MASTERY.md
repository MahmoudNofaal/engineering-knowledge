## 51.01 — LINQ — Every Operator Explained

**Filtering**
```csharp
// Where — filter by predicate
var active = orders.Where(o => o.Status == "Active");

// OfType — filter by type
var strings = objects.OfType<string>();  // Only strings, nulls excluded
```

**Projection**
```csharp
// Select — transform each element
var names = users.Select(u => u.Name);
var dtos = users.Select(u => new UserDto(u.Id, u.Name, u.Email));

// SelectMany — flatten nested collections
var allItems = orders.SelectMany(o => o.Items);
var orderWithItem = orders.SelectMany(
    o => o.Items,
    (order, item) => new { order.Id, item.ProductId });  // Two-param overload

// Zip — combine two sequences element by element
var paired = names.Zip(ages, (name, age) => $"{name} is {age}");
```

**Ordering**
```csharp
var sorted = orders.OrderBy(o => o.CreatedAt)
                   .ThenByDescending(o => o.Total)
                   .ThenBy(o => o.Id);

// Reverse — reverses the order of a sequence
var reversed = sorted.Reverse();
```

**Grouping**
```csharp
// GroupBy — group by key
var byStatus = orders.GroupBy(o => o.Status);
foreach (var group in byStatus)
{
    Console.WriteLine($"{group.Key}: {group.Count()} orders");
    foreach (var order in group) { ... }
}

// ToLookup — like GroupBy but immediately evaluated, allows multiple lookups
var lookup = orders.ToLookup(o => o.CustomerId);
var customerOrders = lookup[42];  // Fast, no re-enumeration

// GroupBy with result selector
var stats = orders.GroupBy(
    o => o.Status,
    (status, group) => new
    {
        Status = status,
        Count = group.Count(),
        Total = group.Sum(o => o.Total),
        Average = group.Average(o => o.Total)
    });
```

**Set operations**
```csharp
var a = new[] { 1, 2, 3, 4, 5 };

a.Union(b)      // 1, 2, 3, 4, 5, 6, 7   (distinct)

```

**Joining**

```csharp
// Inner join
var joined = orders.Join(
```

**Aggregation**

**Quantifiers**

**Element access**

**Conversion**
```csharp

orders.ToArray()
orders.ToDictionary(o => o.Id)
```

**Deferred vs. immediate execution**
```csharp
// Deferred: query is NOT executed here
var query = orders.Where(o => o.Total > 100).Select(o => o.Id);

// Still deferred: building the query
var filtered = query.OrderBy(id => id);

// EXECUTED here (ToList, ToArray, First, Count, Sum, etc.)
var results = filtered.ToList();

// EXECUTED here (foreach)
foreach (var id in filtered) { ... }

// Beware: multiple enumerations = multiple executions
var items = GetItemsFromDb();  // IQueryable or IEnumerable from DB
var count = items.Count();    // Query 1
var list = items.ToList();    // Query 2 (entire table again!)
// Fix: var list = items.ToList(); var count = list.Count;
```

## 51.02 — async/await Deep Understanding

**What async/await actually is — state machines**
```csharp
// What you write:
public async Task<int> GetOrderCountAsync()
{
    var orders = await _db.Orders.ToListAsync();
    return orders.Count;
}

// What the compiler generates (simplified):
public Task<int> GetOrderCountAsync()
{
    var stateMachine = new GetOrderCountAsyncStateMachine();
    stateMachine.builder = AsyncTaskMethodBuilder<int>.Create();
    stateMachine.state = -1;
    stateMachine.builder.Start(ref stateMachine);
    return stateMachine.builder.Task;
}

// The state machine has a MoveNext() method that runs each "step"
// between awaits. When an await suspends, MoveNext() returns.
// When the awaited task completes, it calls MoveNext() again.
```

**ConfigureAwait**
```csharp
// In ASP.NET Core: SynchronizationContext is null — ConfigureAwait(false) has no effect
// In WinForms/WPF: SynchronizationContext exists, ConfigureAwait(false) matters

// Rule for library code: always use ConfigureAwait(false)
// to avoid deadlocks when consumed by UI contexts
public async Task<string> GetDataAsync(CancellationToken ct)
{
    var response = await _httpClient.GetStringAsync("...", ct).ConfigureAwait(false);
    var processed = await ProcessAsync(response).ConfigureAwait(false);
    return processed;
}

// Rule for application code (ASP.NET Core): omit ConfigureAwait
// Context doesn't exist — no deadlock risk — omit for clarity
public async Task<IActionResult> GetData()
{
    var data = await _service.GetDataAsync();  // No ConfigureAwait needed
    return Ok(data);
}
```

**Async void — the pitfall**
```csharp
// WRONG: async void swallows exceptions, can't be awaited
public async void HandleButtonClick(object sender, EventArgs e)
{
    await DoSomethingAsync();  // Exception here → unhandled → app crash
}

// RIGHT: async Task and await it
public async Task HandleButtonClickAsync(object sender, EventArgs e)
{
    await DoSomethingAsync();
}
// Or for event handlers specifically:
public async void HandleButtonClick(object sender, EventArgs e)
{
    try
    {
        await DoSomethingAsync();
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Error handling click");
    }
}
```

**ValueTask vs Task**
```csharp
// Task always allocates a heap object
// ValueTask avoids allocation when result is already available (synchronous path)

// Use ValueTask when:
// - The method often completes synchronously (cache hit, already computed value)
// - High-performance hot paths where allocation matters

public ValueTask<string?> GetFromCacheAsync(string key)
{
    // Fast path: synchronous, no allocation
    if (_cache.TryGetValue(key, out var value))
        return new ValueTask<string?>(value);  // No heap allocation
    
    // Slow path: async, allocates
    return new ValueTask<string?>(GetFromCacheSlowAsync(key));
}

// Rules:
// - ValueTask can only be awaited once (unlike Task which can be awaited multiple times)
// - Don't store ValueTask in a field to await later
// - Don't await ValueTask from multiple threads
```

**Async streams (IAsyncEnumerable)**
```csharp
// Produce asynchronously, yield one by one
public async IAsyncEnumerable<Order> GetOrdersStreamAsync(
    [EnumeratorCancellation] CancellationToken ct = default)
{
    await foreach (var chunk in _db.Orders
        .AsAsyncEnumerable()
        .WithCancellation(ct))
    {
        yield return chunk;
    }
}

// Consume
await foreach (var order in GetOrdersStreamAsync(cancellationToken))
{
    await ProcessOrderAsync(order);
}

// In minimal API — stream response to client
app.MapGet("/api/orders/stream", (AppDbContext db, CancellationToken ct) =>
    db.Orders.AsAsyncEnumerable());
// Client receives NDJSON (newline-delimited JSON) streamed
```

**Task.WhenAll vs Task.WhenAny**
```csharp
// WhenAll — wait for all, get all results
var tasks = new[] { GetOrderAsync(1), GetOrderAsync(2), GetOrderAsync(3) };
var orders = await Task.WhenAll(tasks);  // All 3 run concurrently

// WhenAll exception handling — ALL exceptions collected
try
{
    await Task.WhenAll(tasks);
}
catch (Exception ex)
{
    // Only first exception, others lost
}
// Better: inspect each task
foreach (var task in tasks)
{
    if (task.IsFaulted)
        _logger.LogError(task.Exception, "Task failed");
}

// WhenAny — first to complete wins
var cts = new CancellationTokenSource();
var task1 = GetFromPrimaryAsync(cts.Token);
var task2 = GetFromFallbackAsync(cts.Token);

var winner = await Task.WhenAny(task1, task2);
cts.Cancel();  // Cancel the losing task
var result = await winner;  // Get result (propagates exception if failed)
```

## 51.03 — Pattern Matching — Every Form

```csharp
// Type pattern
if (animal is Dog dog)
    dog.Bark();

// Declaration pattern (same as type pattern above)
object obj = "hello";
if (obj is string s)
    Console.WriteLine(s.Length);

// Constant pattern
if (count is 0)
    return "No items";

// Relational patterns
if (score is >= 90)
    grade = "A";
else if (score is >= 80)
    grade = "B";

// Logical patterns (and, or, not)
if (age is >= 18 and < 65)
    category = "Adult";

if (value is not null)
    Process(value);

if (shape is Circle or Square)
    DrawSimple(shape);

// Property patterns
if (order is { Status: OrderStatus.Pending, Total: > 100 })
    ApplyDiscount(order);

// Nested property patterns
if (order is { Customer: { IsVIP: true }, Total: > 500 })
    ApplyVIPDiscount(order);

// Positional patterns (for types with Deconstruct)
Point point = new(3, 4);
if (point is (> 0, > 0))
    Console.WriteLine("First quadrant");

// List patterns (.NET 7+)
int[] arr = { 1, 2, 3 };
if (arr is [1, 2, 3])
    Console.WriteLine("Exact match");
if (arr is [1, ..])
    Console.WriteLine("Starts with 1");
if (arr is [.., int last])
    Console.WriteLine($"Ends with {last}");
if (arr is [int first, .., int final])
    Console.WriteLine($"First={first}, Last={final}");

// Switch expressions
string category = shape switch
{
    Circle c when c.Radius > 10 => "Large circle",
    Circle c => "Small circle",
    Rectangle r when r.Width == r.Height => "Square",
    Rectangle => "Rectangle",
    Triangle t => $"Triangle with area {t.Area}",
    null => "No shape",
    _ => "Unknown shape"
};

// Switch statement with when guard
switch (order)
{
    case { Status: OrderStatus.Cancelled }:
        return "Already cancelled";
    case { Total: > 1000 } when user.IsVIP:
        ApplyVIPCancellationFee(order);
        break;
    default:
        ApplyStandardCancellationFee(order);
        break;
}

// Tuple patterns
(bool isWeekend, bool isHoliday) = GetDayInfo(date);
string policy = (isWeekend, isHoliday) switch
{
    (true, true) => "Holiday weekend — double pay",
    (true, false) => "Weekend — time and a half",
    (false, true) => "Holiday — double pay",
    (false, false) => "Weekday — standard pay"
};
```

## 51.04 — Records Deep Dive

**Record features**
```csharp
// Positional record — compiler generates constructor, Deconstruct, init-only properties
public record Point(double X, double Y);

// Value equality by default
var p1 = new Point(1, 2);
var p2 = new Point(1, 2);
Console.WriteLine(p1 == p2);      // True (value equality)
Console.WriteLine(p1.Equals(p2)); // True
Console.WriteLine(p1.GetHashCode() == p2.GetHashCode()); // True

// Non-destructive mutation with with expression
var p3 = p1 with { X = 10 };  // New record: (10, 2). p1 unchanged.

// Deconstruction
var (x, y) = p1;

// Record with custom members
public record Order(int Id, decimal Total)
{
    // Additional properties
    public bool IsHighValue => Total > 1000;
    
    // Custom validation
    public Order
    {
        if (Total < 0) throw new ArgumentException("Total cannot be negative");
    }
    
    // Override ToString (records auto-generate one)
    public override string ToString() => $"Order #{Id}: {Total:C}";
}

// Record struct — value type record (no heap allocation for small records)
public record struct Vector2D(float X, float Y)
{
    public float Length => MathF.Sqrt(X * X + Y * Y);
    
    public static Vector2D operator +(Vector2D a, Vector2D b)
        => new(a.X + b.X, a.Y + b.Y);
}

// Inheritance
public record Animal(string Name);
public record Dog(string Name, string Breed) : Animal(Name);

var dog = new Dog("Rex", "German Shepherd");
if (dog is Dog { Breed: "German Shepherd" } gsd)
    Console.WriteLine($"{gsd.Name} is a GSD");
```

## 51.05 — Generics Advanced

**Generic constraints**
```csharp
// Unconstrained — only object members available
void Process<T>(T item) { item.ToString(); }

// where T : class — reference type, can be null
void ProcessRef<T>(T item) where T : class { item.GetHashCode(); }

// where T : struct — value type, not nullable
void ProcessVal<T>(T item) where T : struct { }

// where T : new() — must have parameterless constructor
T Create<T>() where T : new() => new T();

// where T : ISomething — implements interface
void Process<T>(T item) where T : IComparable<T>
    => Console.WriteLine(item.CompareTo(default(T)));

// where T : BaseClass — inherits from type
void Process<T>(T entity) where T : Entity
    => Console.WriteLine(entity.Id);

// Multiple constraints
T CreateAndProcess<T>() where T : class, IDisposable, new()
{
    var t = new T();
    t.Dispose();
    return t;
}

// notnull constraint (.NET 5+)
void Process<T>(T item) where T : notnull { }  // Prevents nullable reference types
```

**Covariance and Contravariance**
```csharp
// Covariance (out) — can use more derived type
// IEnumerable<Dog> can be assigned to IEnumerable<Animal>
IEnumerable<Dog> dogs = new List<Dog>();
IEnumerable<Animal> animals = dogs;  // Works — IEnumerable<out T>

// Contravariance (in) — can use less derived type
// Action<Animal> can be assigned to Action<Dog>
Action<Animal> animalProcessor = a => Console.WriteLine(a.Name);
Action<Dog> dogProcessor = animalProcessor;  // Works — Action<in T>

// Custom covariant interface
public interface IReadOnlyRepo<out T>  // out = covariant
{
    T GetById(int id);
    IEnumerable<T> GetAll();  // Can only RETURN T, not accept T as input
}

// Custom contravariant interface
public interface IComparer<in T>  // in = contravariant
{
    int Compare(T x, T y);  // Can only ACCEPT T as input, not return it
}
```

**Generic math (INumber, .NET 7+)**
```csharp
// Write algorithms that work with any numeric type
T Sum<T>(IEnumerable<T> values) where T : INumber<T>
{
    T total = T.Zero;
    foreach (var value in values)
        total += value;
    return total;
}

Sum(new[] { 1, 2, 3, 4, 5 });           // Returns int: 15
Sum(new[] { 1.5, 2.5, 3.0 });           // Returns double: 7.0
Sum(new[] { 1.5m, 2.5m, 3.0m });        // Returns decimal: 7.0
```
