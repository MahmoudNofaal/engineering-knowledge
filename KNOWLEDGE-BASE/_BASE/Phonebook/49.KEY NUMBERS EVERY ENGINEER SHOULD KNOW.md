# CATEGORY 49 — KEY NUMBERS EVERY ENGINEER SHOULD KNOW

---

## 49.01 — Latency Numbers

| Operation | Latency |
|---|---|
| CPU register access | < 1 ns |
| L1 cache access | ~1 ns |
| L2 cache access | ~4 ns |
| L3 cache access | ~10–20 ns |
| RAM access | ~60–100 ns |
| Compress 1 KB with Snappy | ~3 μs |
| Read 1 MB sequentially from RAM | ~3 μs |
| SSD random read | ~100 μs |
| Network same datacenter | ~0.5 ms |
| SSD read 1 MB sequentially | ~1 ms |
| Disk seek (HDD) | ~5–10 ms |
| Network cross-region (US-EU) | ~70–150 ms |
| Network cross-world | ~150–300 ms |
| DNS lookup | ~1–100 ms |
| TLS handshake (new connection) | ~100–300 ms |
| HTTP request (same datacenter) | ~1–5 ms |

**Implications**
- A database query doing one disk seek: ~10 ms minimum
- 10 sequential database calls: ~100 ms minimum — use batching/eager loading
- Cross-region database: +150 ms per round trip — use read replicas
- DNS lookup: significant for first requests — use persistent connections
