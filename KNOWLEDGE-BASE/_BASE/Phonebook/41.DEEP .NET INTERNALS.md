## 41.01 — The CLR Execution Model

**What happens when you run a .NET app**
```
1. OS loads the host executable (dotnet.exe or your NativeAOT binary)
2. Host initializes the CLR runtime
3. CLR locates and loads the entry-point assembly (your .dll)
4. CLR loads dependent assemblies as they're referenced (lazy loading)
5. CLR finds the entry point method (Program.Main or top-level statements)
6. JIT compiles the entry method IL to native code
7. Execution begins
8. As other methods are called, they are JIT compiled on first call
```

**Assembly loading**
```csharp
// Default ALC (AssemblyLoadContext) loads all app assemblies
// Custom ALCs enable plugin architectures with isolated assembly loading

public class PluginLoadContext : AssemblyLoadContext
{
    private readonly AssemblyDependencyResolver _resolver;
    
    public PluginLoadContext(string pluginPath) : base(isCollectible: true)
    {
        _resolver = new AssemblyDependencyResolver(pluginPath);
    }
    
    protected override Assembly? Load(AssemblyName assemblyName)
    {
        var assemblyPath = _resolver.ResolveAssemblyToPath(assemblyName);
        if (assemblyPath != null)
            return LoadFromAssemblyPath(assemblyPath);
        
        return null;  // Fall back to default context
    }
}

// Load and unload plugins dynamically
var context = new PluginLoadContext("plugins/MyPlugin.dll");
var assembly = context.LoadFromAssemblyPath("plugins/MyPlugin.dll");
var pluginType = assembly.GetTypes().First(t => typeof(IPlugin).IsAssignableFrom(t));
var plugin = (IPlugin)Activator.CreateInstance(pluginType)!;

// Unload the plugin
context.Unload();  // isCollectible: true allows unloading
// GC.Collect() needed to actually free memory
```

**IL (Intermediate Language) structure**
```csharp
// C# source
public static int Add(int a, int b) => a + b;

// IL (conceptual view)
// .method public static int32 Add(int32 a, int32 b)
// {
//   ldarg.0     // Push a onto stack
//   ldarg.1     // Push b onto stack
//   add         // Pop both, push their sum
//   ret         // Return top of stack
// }
```

**Inspecting IL with ILSpy or dnSpy**
These tools decompile .NET assemblies back to C# or show raw IL. Invaluable for:
- Understanding what the compiler generates (e.g., for async state machines)
- Debugging NuGet packages without source
- Understanding framework internals

**Type system fundamentals**
```
System.Object (root of all types)
  ├── ValueType (base of all value types)
  │     ├── Int32, Double, Boolean, Char, ...
  │     ├── Struct types
  │     └── Enum types
  └── Reference types
        ├── String (special: immutable, interned)
        ├── Array types
        ├── Delegate types
        └── Class types (your classes, collections, etc.)
```

## 41.02 — String Internals

**String immutability**
Strings are immutable in .NET. Every "modification" creates a new string object.
```csharp
string a = "Hello";
string b = a;
a = a + " World";  // a now references a NEW string object
// b still references "Hello"
```

**String interning**
Literal strings are interned — stored in a shared pool.
```csharp
string a = "hello";
string b = "hello";
Console.WriteLine(ReferenceEquals(a, b));  // True — same object from intern pool

string c = new string('h', 1) + "ello";  // Not interned by default
Console.WriteLine(ReferenceEquals(a, c));  // False — different objects

string d = string.Intern(c);
Console.WriteLine(ReferenceEquals(a, d));  // True — now interned
```

**String.Empty vs ""**
```csharp
string.IsNullOrEmpty(s)   // Checks null OR empty
string.IsNullOrWhiteSpace(s)  // Checks null, empty, OR whitespace

// "" allocates nothing (literal is interned)
// string.Empty is the same interned empty string
// They are effectively identical
```

**StringComparison**
```csharp
// Ordinal: byte-by-byte, culture-independent, fast
string.Equals("hello", "HELLO", StringComparison.OrdinalIgnoreCase)

// CurrentCulture: language-aware (Turkish "I" problem)
"istanbul".CompareTo("ISTANBUL")  // Wrong in Turkish locale where ı ≠ I

// For most programming purposes, use Ordinal
// For user-facing text display/sort, use CurrentCulture

// Dictionary key comparison
var dict = new Dictionary<string, int>(StringComparer.OrdinalIgnoreCase);
dict["KEY"] = 1;
dict["key"];  // Returns 1 — case-insensitive lookup
```

**String performance**
```csharp
// Concatenation — O(n²) for n strings
string result = "";
for (int i = 0; i < 1000; i++)
    result += i;  // 1000 allocations, each copying all previous content

// StringBuilder — O(n) amortized
var sb = new StringBuilder(capacity: 5000);  // Pre-allocate
for (int i = 0; i < 1000; i++)
    sb.Append(i);
string result = sb.ToString();  // One allocation

// String.Create — best for known final length
int[] values = { 1, 2, 3, 4, 5 };
string result = string.Create(values.Length * 2 - 1, values, (span, arr) =>
{
    for (int i = 0; i < arr.Length; i++)
    {
        arr[i].TryFormat(span, out int written);
        span = span[written..];
        if (i < arr.Length - 1)
        {
            span[0] = ',';
            span = span[1..];
        }
    }
});

// Interpolated strings — in hot paths, use string.Create or StringBuilder
// $"" strings compile to DefaultInterpolatedStringHandler which avoids
// boxing and reduces allocations in .NET 6+
string name = "Alice";
int age = 30;
string s = $"Name: {name}, Age: {age}";  // No boxing, efficient in .NET 6+
```

## 41.03 — Memory Layout of Objects

**Reference type object layout (64-bit)**
```
[8 bytes: Object Header (sync block index, GC bits)]
[8 bytes: Method Table Pointer (type handle)]
[Field 1]
[Field 2]
...

Total overhead: 16 bytes per object minimum
```

**Value type layout (in a struct)**
```csharp
public struct Point
{
    public double X;  // 8 bytes
    public double Y;  // 8 bytes
}
// Size: 16 bytes. No header overhead.
// Array of 1000 Points: 16,000 bytes contiguous
// Array of 1000 classes Point: 16,000 references + 1000 × (16 + 16) = 48,000 bytes scattered
```

**Field padding and alignment**
The CLR may add padding between fields for alignment:
```csharp
public class Padded
{
    public byte A;   // 1 byte
    // 7 bytes padding (for 8-byte alignment of next field)
    public long B;   // 8 bytes
    public byte C;   // 1 byte
    // 7 bytes padding
}
// Total: 24 bytes (not 10 bytes!)

// Control layout with StructLayout
[StructLayout(LayoutKind.Sequential, Pack = 1)]
public struct Packed
{
    public byte A;   // 1 byte
    public long B;   // 8 bytes
    public byte C;   // 1 byte
}
// Total: 10 bytes — no padding
// Penalty: Unaligned access is slower on some CPUs
```

**Understanding memory with ObjectLayoutInspector**
```csharp
// NuGet: ObjectLayoutInspector
TypeLayout.PrintLayout<MyClass>();
// Outputs: field offsets, sizes, padding
```

## 41.04 — Unsafe Code and Pointers

**When to use unsafe code**
- Interop with native libraries (P/Invoke) that expect raw pointers
- Performance-critical code where managed overhead is measurable
- Working with hardware (embedded, drivers)
- Implementing high-performance algorithms on memory buffers

```csharp
// Enable unsafe code in .csproj
<AllowUnsafeBlocks>true</AllowUnsafeBlocks>

unsafe
{
    int x = 42;
    int* ptr = &x;    // Pointer to x
    *ptr = 100;       // Dereference and assign
    Console.WriteLine(x);  // 100
    
    // Pointer arithmetic
    int[] arr = { 1, 2, 3, 4, 5 };
    fixed (int* p = arr)   // Pin array (prevent GC from moving it)
    {
        for (int i = 0; i < arr.Length; i++)
            Console.Write(*(p + i) + " ");  // Pointer arithmetic
    }
    
    // Struct to byte array conversion
    Point point = new Point(1.0, 2.0);
    byte* bytes = (byte*)&point;
    for (int i = 0; i < sizeof(Point); i++)
        Console.Write(bytes[i] + " ");
}

// Safe alternative: MemoryMarshal
Point point = new Point(1.0, 2.0);
ReadOnlySpan<byte> bytes = MemoryMarshal.AsBytes(
    MemoryMarshal.CreateReadOnlySpan(ref point, 1));
```

**P/Invoke (calling native code)**
```csharp
// Call Windows API functions
[DllImport("kernel32.dll", SetLastError = true)]
static extern bool CreateDirectory(string lpPathName, IntPtr lpSecurityAttributes);

[DllImport("user32.dll")]
static extern int MessageBox(IntPtr hWnd, string text, string caption, uint type);

// Modern P/Invoke with LibraryImport (source-generated, .NET 7+)
[LibraryImport("MyNativeLib", EntryPoint = "do_work")]
private static partial int DoWork(int input);
```
