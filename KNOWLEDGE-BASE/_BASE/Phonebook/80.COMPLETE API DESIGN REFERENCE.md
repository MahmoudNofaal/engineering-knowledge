## 80.01 — REST API Best Practices

**URL design**
```
Resources → nouns, not verbs
GET  /api/orders          → List orders
GET  /api/orders/{id}     → Get specific order
POST /api/orders          → Create order
PUT  /api/orders/{id}     → Replace order (full update)
PATCH /api/orders/{id}    → Partial update
DELETE /api/orders/{id}   → Delete order

Sub-resources:
GET  /api/orders/{id}/items            → Items of specific order
POST /api/orders/{id}/items            → Add item to order
DELETE /api/orders/{id}/items/{itemId} → Remove specific item

Actions (verbs, for non-CRUD):
POST /api/orders/{id}/cancel    → Cancel order (action, not CRUD)
POST /api/orders/{id}/ship      → Ship order
POST /api/payments/{id}/refund  → Refund payment
```

**HTTP status codes — precise usage**
```
2xx Success:
200 OK              — GET, PUT, PATCH success
201 Created         — POST success (include Location header)
202 Accepted        — Async operation started (will complete later)
204 No Content      — DELETE success, or PUT/PATCH with no body to return

3xx Redirection:
301 Moved Permanently — Resource URL has changed permanently
304 Not Modified      — ETags matched, use cached version

4xx Client Errors:
400 Bad Request       — Malformed request, validation failure
401 Unauthorized      — Not authenticated (misleading name)
403 Forbidden         — Authenticated but not authorized
404 Not Found         — Resource doesn't exist
405 Method Not Allowed — Right URL, wrong HTTP method
409 Conflict          — Concurrent modification conflict, business rule violation
410 Gone              — Resource existed but was permanently deleted
422 Unprocessable Entity — Syntactically correct but semantically invalid (domain rules)
429 Too Many Requests — Rate limited
                        Include: Retry-After header

5xx Server Errors:
500 Internal Server Error — Unexpected server error
502 Bad Gateway           — Upstream service error
503 Service Unavailable   — Server overloaded or down (use for health check failures)
504 Gateway Timeout       — Upstream service timeout
```

**Versioning strategies**
```
1. URL versioning (most common, most visible):
   /api/v1/orders
   /api/v2/orders
   
2. Header versioning (cleaner URLs):
   GET /api/orders
   Api-Version: 2
   
3. Query parameter:
   GET /api/orders?version=2
   
4. Content negotiation:
   Accept: application/vnd.yourapi.v2+json
   
Recommendation: URL versioning for public APIs (clear, cacheable, debuggable)
```

**Consistent error response format**
```json
{
  "type": "https://tools.ietf.org/html/rfc7231#section-6.5.1",
  "title": "Validation Error",
  "status": 400,
  "detail": "One or more validation errors occurred",
  "instance": "/api/orders",
  "traceId": "abc123xyz",
  "errors": {
    "Items": ["Order must have at least one item"],
    "CustomerId": ["Customer not found"]
  }
}
```

```csharp
// ASP.NET Core ProblemDetails — RFC 7807 compliant
builder.Services.AddProblemDetails(options =>
{
    options.CustomizeProblemDetails = ctx =>
    {
        ctx.ProblemDetails.Extensions["traceId"] =
            Activity.Current?.TraceId.ToString() ??
            ctx.HttpContext.TraceIdentifier;
        ctx.ProblemDetails.Extensions["nodeId"] = Environment.MachineName;
    };
});

// Global exception handler mapping to RFC 7807
public class GlobalExceptionHandler : IExceptionHandler
{
    public async ValueTask<bool> TryHandleAsync(
        HttpContext ctx, Exception ex, CancellationToken ct)
    {
        var (status, title) = ex switch
        {
            ValidationException vex => (400, "Validation Failed"),
            NotFoundException => (404, "Resource Not Found"),
            ForbiddenException => (403, "Access Denied"),
            DomainException => (422, "Business Rule Violation"),
            _ => (500, "An unexpected error occurred")
        };
        
        ctx.Response.StatusCode = status;
        await ctx.Response.WriteAsJsonAsync(new ProblemDetails
        {
            Status = status,
            Title = title,
            Detail = ex.Message,
            Instance = ctx.Request.Path
        }, ct);
        
        return true;
    }
}
```

## 80.03 — HATEOAS (Hypermedia as the Engine of Application State)

**What it is**
Each response includes links to related actions, so clients can discover what they can do next.

```json
{
  "id": 42,
  "status": "Pending",
  "total": 299.99,
  "_links": {
    "self": { "href": "/api/orders/42", "method": "GET" },
    "cancel": { "href": "/api/orders/42/cancel", "method": "POST" },
    "items": { "href": "/api/orders/42/items", "method": "GET" },
    "customer": { "href": "/api/customers/7", "method": "GET" }
  }
}
```

**When to use:** Public APIs where clients shouldn't hard-code URLs. Rarely used in practice due to complexity, but important to know.
