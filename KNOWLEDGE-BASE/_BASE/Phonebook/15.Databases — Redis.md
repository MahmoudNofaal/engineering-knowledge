## 15.01 — Redis Data Types and Use Cases

**Strings**
```bash
SET user:1:name "Alice"          # Set with no expiry
SET session:token "abc123" EX 3600  # Set with 60-minute expiry
GET user:1:name                  # Get value
INCR page:views:home             # Atomic increment
INCRBY user:1:score 10           # Increment by value
APPEND log:today " new entry"   # Append to string
```

In .NET:
```csharp
var db = redis.GetDatabase();
await db.StringSetAsync("key", "value", TimeSpan.FromHours(1));
var value = await db.StringGetAsync("key");
var newVal = await db.StringIncrementAsync("counter");
```

**Hashes — map of field-value pairs**
```bash
HSET user:1 name "Alice" email "alice@example.com" age 30
HGET user:1 name             # "Alice"
HMGET user:1 name email      # ["Alice", "alice@example.com"]
HGETALL user:1               # All fields
HINCRBY user:1 age 1         # Increment field
HDEL user:1 email            # Delete field
```

Use case: Store user sessions, object attributes without JSON serialization.

**Lists — ordered sequence, push/pop from either end**
```bash
LPUSH queue:jobs "job1"      # Add to left (head)
RPUSH queue:jobs "job2"      # Add to right (tail)
LRANGE queue:jobs 0 -1       # Get all elements
LPOP queue:jobs              # Remove and return from left
RPOP queue:jobs              # Remove and return from right
LLEN queue:jobs              # Length
BRPOP queue:jobs 30          # Blocking pop — wait up to 30 seconds
```

Use case: Message queues, recent activity lists, task queues.

**Sets — unordered collection of unique strings**
```bash
SADD users:online "user1" "user2" "user3"
SISMEMBER users:online "user1"     # Is member?
SMEMBERS users:online              # All members
SCARD users:online                 # Count
SREM users:online "user1"          # Remove
SUNION set1 set2                   # Union
SINTER set1 set2                   # Intersection
SDIFF set1 set2                    # Difference
```

Use case: Tags, unique visitors, follows/followers.

**Sorted Sets — ordered set with scores**
```bash
ZADD leaderboard 1500 "alice" 1200 "bob" 1800 "charlie"
ZRANGE leaderboard 0 -1 WITHSCORES  # All, ascending
ZREVRANGE leaderboard 0 2 WITHSCORES # Top 3, descending
ZRANK leaderboard "alice"           # Rank (0-indexed)
ZSCORE leaderboard "alice"          # Score
ZINCRBY leaderboard 100 "alice"     # Update score
```

Use case: Leaderboards, rate limiting (sliding window), priority queues, time-ordered data.

**In .NET — StackExchange.Redis**
```csharp
var redis = ConnectionMultiplexer.Connect("localhost:6379");
var db = redis.GetDatabase();

// Sorted set rate limiting
var key = $"rate:{userId}";
var now = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds();
var windowStart = now - 60_000; // 60 seconds

await db.SortedSetRemoveRangeByScoreAsync(key, 0, windowStart);
var count = await db.SortedSetLengthAsync(key);

if (count < limit)
{
    await db.SortedSetAddAsync(key, Guid.NewGuid().ToString(), now);
    await db.KeyExpireAsync(key, TimeSpan.FromMinutes(2));
    return true; // allowed
}
return false; // rate limited
```

## 15.02 — Redis Pub/Sub

```csharp
var subscriber = redis.GetSubscriber();

// Publisher
await subscriber.PublishAsync("notifications", JsonSerializer.Serialize(notification));

// Subscriber
await subscriber.SubscribeAsync("notifications", (channel, message) =>
{
    var notification = JsonSerializer.Deserialize<Notification>(message!);
    ProcessNotification(notification);
});

// Pattern subscribe
await subscriber.SubscribeAsync("user.*", (channel, message) =>
{
    // Receives messages from user.created, user.updated, etc.
});
```

## 15.03 — Redis Distributed Locks (Redlock)

```csharp
// Simple distributed lock
var lockKey = $"lock:order:{orderId}";
var lockValue = Guid.NewGuid().ToString();
var expiry = TimeSpan.FromSeconds(30);

// Try to acquire lock (SET NX EX)
bool acquired = await db.StringSetAsync(lockKey, lockValue, expiry,
    When.NotExists);  // NX: set only if not exists

if (!acquired)
    throw new ConcurrencyException("Order is being processed");

try
{
    await ProcessOrderAsync(orderId);
}
finally
{
    // Only release if we own the lock (compare-and-delete via Lua)
    var script = @"
        if redis.call('get', KEYS[1]) == ARGV[1] then
            return redis.call('del', KEYS[1])
        else
            return 0
        end";
    await db.ScriptEvaluateAsync(script,
        new RedisKey[] { lockKey },
        new RedisValue[] { lockValue });
}
```

## 15.04 — Redis Persistence

**RDB (Redis Database) — Snapshots**
- Periodic snapshots of the dataset to disk
- Configure: `SAVE 900 1` (save if 1 key changed in 900 seconds)
- Fast restart, compact
- Risk: data loss between snapshots (last snapshot to crash)

**AOF (Append-Only File)**
- Log every write operation
- `appendfsync everysec` — sync every second (recommended)
- `appendfsync always` — sync every write (safest, slowest)
- `appendfsync no` — let OS decide (fastest, least safe)
- Better durability, larger files, slower restart

**RDB + AOF combined (recommended for production)**
Best of both: AOF for durability, RDB for faster restarts.
