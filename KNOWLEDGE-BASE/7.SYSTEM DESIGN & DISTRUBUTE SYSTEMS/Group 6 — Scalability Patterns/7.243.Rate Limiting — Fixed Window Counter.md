Simplest rate limiting: counter per time window, reset at boundary. Problem: boundary spike (2× limit at window edges). Use for non-critical limits.

```csharp
// In-memory
public bool TryConsume() {
    if (DateTime.UtcNow - _windowStart >= _window) { _counter = 0; _windowStart = DateTime.UtcNow; }
    if (_counter >= _limit) return false;
    Interlocked.Increment(ref _counter);
    return true;
}

// Distributed (Redis)
var key = $"rl:{clientId}:{DateTime.UtcNow:yyyyMMddHHmm}";
var count = await redis.StringIncrementAsync(key);
if (count == 1) await redis.KeyExpireAsync(key, TimeSpan.FromSeconds(60));
return count <= limit;
```

**ADR:** Non-critical health check endpoint. 1,000 req/min per IP. Boundary spikes acceptable. ✅ Simplest; ⚠️ 2× spike possible.
