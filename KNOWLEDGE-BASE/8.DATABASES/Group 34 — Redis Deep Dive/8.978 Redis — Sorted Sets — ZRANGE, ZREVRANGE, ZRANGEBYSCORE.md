# Redis — Sorted Sets — ZRANGE, ZREVRANGE, ZRANGEBYSCORE

---

## Section 1 — Overview

Sorted sets in Redis are one of the most powerful data structures. Each member has an associated score (a double-precision floating point number), and members are always ordered by score. The commands ZRANGE, ZREVRANGE, ZRANGEBYSCORE, and ZREVRANGEBYSCORE allow you to query members by their position in the sorted set or by their score range.

ZRANGE returns members in ascending order of score (low to high). ZREVRANGE returns members in descending order (high to low). Both take a start and stop index (0-based). ZRANGEBYSCORE and ZREVRANGEBYSCORE return members whose scores fall within a given range.

These commands are the primary way to read data from sorted sets. They are used extensively in leaderboards, time-series data, rangable indexes, and any pattern that requires ordered access to data.

---

### 1.1 — Core Concept

A sorted set is like a hybrid of a Set (unique members) and a List (ordered by score). The score determines position. When scores are equal, members are ordered lexicographically by their string value.

```
Redis Sorted Set Internal Structure (skiplist + hash table)
┌─────────────────────────────────────────────────────┐
│  member "player1"  ←→  score 1500                   │
│  member "player2"  ←→  score 2200                   │
│  member "player3"  ←→  score 1800                   │
│  ...                                                 │
│  Ordered by score ascending (skiplist)               │
│  O(1) score lookup (hash table)                      │
└─────────────────────────────────────────────────────┘
```

---

### 1.2 — When to Use

Use ZRANGE / ZREVRANGE when you need:
- Top N or bottom N items in a leaderboard
- Paginated results from a sorted collection
- A window of items around a known position

Use ZRANGEBYSCORE / ZREVRANGEBYSCORE when you need:
- All items within a specific score range
- Time-window queries (score = timestamp)
- Filtering by score thresholds

## Section 2 — Command Reference

---

### 2.1 — ZRANGE

Returns a range of members by index (position) in ascending order (low score first).

**Syntax:**
```
ZRANGE key start stop [WITHSCORES]
```

**Parameters:**
- `key` — the sorted set key
- `start` — 0-based start index (inclusive)
- `stop` — 0-based stop index (inclusive). Use `-1` for the last element
- `WITHSCORES` — optional flag to return scores alongside members

**Complexity:** O(log N + M) where N is the number of elements in the sorted set and M is the number of returned elements.

**Return:** Array of members (and optionally scores) in ascending score order.

**Redis CLI Example:**
```
> ZADD leaderboard 100 player1 200 player2 50 player3 300 player4
(integer) 4

> ZRANGE leaderboard 0 -1
1) "player3"
2) "player1"
3) "player2"
4) "player4"

> ZRANGE leaderboard 0 2 WITHSCORES
1) "player3"
2) "50"
3) "player1"
4) "100"
5) "player2"
6) "200"
```

---

### 2.2 — ZREVRANGE

Returns a range of members by index in descending order (high score first). Identical semantics to ZRANGE but reversed.

**Syntax:**
```
ZREVRANGE key start stop [WITHSCORES]
```

**Redis CLI Example:**
```
> ZREVRANGE leaderboard 0 2 WITHSCORES
1) "player4"
2) "300"
3) "player2"
4) "200"
5) "player1"
6) "100"
```

**Pagination Pattern:**
```
Page 1 (top 10):  ZREVRANGE leaderboard 0 9 WITHSCORES
Page 2 (next 10): ZREVRANGE leaderboard 10 19 WITHSCORES
Page 3 (next 10): ZREVRANGE leaderboard 20 29 WITHSCORES
```

---

### 2.3 — ZRANGEBYSCORE

Returns members with scores within a specified range (inclusive by default).

**Syntax:**
```
ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]
```

**Parameters:**
- `min` — minimum score (use `-inf` for unbounded)
- `max` — maximum score (use `+inf` for unbounded)
- `(` prefix on min/max makes the bound exclusive (e.g., `(100` means scores > 100)
- `WITHSCORES` — include scores in result
- `LIMIT offset count` — similar to SQL LIMIT for pagination within score range

**Redis CLI Example:**
```
> ZRANGEBYSCORE leaderboard 100 200 WITHSCORES
1) "player1"
2) "100"
3) "player2"
4) "200"

> ZRANGEBYSCORE leaderboard (100 +inf
1) "player2"
2) "player4"

> ZRANGEBYSCORE leaderboard -inf +inf LIMIT 1 2
1) "player1"
2) "player2"
```

---

### 2.4 — ZREVRANGEBYSCORE

Same as ZRANGEBYSCORE but returns members in descending score order. Note the argument order: max comes before min (since we start from highest).

**Syntax:**
```
ZREVRANGEBYSCORE key max min [WITHSCORES] [LIMIT offset count]
```

**Redis CLI Example:**
```
> ZREVRANGEBYSCORE leaderboard +inf 100 WITHSCORES
1) "player4"
2) "300"
3) "player2"
4) "200"
5) "player1"
6) "100"
```

---

### 2.5 — ZREVRANGEBYLEX / ZRANGEBYLEX

Lexicographical variants. For sorted sets where all members have the same score, these commands allow you to query by lexicographic range.

**Syntax:**
```
ZRANGEBYLEX key min max [LIMIT offset count]
ZREVRANGEBYLEX key max min [LIMIT offset count]
```

**Note:** These only work correctly when all members in the sorted set have the same score. Redis 6.2+ introduced `ZRANGE` with a `BYSCORE` and `BYLEX` argument, making them somewhat redundant.

---

### 2.6 — ZRANGE (Redis 6.2+ Unified Command)

Redis 6.2+ introduced a new unified ZRANGE syntax that replaces many of the above commands:

```
ZRANGE key start stop [BYSCORE | BYLEX] [REV] [LIMIT offset count] [WITHSCORES]
```

**Examples:**
```
> ZRANGE leaderboard 0 9 REV WITHSCORES
(same as ZREVRANGE leaderboard 0 9 WITHSCORES)

> ZRANGE leaderboard 100 200 BYSCORE WITHSCORES
(same as ZRANGEBYSCORE leaderboard 100 200 WITHSCORES)

> ZRANGE leaderboard +inf 100 BYSCORE REV WITHSCORES
(same as ZREVRANGEBYSCORE leaderboard +inf 100 WITHSCORES)
```

The unified syntax is the recommended approach for Redis 6.2+. However, many client libraries (including StackExchange.Redis) still expose the traditional method names for clarity.

### 3.1 — Available Methods

StackExchange.Redis maps sorted set range commands to methods on `IDatabase`:

| Redis Command | StackExchange.Redis Method | Returns |
|---|---|---|
| ZRANGE | `SortedSetRangeByRank` / `SortedSetRangeByRankAsync` | `RedisValue[]` |
| ZRANGE WITHSCORES | `SortedSetRangeByRankWithScores` / `SortedSetRangeByRankWithScoresAsync` | `SortedSetEntry[]` |
| ZRANGEBYSCORE | `SortedSetRangeByScore` / `SortedSetRangeByScoreAsync` | `RedisValue[]` |
| ZRANGEBYSCORE WITHSCORES | `SortedSetRangeByScoreWithScores` / `SortedSetRangeByScoreWithScoresAsync` | `SortedSetEntry[]` |
| ZRANGEBYVALUE | `SortedSetRangeByValue` / `SortedSetRangeByValueAsync` | `RedisValue[]` |
| ZRANK / ZREVRANK | `SortedSetRank` / `SortedSetRankAsync` | `long?` |
| ZCARD | `SortedSetLength` / `SortedSetLengthAsync` | `long` |
| ZCOUNT | `SortedSetLengthByValue` / `SortedSetLengthByValueAsync` | `long` |

### 3.3 — Order and Exclude Enums

```csharp
public enum Order
{
    Ascending,   // default — low to high
    Descending   // high to low (equivalent to REV)
}

public enum Exclude
{
    None,        // inclusive on both sides (default)
    Both,        // exclusive on both sides
    Start,       // exclusive on start/min
    End          // exclusive on stop/max
}
```

### 4.1 — Basic ZRANGE and ZREVRANGE Operations

```csharp
using StackExchange.Redis;
using System;
using System.Threading.Tasks;

public class SortedSetRangeExamples
{
    private readonly IDatabase _db;

    public SortedSetRangeExamples(IDatabase db)
    {
        _db = db ?? throw new ArgumentNullException(nameof(db));
    }

    /// <summary>
    /// Get top N players by highest score (ZREVRANGE)
    /// </summary>
    public async Task<SortedSetEntry[]> GetTopPlayersAsync(string leaderboardKey, int count)
    {
        try
        {
            // ZREVRANGE key 0 count-1 WITHSCORES
            return await _db.SortedSetRangeByRankWithScoresAsync(
                leaderboardKey,
                start: 0,
                stop: count - 1,
                order: Order.Descending);
        }
        catch (RedisConnectionException ex)
        {
            Console.Error.WriteLine($"Redis connection error: {ex.Message}");
            return Array.Empty<SortedSetEntry>();
        }
        catch (RedisTimeoutException ex)
        {
            Console.Error.WriteLine($"Redis timeout error: {ex.Message}");
            return Array.Empty<SortedSetEntry>();
        }
    }

    /// <summary>
    /// Get bottom N players by lowest score (ZRANGE)
    /// </summary>
    public async Task<SortedSetEntry[]> GetBottomPlayersAsync(string leaderboardKey, int count)
    {
        // ZRANGE key 0 count-1 WITHSCORES
        return await _db.SortedSetRangeByRankWithScoresAsync(
            leaderboardKey,
            start: 0,
            stop: count - 1,
            order: Order.Ascending);
    }

    /// <summary>
    /// Get a paginated page from the leaderboard (ZREVRANGE with offset)
    /// </summary>
    public async Task<SortedSetEntry[]> GetLeaderboardPageAsync(
        string leaderboardKey, int pageNumber, int pageSize)
    {
        // Page 1 = 0..pageSize-1, Page 2 = pageSize..2*pageSize-1, etc.
        long start = (pageNumber - 1) * pageSize;
        long stop = start + pageSize - 1;

        return await _db.SortedSetRangeByRankWithScoresAsync(
            leaderboardKey,
            start: start,
            stop: stop,
            order: Order.Descending);
    }
}
```

### 4.2 — ZRANGEBYSCORE Operations

```csharp
using StackExchange.Redis;
using System;
using System.Threading.Tasks;

public class ScoreRangeExamples
{
    private readonly IDatabase _db;

    public ScoreRangeExamples(IDatabase db)
    {
        _db = db;
    }

    /// <summary>
    /// Get members within a specific score range (inclusive)
    /// </summary>
    public async Task<SortedSetEntry[]> GetPlayersByScoreRangeAsync(
        string key, double minScore, double maxScore)
    {
        // ZRANGEBYSCORE key minScore maxScore WITHSCORES
        return await _db.SortedSetRangeByScoreWithScoresAsync(
            key,
            start: minScore,
            stop: maxScore,
            exclude: Exclude.None,
            order: Order.Ascending);
    }

    /// <summary>
    /// Get top N players with score above a threshold (exclusive)
    /// </summary>
    public async Task<SortedSetEntry[]> GetTopPlayersAboveScoreAsync(
        string key, double minScore, int count)
    {
        // ZREVRANGEBYSCORE key +inf (minScore WITHSCORES LIMIT 0 count
        return await _db.SortedSetRangeByScoreWithScoresAsync(
            key,
            start: minScore,
            stop: double.PositiveInfinity,
            exclude: Exclude.Start,  // (minScore — exclusive
            order: Order.Descending,
            skip: 0,
            take: count);
    }

    /// <summary>
    /// Count members in score range (ZCOUNT equivalent)
    /// </summary>
    public async Task<long> CountPlayersInScoreRangeAsync(
        string key, double minScore, double maxScore)
    {
        // ZCOUNT key minScore maxScore
        return await _db.SortedSetLengthAsync(
            key,
            min: minScore,
            max: maxScore);
    }

    /// <summary>
    /// Get players who joined between two timestamps (score = timestamp)
    /// </summary>
    public async Task<RedisValue[]> GetPlayersByTimeRangeAsync(
        string key, DateTime from, DateTime to)
    {
        double fromTimestamp = ((DateTimeOffset)from).ToUnixTimeMilliseconds();
        double toTimestamp = ((DateTimeOffset)to).ToUnixTimeMilliseconds();

        return await _db.SortedSetRangeByScoreAsync(
            key,
            start: fromTimestamp,
            stop: toTimestamp,
            exclude: Exclude.None,
            order: Order.Ascending);
    }
}
```

### 4.3 — Paginated Leaderboard Service

```csharp
using StackExchange.Redis;
using System;
using System.Collections.Generic;
using System.Threading.Tasks;

public class LeaderboardPaginationService
{
    private readonly IDatabase _db;
    private const string DefaultLeaderboardKey = "leaderboard:game1";

    public LeaderboardPaginationService(IDatabase db)
    {
        _db = db;
    }

    public async Task<LeaderboardPageResult> GetLeaderboardPageAsync(
        int pageNumber,
        int pageSize = 10,
        string key = DefaultLeaderboardKey)
    {
        long totalMembers = await _db.SortedSetLengthAsync(key);
        long start = (pageNumber - 1) * pageSize;
        long stop = start + pageSize - 1;

        if (start >= totalMembers)
        {
            return new LeaderboardPageResult
            {
                Page = pageNumber,
                PageSize = pageSize,
                TotalMembers = totalMembers,
                TotalPages = (int)Math.Ceiling((double)totalMembers / pageSize),
                Entries = Array.Empty<LeaderboardEntry>()
            };
        }

        SortedSetEntry[] entries = await _db.SortedSetRangeByRankWithScoresAsync(
            key,
            start: start,
            stop: stop,
            order: Order.Descending);

        var result = new List<LeaderboardEntry>();
        int rank = (int)start + 1; // 1-based rank for display

        foreach (SortedSetEntry entry in entries)
        {
            result.Add(new LeaderboardEntry
            {
                Rank = rank,
                PlayerId = entry.Element.ToString(),
                Score = (long)entry.Score
            });
            rank++;
        }

        return new LeaderboardPageResult
        {
            Page = pageNumber,
            PageSize = pageSize,
            TotalMembers = totalMembers,
            TotalPages = (int)Math.Ceiling((double)totalMembers / pageSize),
            Entries = result.ToArray()
        };
    }
}

public class LeaderboardPageResult
{
    public int Page { get; set; }
    public int PageSize { get; set; }
    public long TotalMembers { get; set; }
    public int TotalPages { get; set; }
    public LeaderboardEntry[] Entries { get; set; } = Array.Empty<LeaderboardEntry>();
}

public class LeaderboardEntry
{
    public int Rank { get; set; }
    public string PlayerId { get; set; } = string.Empty;
    public long Score { get; set; }
}
```

### 4.4 — ZRANGEBYSCORE with Time-Window Query

```csharp
using StackExchange.Redis;
using System;
using System.Threading.Tasks;

public class TimeSeriesQueryService
{
    private readonly IDatabase _db;

    public TimeSeriesQueryService(IDatabase db)
    {
        _db = db;
    }

    /// <summary>
    /// Query events within a sliding time window using score = timestamp_ms.
    /// Members are event IDs, scores are Unix timestamps in milliseconds.

    {

    /// </summary>

        DateTime startTime,
        DateTime endTime)
    {
        double startMs = ((DateTimeOffset)startTime).ToUnixTimeMilliseconds();
        double endMs = ((DateTimeOffset)endTime).ToUnixTimeMilliseconds();

        return await _db.SortedSetRangeByScoreWithScoresAsync(
            key,
            start: startMs,
            stop: endMs,
            order: Order.Ascending);
```

### 4.5 — Error Handling Patterns

```csharp
using StackExchange.Redis;
using System;
using System.Threading.Tasks;
using Microsoft.Extensions.Logging;

public class ResilientSortedSetService
{
    private readonly IDatabase _db;
    private readonly ILogger<ResilientSortedSetService> _logger;
    private const int MaxRetries = 3;

    public ResilientSortedSetService(
        IDatabase db,
        ILogger<ResilientSortedSetService> logger)
    {
        _db = db;
        _logger = logger;
    }

    public async Task<SortedSetEntry[]> SafeRangeByRankWithScoresAsync(
        RedisKey key,
        long start,
        long stop,
        Order order = Order.Ascending)
    {
        for (int attempt = 1; attempt <= MaxRetries; attempt++)
        {
            try
            {
                return await _db.SortedSetRangeByRankWithScoresAsync(
                    key, start, stop, order);
            }
            catch (RedisConnectionException ex)
            {
                _logger.LogWarning(ex,
                    "Redis connection error on attempt {Attempt}/{MaxRetries}",
                    attempt, MaxRetries);

                if (attempt == MaxRetries)
                {
                    _logger.LogError(ex,
                        "Failed to query sorted set {Key} after {MaxRetries} attempts",
                        key, MaxRetries);
                    return Array.Empty<SortedSetEntry>();
                }

                await Task.Delay(TimeSpan.FromMilliseconds(100 * attempt));
            }
            catch (RedisTimeoutException ex)
            {
                _logger.LogWarning(ex,
                    "Redis timeout on attempt {Attempt}/{MaxRetries}",
                    attempt, MaxRetries);

                if (attempt == MaxRetries)
                    throw; // re-throw timeouts on last attempt

                await Task.Delay(TimeSpan.FromMilliseconds(200 * attempt));
            }
        }

        return Array.Empty<SortedSetEntry>();
    }

    public async Task<RedisValue[]> SafeRangeByScoreAsync(
        RedisKey key,
        double start,
        double stop,
        Exclude exclude = Exclude.None,
        Order order = Order.Ascending,
        long skip = 0,
        long take = -1)
    {
        try
        {
            return await _db.SortedSetRangeByScoreAsync(
                key, start, stop, exclude, order, skip, take);
        }
        catch (RedisServerException ex) when (ex.Message.Contains("WRONGTYPE"))
        {
            _logger.LogError(ex,
                "Key {Key} is not a sorted set", key);
            return Array.Empty<RedisValue>();
        }
        catch (RedisException ex)
        {
            _logger.LogError(ex,
                "Redis error querying sorted set {Key}", key);
            return Array.Empty<RedisValue>();
        }
    }
}
```

### 4.6 — Bulk Range Operations with Pipeline

```csharp
using StackExchange.Redis;
using System;
using System.Threading.Tasks;

public class BulkRangeService
{
    private readonly ConnectionMultiplexer _muxer;

    public BulkRangeService(ConnectionMultiplexer muxer)
    {
        _muxer = muxer;
    }

    /// <summary>
    /// Query multiple leaderboard pages in a single round trip
    /// using a batch/pipeline.
    /// </summary>
    public async Task<SortedSetEntry[][]> GetMultiplePagesAsync(
        string key,
        int[] pageNumbers,
        int pageSize = 10)
    {
        IDatabase db = _muxer.GetDatabase();
        IBatch batch = db.CreateBatch();

        var tasks = new Task<SortedSetEntry[]>[pageNumbers.Length];

        for (int i = 0; i < pageNumbers.Length; i++)
        {
            long start = (pageNumbers[i] - 1) * pageSize;
            long stop = start + pageSize - 1;

            tasks[i] = batch.SortedSetRangeByRankWithScoresAsync(
                key,
                start: start,
                stop: stop,
                order: Order.Descending);
        }

        batch.Execute();

        var results = new SortedSetEntry[pageNumbers.Length][];
        for (int i = 0; i < tasks.Length; i++)
        {
            results[i] = await tasks[i];
        }

        return results;
    }
}
```

## Section 5 — Use Cases

---

### 5.1 — Game Leaderboards

The most common use case. Player IDs are members, scores are their total points.

```
Key: leaderboard:game:{gameId}
Members: playerId
Scores: totalScore
```

**Pattern:**
- ZREVRANGE to get top 10 players (highest scores)
- ZRANGE to get bottom 10 (lowest scores)
- ZREVRANGE BYSCORE to get all players above a threshold (e.g., 1000 points)
- Paginated ZREVRANGE for "load more" functionality

**Game Server Example:**
```csharp
public async Task<List<LeaderboardEntry>> GetWeeklyLeaderboardAsync(
    string gameId, int topN = 10)
{
    string key = $"leaderboard:weekly:{gameId}";

    SortedSetEntry[] top = await _db.SortedSetRangeByRankWithScoresAsync(
        key, start: 0, stop: topN - 1, order: Order.Descending);

    var result = new List<LeaderboardEntry>();
    for (int i = 0; i < top.Length; i++)
    {
        result.Add(new LeaderboardEntry
        {
            Rank = i + 1,
            PlayerId = top[i].Element,
            Score = (long)top[i].Score
        });
    }

    return result;
}
```

### 5.2 — Time-Series / Event Log

Use the sorted set as a time-ordered index where score = timestamp.

```
Key: events:{eventType}
Members: eventId
Scores: timestamp_ms
```

**Pattern:**
- ZADD events:login "event_12345" 1704067200000
- ZRANGEBYSCORE events:login 1704067200000 1704153600000 (events in a day)
- ZREMRANGEBYSCORE events:login -inf 1704067199999 (events older than cutoff)

**Time-Series Example:**
```csharp
public class EventLogService
{
    private readonly IDatabase _db;
    private const string EventKey = "events:api:calls";

    public EventLogService(IDatabase db) => _db = db;

    public async Task RecordEventAsync(string eventId)
    {
        long timestampMs = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds();
        await _db.SortedSetAddAsync(EventKey, eventId, timestampMs);
    }

    public async Task<SortedSetEntry[]> GetEventsLastHourAsync()
    {
        long nowMs = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds();
        long oneHourAgoMs = nowMs - TimeSpan.FromHours(1).TotalMilliseconds;

        return await _db.SortedSetRangeByScoreWithScoresAsync(
            EventKey,
            start: oneHourAgoMs,
            stop: nowMs,
            order: Order.Descending);
    }

    public async Task CleanupOldEventsAsync(TimeSpan retention)
    {
        long cutoffMs = DateTimeOffset.UtcNow
            .Subtract(retention)
            .ToUnixTimeMilliseconds();

        // ZREMRANGEBYSCORE key -inf cutoffMs
        long removed = await _db.SortedSetRemoveRangeByScoreAsync(
            EventKey,
            start: double.NegativeInfinity,
            stop: cutoffMs);

        if (removed > 0)
        {
            Console.WriteLine($"Cleaned up {removed} old events");
        }
    }
}
```

### 5.3 — Paginated Search Results

Store ranked search results in a sorted set (score = relevance ranking), then paginate with ZRANGE.

```csharp
public class SearchResultsService
{
    private readonly IDatabase _db;

    public SearchResultsService(IDatabase db) => _db = db;

    public async Task StoreSearchResultsAsync(
        string queryHash,
        Dictionary<string, double> results)
    {
        string key = $"search:{queryHash}";
        
        // Store with TTL (search results expire)
        var entries = results.Select(r => new SortedSetEntry(r.Key, r.Value))
                             .ToArray();

        await _db.SortedSetAddAsync(key, entries);
        await _db.KeyExpireAsync(key, TimeSpan.FromMinutes(5));
    }

    public async Task<SortedSetEntry[]> GetSearchPageAsync(
        string queryHash, int page, int pageSize = 10)
    {
        string key = $"search:{queryHash}";
        long start = (page - 1) * pageSize;
        long stop = start + pageSize - 1;

        // ZREVRANGE — highest relevance first
        return await _db.SortedSetRangeByRankWithScoresAsync(
            key, start, stop, Order.Descending);
    }
}
```

### 5.5 — Gaming Leaderboard with Score Tiebreaker

When two players have the same score, lexicographic order applies. To control tiebreaker behavior, encode a tiebreaker in the score:

```csharp
/// <summary>
/// Encode score and tiebreaker into a single double.
/// score * 1_000_000_000 + (999_999_999 - timestamp_ms_suffix)
/// Higher timestamp_suffix wins ties (earlier = worse for freshness)
/// </summary>
public double EncodeScoreWithTiebreaker(long score, long timestampMs)
{
    // Use lower bits of timestamp as fractional tiebreaker
    // Assuming scores < 10^12
    const long tiebreakerMax = 999_999;
    long tiebreaker = tiebreakerMax - (timestampMs % (tiebreakerMax + 1));
    return score * (tiebreakerMax + 1) + tiebreaker;
}

public (long score, long timestampMs) DecodeScore(double encoded)
{
    const long tiebreakerMax = 999_999;
    long score = (long)(encoded / (tiebreakerMax + 1));
    long tiebreaker = (long)(encoded % (tiebreakerMax + 1));
    long timestampMs = tiebreakerMax - tiebreaker;
    return (score, timestampMs);
}
```

Usage in leaderboard:
```csharp
public async Task SubmitScoreWithTiebreakerAsync(
    string leaderboardKey, string playerId, long score)
{
    long nowMs = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds();
    double encodedScore = EncodeScoreWithTiebreaker(score, nowMs);

    await _db.SortedSetAddAsync(leaderboardKey, playerId, encodedScore);
}
```

## Section 6 — Performance and Complexity

---

### 6.1 — Time Complexity

| Command | Complexity | Notes |
|---|---|---|
| ZRANGE | O(log N + M) | N = size of sorted set, M = returned elements |
| ZREVRANGE | O(log N + M) | Same as ZRANGE |
| ZRANGEBYSCORE | O(log N + M) | N = size, M = returned elements |
| ZREVRANGEBYSCORE | O(log N + M) | Same as ZRANGEBYSCORE |
| ZRANGEBYLEX | O(log N + M) | Only works when all scores equal |

---

### 6.2 — Memory Considerations

When using WITHSCORES:
- Each member is returned as a bulk string
- Each score is returned as a bulk string (double representation)
- Response size doubles (roughly)
- Large result sets can cause network buffering issues

**Best practice:** Always use LIMIT or small ranges. Never call `ZRANGE key 0 -1` on a large sorted set in production unless you know it's small.

```csharp
// BAD — fetches everything
RedisValue[] all = await db.SortedSetRangeByRankAsync("huge:key", 0, -1);

// GOOD — paginate
SortedSetEntry[] page = await db.SortedSetRangeByRankWithScoresAsync(
    "huge:key", 0, 99, Order.Descending);
```

---

### 6.3 — Sorted Set Internal Data Structure

Redis sorted sets use a combination of:
- **Skiplist** — allows O(log N) range queries by score
- **Hash table** — allows O(1) score lookups by member

When a sorted set is small (fewer than 128 elements and all elements are < 64 bytes), Redis uses a **ziplist** encoding for memory efficiency. The encoding is transparent to the client.

```bash
> OBJECT ENCODING leaderboard
"skiplist"   # for large sets
"ziplist"    # for small sets (< 128 elems, < 64 bytes each)
```

---

### 6.4 — Network Round-Trips

| Pattern | Round-Trips | Notes |
|---|---|---|
| Single ZRANGE | 1 | Optimal |
| Multiple ZRANGEs (sequential) | N | Use pipeline/batch |
| ZRANGE + ZRANK per member | 2 | Consider Lua script |
| Paginated pages (3 pages) | 3 | Use batch for 1 round-trip |

---

### 6.5 — Benchmark Estimates

```
Single node, localhost, 1000 elements in sorted set:

ZRANGE 0 9 (10 elements):     ~0.1ms
ZRANGE 0 99 (100 elements):   ~0.3ms
ZRANGE 0 -1 (1000 elements):  ~1.5ms
ZRANGEBYSCORE with LIMIT 10:  ~0.2ms
```

In production (network latency ~1ms), expect ~2ms per command plus transfer time.

## Section 7 — Gotchas and Pitfalls

---

### 7.1 — Score is a Double-Precision Float

Redis uses IEEE 754 double-precision floating point for scores. This means:
- Scores above 2^53 (9,007,199,254,740,991) lose precision
- Decimal fractions may have rounding errors
- Comparing scores for exact equality can be unreliable

```csharp
// Potential precision issue
double score = 12345678901234567890.0;
// Stored as ~12345678901234567168 (precision loss)

// Solution: use long for integer scores
long safeScore = 1234567890123456789L; // < 2^53 is safe
```

---

### 7.2 — WITHSCORES Doubles Response Size

Without WITHSCORES, ZRANGE returns only member names. WITHSCORES interleaves scores, doubling the number of returned elements. Be aware of this when parsing:

```csharp
// Without WITHSCORES — returns member names only
RedisValue[] members = await db.SortedSetRangeByRankAsync(key, 0, 9);

// With WITHSCORES — returns member + score pairs
SortedSetEntry[] entries = await db.SortedSetRangeByRankWithScoresAsync(key, 0, 9);
foreach (SortedSetEntry entry in entries)
{
    string member = entry.Element;  // member name
    double score = entry.Score;     // associated score
}
```

---

### 7.3 — Inclusive vs Exclusive Bounds in ZRANGEBYSCORE

Default bounds are inclusive. Use `(` prefix for exclusive:

```bash
# Inclusive: scores 100 to 200
ZRANGEBYSCORE leaderboard 100 200

# Exclusive min: scores > 100
ZRANGEBYSCORE leaderboard (100 200

# Exclusive max: scores >= 100 and < 200
ZRANGEBYSCORE leaderboard 100 (200
```

In StackExchange.Redis, use `Exclude` enum:
```csharp
Exclude.None      // inclusive both sides
Exclude.Start     // exclusive min
Exclude.End       // exclusive max
Exclude.Both      // exclusive both sides
```

---

### 7.4 — Negative Indices

ZRANGE and ZREVRANGE support negative indices:
- `-1` = last element
- `-2` = second-to-last

```bash
# Get all elements except the last
ZRANGE leaderboard 0 -2

# Get last 3 elements (lowest scores)
ZRANGE leaderboard -3 -1
```

---

### 7.5 — Empty Sets and Missing Keys

If the key does not exist or the sorted set is empty:
- ZRANGE, ZREVRANGE, ZRANGEBYSCORE all return an empty array
- ZCARD returns 0
- ZRANK returns nil (null in .NET)

```csharp
SortedSetEntry[] result = await db.SortedSetRangeByRankWithScoresAsync(
    "nonexistent:key", 0, 9);
// result is empty array, not null

long? count = await db.SortedSetLengthAsync("nonexistent:key");
// count = 0, not null
```

---

### 7.6 — WRONGTYPE Error

If the key holds a non-sorted-set type, all Z* commands return a WRONGTYPE error:

```csharp
try
{
    SortedSetEntry[] result = await db.SortedSetRangeByRankWithScoresAsync(
        "key:that:is:actually:a:string", 0, 9);
}
catch (RedisServerException ex) when (ex.Message.Contains("WRONGTYPE"))
{
    // Handle type mismatch
    Console.Error.WriteLine($"Key is not a sorted set: {ex.Message}");
}
```

---

### 7.7 — Large Result Sets Block the Server

ZRANGE with a very large range (e.g., `0 -1` on a set with millions of elements) blocks the Redis server while executing and creates a massive network response. Always use LIMIT or paginate.

```bash
# POTENTIALLY DANGEROUS — blocks Redis for large sets
ZRANGE leaderboard 0 -1

# SAFE — use pagination
ZRANGE leaderboard 0 99
ZRANGE leaderboard 100 199
# ...
```

---

### 7.8 — Lexicographic Ordering for Ties

When multiple members have the same score, they are ordered lexicographically by their string value. This can be surprising:

```bash
> ZADD test 0 aaa 0 bbb 0 AAA 0 BBB
> ZRANGE test 0 -1
1) "AAA"
2) "BBB"
3) "aaa"
4) "bbb"
```

Lexicographic order is case-sensitive (uppercase < lowercase in ASCII). For predictable tiebreaking, use a composite score or encode the tiebreaker in the member name.

---

### 7.9 — ZRANGEBYSCORE with +inf/-inf

Use `+inf` and `-inf` for unbounded ranges in ZRANGEBYSCORE:

```bash
# All scores >= 100
ZRANGEBYSCORE leaderboard 100 +inf

# All scores < 100
ZRANGEBYSCORE leaderboard -inf (100

# All scores
ZRANGEBYSCORE leaderboard -inf +inf
```

In StackExchange.Redis, use `double.PositiveInfinity` and `double.NegativeInfinity`:

```csharp
RedisValue[] all = await db.SortedSetRangeByScoreAsync(
    key,
    double.NegativeInfinity,
    double.PositiveInfinity);
```

---

### 7.10 — ZREVRANGEBYSCORE Argument Order

Note that ZREVRANGEBYSCORE takes `max` first, then `min` (because it reads from highest to lowest):

```bash
# Correct: max first, then min
ZREVRANGEBYSCORE leaderboard +inf 100

# Wrong (will not error but will return unexpected results):
ZREVRANGEBYSCORE leaderboard 100 +inf
```

## Section 8 — Related Patterns and Comparisons

---

### 8.1 — ZRANGE vs ZRANK

| Purpose | Command | Returns |
|---|---|---|
| Get members by position | ZRANGE | Members (with optional scores) |
| Get position of a member | ZRANK | Integer rank (0-based) |

Use ZRANGE when you want to list members. Use ZRANK when you want to find where a specific member sits.

---

### 8.2 — ZRANGEBYSCORE vs ZCOUNT

| Command | Returns | Use When |
|---|---|---|
| ZRANGEBYSCORE | Actual members | You need the member data |
| ZCOUNT | Count of members | You only need the count |

ZCOUNT is more efficient when you don't need the actual members since it returns just a number.

```csharp
long count = await _db.SortedSetLengthAsync(key, minScore, maxScore);
// vs
SortedSetEntry[] data = await _db.SortedSetRangeByScoreWithScoresAsync(
    key, minScore, maxScore);
```

---

### 8.3 — ZRANGE on Different Encodings

| Encoding | Behavior | Notes |
|---|---|---|
| ziplist (small set) | O(N) scan | Very fast for small N |
| skiplist (large set) | O(log N + M) | Scales well |

Redis handles the encoding internally. You generally don't need to worry about it unless you're debugging performance.

---

### 8.4 — Sorted Sets vs Lists for Ranking

| Feature | Sorted Set | List |
|---|---|---|
| Order maintained by | Score (sorted) | Insertion order |
| Update score | O(log N) | N/A (must re-insert) |
| Get by rank | O(log N) | O(N) for LPOP |
| Unique members | Yes | No |
| Memory per element | ~72 bytes (skiplist) | ~40 bytes |

Sorted sets are the clear winner for ranking/leaderboard scenarios.

---

### 8.5 — ZRANGEBYSCORE vs SCAN

| ZRANGEBYSCORE | SCAN with pattern |
|---|---|
| O(log N + M) | O(N) |
| Ordered by score | Unordered |
| Supports range queries | Supports pattern matching |
| Blocking for large ranges | Non-blocking (cursor-based) |

For range queries by score, ZRANGEBYSCORE is always preferred. Use SCAN only when you don't have a score-based access pattern.

### 9.3 — Key Design Principles

1. **Use WITHSCORES sparingly** — only include when you actually need the score data
2. **Always paginate** — never use unbounded ranges on unknown-sized sets
3. **Handle empty results** — methods return empty arrays, not null
4. **Use LIMIT with ZRANGEBYSCORE** — score ranges can match many elements
5. **Prefer Order enum** — use `Order.Descending` instead of separate ZREV methods
6. **Tiebreaker strategy** — plan for equal scores (lexicographic or composite score)
7. **Pipeline for multiple pages** — batch queries when requesting multiple ranges
