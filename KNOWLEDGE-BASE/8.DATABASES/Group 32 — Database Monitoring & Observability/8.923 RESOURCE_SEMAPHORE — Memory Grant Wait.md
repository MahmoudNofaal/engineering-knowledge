## Overview — RESOURCE_SEMAPHORE Wait Type

RESOURCE_SEMAPHORE occurs when a query requires a memory grant (for sorts, hash joins, batch mode operations, or index builds) but the total memory grants in use exceed the resource semaphore's limit. The query must wait until memory becomes available.

SQL Server's memory grant system works as follows:

1. **Query optimizer estimates** memory requirements based on cardinality estimates.
2. **Query requests a grant** from the resource semaphore.
3. **Resource semaphore checks** available memory.
4. **If memory is available**, the grant is awarded and the query executes.
5. **If memory is insufficient**, the query waits on RESOURCE_SEMAPHORE.

### Key Metrics

| Metric | Description | Target |
|--------|-------------|--------|
| waiter_count | Queries waiting for memory | 0 (any >0 is concern) |
| grantee_count | Currently executing with grants | Varies by workload |
| timeout_error_count | Memory grant timeouts (error 8645) | 0 |
| available_memory_pct | Free resource semaphore memory | >25% |
| avg_wait_ms | Average RESOURCE_SEMAPHORE wait | <100ms |

`sql
-- Current RESOURCE_SEMAPHORE waits at instance level
SELECT
    wait_type,
    waiting_tasks_count,
    wait_time_ms,
    max_wait_time_ms,
    signal_wait_time_ms,
    wait_time_ms / NULLIF(waiting_tasks_count, 0) AS avg_wait_ms,
    signal_wait_time_ms / NULLIF(wait_time_ms, 0) * 100 AS signal_wait_pct
FROM sys.dm_os_wait_stats
WHERE wait_type = 'RESOURCE_SEMAPHORE'
ORDER BY wait_time_ms DESC;
`

### Common Causes of RESOURCE_SEMAPHORE

| Cause | Description | Example |
|-------|-------------|---------|
| Large hash joins | Hash join builds a hash table in memory | Joining two large unindexed tables |
| Sort operations | ORDER BY or GROUP BY on large datasets | Reporting query sorting millions of rows |
| Batch mode | Columnstore batch mode requires large grants | DW query scanning columnstore |
| Index builds | CREATE/ALTER INDEX requires memory | Online index rebuild on large table |
| Memory pressure | Max server memory is too low | Instance configured with 4GB for 100GB database |
| Concurrent large queries | Multiple queries requesting grants simultaneously | Morning reporting rush |
| Bad cardinality estimates | Optimizer over-estimates row counts | Out of date statistics |

-- Currently executing queries and their memory grants
SELECT

    required_memory_kb / 1024 AS required_memory_mb,
    (granted_memory_kb - used_memory_kb) / 1024 AS unused_grant_mb,
    CASE
        WHEN grant_time IS NULL THEN 'WAITING — query has not been granted memory'
        ELSE 'GRANTED — query is executing with memory'
    END AS grant_status,
    query_text = (
        SELECT SUBSTRING(st.text,
            (r.statement_start_offset / 2) + 1,
            CASE
                WHEN r.statement_end_offset = -1 THEN LEN(CONVERT(NVARCHAR(MAX), st.text))
                ELSE (r.statement_end_offset - r.statement_start_offset) / 2
            END)
        FROM sys.dm_exec_requests er
        CROSS APPLY sys.dm_exec_sql_text(er.sql_handle) st
        WHERE er.session_id = mg.session_id
    )
FROM sys.dm_exec_query_memory_grants mg
ORDER BY granted_memory_kb DESC;

### Queries Waiting for Memory Grants

`sql
-- Queries currently waiting on RESOURCE_SEMAPHORE
SELECT
    mg.session_id,
    mg.request_id,
    mg.required_memory_kb / 1024 AS required_mb,
    mg.request_time,
    mg.wait_time_ms,
    mg.wait_order,
    mg.is_next_candidate,
    mg.queue_id,
    SUBSTRING(st.text,
        (r.statement_start_offset / 2) + 1,
        CASE
            WHEN r.statement_end_offset = -1 THEN LEN(CONVERT(NVARCHAR(MAX), st.text))
            ELSE (r.statement_end_offset - r.statement_start_offset) / 2
        END
    ) AS statement_text,
    qp.query_plan,
    s.login_name,
    s.host_name,
    s.program_name,
    s.status
FROM sys.dm_exec_query_memory_grants mg
JOIN sys.dm_exec_sessions s
    ON mg.session_id = s.session_id
LEFT JOIN sys.dm_exec_requests r
    ON mg.session_id = r.session_id
       AND mg.request_id = r.request_id
OUTER APPLY sys.dm_exec_sql_text(r.sql_handle) st
OUTER APPLY sys.dm_exec_query_plan(r.plan_handle) qp
WHERE mg.grant_time IS NULL
ORDER BY mg.wait_time_ms DESC;
`

## Queries — Memory Grant Analysis

### Memory Grant Wait Time Distribution

`sql
-- RESOURCE_SEMAPHORE wait trend (requires history table)
IF OBJECT_ID('dbo.ResourceSemaphoreHistory') IS NULL
CREATE TABLE dbo.ResourceSemaphoreHistory (
    capture_time DATETIME2 NOT NULL,
    wait_type NVARCHAR(60) NOT NULL,
    waiting_tasks_count BIGINT,
    wait_time_ms BIGINT,
    CONSTRAINT PK_ResourceSemaphoreHistory PRIMARY KEY (capture_time, wait_type)
);

INSERT INTO dbo.ResourceSemaphoreHistory (capture_time, wait_type, waiting_tasks_count, wait_time_ms)
SELECT GETDATE(), wait_type, waiting_tasks_count, wait_time_ms
FROM sys.dm_os_wait_stats
WHERE wait_type = 'RESOURCE_SEMAPHORE';

-- Trend over last 24 hours
SELECT
    capture_time,
    wait_time_ms,
    waiting_tasks_count,
    wait_time_ms / NULLIF(waiting_tasks_count, 0) AS avg_wait_ms,
    wait_time_ms - LAG(wait_time_ms, 1, 0) OVER (ORDER BY capture_time) AS wait_time_delta,
    waiting_tasks_count - LAG(waiting_tasks_count, 1, 0) OVER (ORDER BY capture_time) AS tasks_delta
FROM dbo.ResourceSemaphoreHistory
WHERE capture_time >= DATEADD(HOUR, -24, GETDATE())
ORDER BY capture_time DESC;
`

### Queries with High Memory Grant Waste

Queries that receive large grants but use very little are wasting memory and causing RESOURCE_SEMAPHORE waits for others.

`sql
-- Queries wasting memory grants (granted much more than used)
SELECT TOP 20
    qs.execution_count,
    qs.total_grant_kb / 1024 AS total_grant_mb,
    qs.total_used_grant_kb / 1024 AS total_used_mb,
    (qs.total_grant_kb - qs.total_used_grant_kb) / 1024 AS total_wasted_mb,
    CASE
        WHEN qs.total_grant_kb = 0 THEN 0
        ELSE (qs.total_grant_kb - qs.total_used_grant_kb) * 100.0 / qs.total_grant_kb
    END AS wasted_pct,
    qs.last_grant_kb / 1024 AS last_grant_mb,
    qs.last_used_grant_kb / 1024 AS last_used_mb,
    qs.min_grant_kb / 1024 AS min_grant_mb,
    qs.max_grant_kb / 1024 AS max_grant_mb,
    SUBSTRING(st.text, 1, 200) AS query_text
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
WHERE qs.total_grant_kb > 0
  AND qs.total_grant_kb > qs.total_used_grant_kb * 2
ORDER BY total_wasted_mb DESC;
`

### Max Server Memory Configuration

`sql
-- Recommended max server memory calculation
-- Rule of thumb: reserve 1-4 GB for OS, 1 GB per 16 GB RAM for other processes
DECLARE @total_physical_memory_gb INT = (
    SELECT physical_memory_kb / 1024 / 1024 FROM sys.dm_os_sys_info
);

DECLARE @os_reservation_gb INT = CASE
    WHEN @total_physical_memory_gb <= 4 THEN 1
    WHEN @total_physical_memory_gb <= 16 THEN 2
    WHEN @total_physical_memory_gb <= 64 THEN 4
    WHEN @total_physical_memory_gb <= 128 THEN 8
    ELSE 16
END;

SELECT
    @total_physical_memory_gb AS total_ram_gb,
    @os_reservation_gb AS os_reservation_gb,
    @total_physical_memory_gb - @os_reservation_gb AS recommended_max_server_memory_gb,
    CEIL((@total_physical_memory_gb - @os_reservation_gb) * 1024) AS recommended_max_server_memory_mb;
`

## Memory Grant Feedback — Adaptive Memory

SQL Server 2017+ introduced **Memory Grant Feedback** (MGF) which adjusts memory grants for repeated queries based on actual usage.

### Memory Grant Feedback in Action

`sql
-- Check which queries have Memory Grant Feedback applied
SELECT
    qp.query_id,
    qp.plan_id,
    qp.physical_operator_name,
    qp.is_adaptive,
    qp.avg_grant_percent,
    qp.avg_ideal_grant_percent,
    qp.avg_grant_percent - qp.avg_ideal_grant_percent AS excess_grant_pct,
    qp.feedback_occurred_count,
    qp.last_feedback_time,
    qp.change_count,
    qt.query_sql_text
FROM sys.query_store_plan_feedback qp
JOIN sys.query_store_query q ON qp.query_id = q.query_id
JOIN sys.query_store_query_text qt ON q.query_text_id = qt.query_text_id
WHERE qp.feedback_occurred_count > 0
ORDER BY qp.feedback_occurred_count DESC;
`

### Memory Grant Feedback Modes

| SQL Server Version | Feature | Description |
|-------------------|---------|-------------|
| 2017 | Batch Mode MGF | Adjusts grants for batch mode queries |
| 2019 | Row Mode MGF | Adjusts grants for row mode queries |
| 2022 | Percentile MGF | Persists feedback across plan recompiles |

`sql
-- Enable/disable Memory Grant Feedback (SQL Server 2019+)
ALTER DATABASE SCOPED CONFIGURATION SET ROW_MODE_MEMORY_GRANT_FEEDBACK = ON;
ALTER DATABASE SCOPED CONFIGURATION SET BATCH_MODE_MEMORY_GRANT_FEEDBACK = ON;

-- Check current configuration
SELECT
    name,
    value,
    value_for_secondary
FROM sys.database_scoped_configurations
WHERE name LIKE '%memory_grant_feedback%';
`

## .NET Integration — Memory Grant Monitoring

### C# Memory Grant Monitor

`csharp
using Microsoft.Data.SqlClient;
using System;
using System.Collections.Generic;
using System.Threading;
using System.Threading.Tasks;

public sealed class MemoryGrantMonitor : IDisposable
{
    private readonly string _connectionString;
    private readonly TimeSpan _pollingInterval = TimeSpan.FromSeconds(30);
    private readonly CancellationTokenSource _cts = new();

    public MemoryGrantMonitor(string connectionString)
    {
        _connectionString = connectionString;
    }

    public Task StartAsync(CancellationToken cancellation = default)
    {
        var combined = CancellationTokenSource.CreateLinkedTokenSource(_cts.Token, cancellation);
        return RunAsync(combined.Token);
    }

    private async Task RunAsync(CancellationToken cancellation)
    {
        var query = @"
SELECT
    rs.resource_semaphore_id,
    rs.total_memory_kb / 1024 AS total_memory_mb,
    rs.available_memory_kb / 1024 AS available_memory_mb,
    rs.granted_memory_kb / 1024 AS granted_memory_mb,
    rs.used_memory_kb / 1024 AS used_memory_mb,
    rs.grantee_count,
    rs.waiter_count,
    rs.timeout_error_count,
    ISNULL(SUM(mg.required_memory_kb), 0) / 1024 AS waiting_queries_required_mb,
    ISNULL(COUNT(CASE WHEN mg.grant_time IS NULL THEN 1 END), 0) AS waiting_queries_count
FROM sys.dm_exec_query_resource_semaphores rs
LEFT JOIN sys.dm_exec_query_memory_grants mg
    ON rs.resource_semaphore_id = mg.resource_semaphore_id
GROUP BY rs.resource_semaphore_id, rs.total_memory_kb, rs.available_memory_kb,
    rs.granted_memory_kb, rs.used_memory_kb, rs.grantee_count, rs.waiter_count,
    rs.timeout_error_count;";

        while (!cancellation.IsCancellationRequested)
        {
            try
            {
                await using var conn = new SqlConnection(_connectionString);
                await conn.OpenAsync(cancellation);

                await using var cmd = new SqlCommand(query, conn);
                await using var reader = await cmd.ExecuteReaderAsync(cancellation);

                var metrics = new List<SemaphoreMetrics>();
                while (await reader.ReadAsync(cancellation))
                {
                    metrics.Add(new SemaphoreMetrics
                    {
                        SemaphoreId = reader.GetInt64(0),
                        TotalMemoryMb = reader.GetInt64(1),
                        AvailableMemoryMb = reader.GetInt64(2),
                        GrantedMemoryMb = reader.GetInt64(3),
                        UsedMemoryMb = reader.GetInt64(4),
                        GranteeCount = reader.GetInt32(5),
                        WaiterCount = reader.GetInt32(6),
                        TimeoutErrorCount = reader.GetInt64(7),
                        WaitingRequiredMb = reader.GetInt64(8)
                    });
                }

                foreach (var m in metrics)
                {
                    var availablePct = m.TotalMemoryMb > 0
                        ? m.AvailableMemoryMb * 100.0 / m.TotalMemoryMb : 0;

                    if (m.WaiterCount > 0 || availablePct < 10)
                    {
                        Console.Error.WriteLine(
                            $"[MEMORY GRANT ALERT] Semaphore {m.SemaphoreId}: " +
                            $"Available={m.AvailableMemoryMb}MB ({availablePct:F1}%), " +
                            $"Waiters={m.WaiterCount}, " +
                            $"WaitingRequired={m.WaitingRequiredMb}MB, " +
                            $"Timeouts={m.TimeoutErrorCount}");
                    }
                }
            }
            catch (Exception ex) when (ex is not OperationCanceledException)
            {
                Console.Error.WriteLine($"[MEMORY MONITOR] {ex.Message}");
            }

            await Task.Delay(_pollingInterval, cancellation);
        }
    }

    public void Dispose() => _cts.Cancel();

    private sealed class SemaphoreMetrics
    {
        public long SemaphoreId { get; set; }
        public long TotalMemoryMb { get; set; }
        public long AvailableMemoryMb { get; set; }
        public long GrantedMemoryMb { get; set; }
        public long UsedMemoryMb { get; set; }
        public int GranteeCount { get; set; }
        public int WaiterCount { get; set; }
        public long TimeoutErrorCount { get; set; }
        public long WaitingRequiredMb { get; set; }
    }
}
`

### ASP.NET Core Health Check for Memory Grants

`csharp
using Microsoft.Data.SqlClient;
using Microsoft.Extensions.Diagnostics.HealthChecks;
using System.Collections.Generic;
using System.Threading;
using System.Threading.Tasks;

public sealed class MemoryGrantHealthCheck : IHealthCheck
{
    private readonly string _connectionString;

    public MemoryGrantHealthCheck(string connectionString)
    {
        _connectionString = connectionString;
    }

    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context,
        CancellationToken cancellation = default)
    {
        await using var conn = new SqlConnection(_connectionString);
        await conn.OpenAsync(cancellation);

        var cmd = new SqlCommand(@"
SELECT
    (SELECT COUNT(*) FROM sys.dm_exec_query_memory_grants WHERE grant_time IS NULL) AS waiting_queries,
    (SELECT MAX(waiter_count) FROM sys.dm_exec_query_resource_semaphores) AS max_waiters,
    (SELECT MIN(available_memory_kb * 1.0 / NULLIF(target_memory_kb, 1))
        FROM sys.dm_exec_query_resource_semaphores) AS min_available_ratio,
    (SELECT SUM(wait_time_ms) FROM sys.dm_os_wait_stats WHERE wait_type = 'RESOURCE_SEMAPHORE') AS total_wait_ms,
    (SELECT COUNT(*) FROM sys.dm_exec_query_memory_grants) AS active_grants;", conn);

        await using var reader = await cmd.ExecuteReaderAsync(cancellation);
        await reader.ReadAsync(cancellation);

        var waitingQueries = reader.GetInt32(0);
        var maxWaiters = reader.GetInt32(1);
        var minAvailableRatio = reader.GetDouble(2);
        var totalWaitMs = reader.GetInt64(3);
        var activeGrants = reader.GetInt32(4);

        var data = new Dictionary<string, object>
        {
            ["WaitingQueries"] = waitingQueries,
            ["MaxWaiters"] = maxWaiters,
            ["MinAvailableRatio"] = minAvailableRatio,
            ["TotalWaitMs"] = totalWaitMs,
            ["ActiveGrants"] = activeGrants
        };

        if (waitingQueries > 5 || maxWaiters > 3 || minAvailableRatio < 0.05)
            return HealthCheckResult.Unhealthy(
                $"Memory grant pressure — {waitingQueries} waiting, " +
                $"min available {minAvailableRatio:P}", data);

        if (waitingQueries > 0 || maxWaiters > 0 || minAvailableRatio < 0.15)
            return HealthCheckResult.Degraded(
                $"Memory grant contention — {waitingQueries} waiting, " +
                $"min available {minAvailableRatio:P}", data);

        return HealthCheckResult.Healthy(
            $"Memory grants healthy — {activeGrants} active grants", data);
    }
}
`

### Programmatic Memory Grant Data Collection

`csharp
using Microsoft.Data.SqlClient;
using System;
using System.Threading.Tasks;

public static class MemoryGrantDiagnostics
{
    public static async Task<MemoryGrantReport> GetMemoryGrantReportAsync(
        string connectionString)
    {
        await using var conn = new SqlConnection(connectionString);
        await conn.OpenAsync();

        var cmd = new SqlCommand(@"
SELECT
    (SELECT SUM(wait_time_ms) FROM sys.dm_os_wait_stats
        WHERE wait_type = 'RESOURCE_SEMAPHORE') AS total_semaphore_wait_ms,
    (SELECT SUM(waiting_tasks_count) FROM sys.dm_os_wait_stats
        WHERE wait_type = 'RESOURCE_SEMAPHORE') AS total_semaphore_waits,
    (SELECT SUM(granted_memory_kb) FROM sys.dm_exec_query_memory_grants) / 1024 AS total_granted_mb,
    (SELECT SUM(required_memory_kb) FROM sys.dm_exec_query_memory_grants
        WHERE grant_time IS NULL) / 1024 AS total_waiting_required_mb,
    (SELECT COUNT(*) FROM sys.dm_exec_query_memory_grants) AS total_active_grants,
    (SELECT COUNT(*) FROM sys.dm_exec_query_memory_grants WHERE grant_time IS NULL) AS total_waiting,
    (SELECT cntr_value FROM sys.dm_os_performance_counters
        WHERE counter_name = 'Total Server Memory (KB)') / 1024 AS total_server_memory_mb,
    (SELECT cntr_value FROM sys.dm_os_performance_counters
        WHERE counter_name = 'Target Server Memory (KB)') / 1024 AS target_server_memory_mb;", conn);

        await using var reader = await cmd.ExecuteReaderAsync();
        await reader.ReadAsync();

        return new MemoryGrantReport
        {
            TotalSemaphoreWaitMs = reader.GetInt64(0),
            TotalSemaphoreWaits = reader.GetInt64(1),
            TotalGrantedMb = reader.GetInt64(2),
            TotalWaitingRequiredMb = reader.GetInt64(3),
            TotalActiveGrants = reader.GetInt32(4),
            TotalWaiting = reader.GetInt32(5),
            TotalServerMemoryMb = reader.GetInt64(6),
            TargetServerMemoryMb = reader.GetInt64(7)
        };
    }
}

public sealed class MemoryGrantReport
{
    public long TotalSemaphoreWaitMs { get; set; }
    public long TotalSemaphoreWaits { get; set; }
    public double AvgWaitMs =>
        TotalSemaphoreWaits > 0
            ? (double)TotalSemaphoreWaitMs / TotalSemaphoreWaits : 0;
    public long TotalGrantedMb { get; set; }
    public long TotalWaitingRequiredMb { get; set; }
    public int TotalActiveGrants { get; set; }
    public int TotalWaiting { get; set; }
    public long TotalServerMemoryMb { get; set; }
    public long TargetServerMemoryMb { get; set; }
    public double MemoryUtilizationPct =>
        TargetServerMemoryMb > 0
            ? TotalServerMemoryMb * 100.0 / TargetServerMemoryMb : 0;
}
`

## Remediation — Reducing Memory Grant Pressure

### Short-Term: Clear Blocked Memory Grants

`sql
-- Identify and kill sessions waiting on memory grants
SELECT
    session_id,
    required_memory_kb / 1024 AS required_mb,
    wait_time_ms,
    request_time
FROM sys.dm_exec_query_memory_grants
WHERE grant_time IS NULL
ORDER BY wait_time_ms DESC;

-- Kill a specific session (use with extreme caution):
-- KILL <session_id>;
`

### Short-Term: Increase Max Server Memory

`sql
-- Increase max server memory if physical RAM is available
EXEC sp_configure 'max server memory (MB)', 8192;
RECONFIGURE;
`

### Medium-Term: Optimize Queries

`sql
-- Find queries causing large memory grants
SELECT TOP 20
    qs.total_grant_kb / 1024 AS total_grant_mb,
    qs.execution_count,
    qs.total_grant_kb / NULLIF(qs.execution_count, 0) / 1024 AS avg_grant_mb,
    qs.last_grant_kb / 1024 AS last_grant_mb,
    qs.max_grant_kb / 1024 AS max_grant_mb,
    SUBSTRING(st.text, 1, 200) AS query_text
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
WHERE qs.total_grant_kb > 0
ORDER BY qs.total_grant_kb DESC;
`

Rewrite large hash joins as loop joins or add indexes to avoid hashing:

`sql
-- Force LOOP JOIN to avoid hash join memory grant
SELECT  o.*, od.*
FROM dbo.Orders o
INNER LOOP JOIN dbo.OrderDetails od
    ON o.OrderID = od.OrderID
WHERE o.OrderDate >= '2024-01-01';
`

### Medium-Term: Update Statistics

`sql
-- Outdated stats cause bad cardinality estimates -> bad memory grants
-- Find tables with stale statistics
SELECT
    sp.stats_id,
    OBJECT_NAME(sp.object_id) AS table_name,
    s.name AS stats_name,
    sp.last_updated,
    sp.rows_sampled,
    sp.rows,
    sp.unfiltered_rows,
    sp.modification_counter,
    CASE
        WHEN sp.modification_counter > sp.rows * 0.2 THEN 'Critical — update immediately'
        WHEN sp.modification_counter > sp.rows * 0.1 THEN 'Warning — update recommended'
        WHEN sp.modification_counter > sp.rows * 0.05 THEN 'Moderate — consider updating'
        ELSE 'OK'
    END AS stats_health
FROM sys.dm_db_stats_properties(OBJECT_ID('dbo.Orders'), NULL) sp
CROSS APPLY (SELECT name FROM sys.stats WHERE object_id = sp.object_id AND stats_id = sp.stats_id) s;

-- Update statistics
UPDATE STATISTICS dbo.Orders WITH FULLSCAN;
UPDATE STATISTICS dbo.OrderDetails WITH FULLSCAN;
`

| Action | Effect | Consideration |
|--------|--------|---------------|
| Increase max server memory | More memory available for grants | Ensure OS has enough RAM |

## Additional — Hash Joins & Sort Memory

### Hash Join Memory Dynamics

Hash joins build a hash table on the smaller input (build input), then probe with the larger input (probe input). Memory requirement = build input size.

`sql
-- Identify hash join operations in plan cache
SELECT TOP 20
    qs.total_elapsed_time / 1000 AS total_elapsed_ms,
    qs.total_grant_kb / 1024 AS grant_mb,
    qs.execution_count,
    qp.query_plan.value('declare namespace p="http://schemas.microsoft.com/sqlserver/2004/07/showplan";
        (//p:Hash)[1]/../@NodeId', 'INT') AS has_hash_join,
    SUBSTRING(st.text, 1, 200) AS query_text
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
CROSS APPLY sys.dm_exec_query_plan(qs.plan_handle) qp
WHERE qp.query_plan.exist('declare namespace p="http://schemas.microsoft.com/sqlserver/2004/07/showplan";
    //p:Hash') = 1
ORDER BY qs.total_grant_kb DESC;
`

### Sort Memory

Sort operations require memory proportional to input size. Spills to tempdb occur when insufficient memory.

`sql
-- Check for sort spills (SQL Server 2019+)
SELECT
    session_id,
    physical_operator_name,
    type_desc,
    spill_count,
    granted_memory_kb,
    used_memory_kb,
    ideal_memory_kb
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
CROSS APPLY sys.dm_exec_query_plan(qs.plan_handle) qp
CROSS APPLY (
    SELECT
        n.value('@PhysicalOp', 'NVARCHAR(50)') AS physical_operator_name,
        n.value('@NodeId', 'INT') AS node_id
    FROM qp.query_plan.nodes('//*:RelOp') AS r(n)
    WHERE n.value('@PhysicalOp', 'NVARCHAR(50)') IN ('Sort', 'Hash Match')
) ops
WHERE qp.query_plan.exist('//*:SpillToTempDb') = 1;
`

### Spill to Tempdb

When memory grants are insufficient, sort/hash spills to tempdb — drastically reducing performance and increasing IO:

`sql
-- Tempdb spills — sort/hash warnings in plan cache
SELECT
    qs.execution_count,
    qs.total_elapsed_time / 1000 AS total_elapsed_ms,
    qs.total_grant_kb / 1024 AS grant_mb,
    qs.total_worker_time / 1000 AS total_cpu_ms,
    st.text AS query_text,
    'Spill detected' AS warning
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
CROSS APPLY sys.dm_exec_query_plan(qs.plan_handle) qp
WHERE qp.query_plan.exist('//*:SpillToTempDb') = 1
ORDER BY qs.total_elapsed_time DESC;
`

-- Quick one-liner: Memory grant health summary
SELECT
    CASE
        WHEN waiting_count = 0 AND available_pct > 25 THEN 'Healthy'
        WHEN waiting_count = 0 AND available_pct > 10 THEN 'Adequate'
        WHEN waiting_count <= 2 THEN 'Moderate — some queries queued'
        WHEN waiting_count <= 5 THEN 'Elevated — investigate'
        ELSE 'Critical — immediate action needed'
    END AS memory_grant_status,
    waiting_count,
    granted_count,
    CAST(available_pct AS DECIMAL(5,1)) AS available_pct

        (SELECT COUNT(*) FROM sys.dm_exec_query_memory_grants WHERE grant_time IS NULL) AS waiting_count,
        (SELECT COUNT(*) FROM sys.dm_exec_query_memory_grants WHERE grant_time IS NOT NULL) AS granted_count,
        (SELECT AVG(available_memory_kb * 100.0 / NULLIF(target_memory_kb, 1))
            FROM sys.dm_exec_query_resource_semaphores) AS available_pct
) AS summary;
